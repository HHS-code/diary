# Step 2: asset-import-panel

## 이 단계를 왜 했나?

Step 0은 창고(IndexedDB)를 만들었고, Step 1은 예전 사진들을 그 창고로 이사시켰다. 하지만 지금까지는 사용자가 "새 사진 파일을 직접 골라서 창고에 넣는" 화면 자체가 없었다. 배경 이미지는 여전히 예전 버튼 하나로만 등록할 수 있고, 폰트는 등록할 방법이 아예 없었다.

이번 phase의 다음 phase(스티커 전용 화면)도 결국 "이미지·폰트 파일을 여러 개 한꺼번에 등록"하는 같은 화면을 필요로 한다. 그래서 이번엔 배경이나 폰트 한 곳에만 붙는 전용 버튼이 아니라, 어디서든 갖다 붙일 수 있는 "에셋 등록 부품"을 하나 만들었다.

## 무엇을 만들었나?

두 개의 파일을 새로 만들었다.

**`useAssetLibrary` (등록 창구)** — 지금까지 창고(Step 0의 `assetStorage`)에 등록된 이미지·폰트 목록을 화면에 띄워주고, 새 파일을 등록하는 함수를 제공하는 부품이다. 파일 하나를 넘기면 이미지인지 폰트인지 확장자로 스스로 구분해서(jpg/png/gif/webp → 이미지, ttf/otf/woff/woff2 → 폰트, 그 외는 무시) 창고에 저장하고, 등록이 끝나면 목록을 새로고침한다. 폰트를 등록할 때는 파일 이름(확장자 뺀 부분, 예: `MyFont.ttf` → `MyFont`)을 폰트 이름으로 쓰는데, 이미 같은 이름이 있으면 뒤에 짧은 구분값(예: `MyFont-3f2a`)을 붙여 겹치지 않게 한다.

**`AssetImportPanel` (등록 화면)** — 실제로 사용자가 보는 패널이다. 안에 "파일 선택" 버튼, "폴더 선택" 버튼, 그리고 파일을 직접 끌어다 놓을 수 있는 영역이 있다. 세 가지 입력 경로 모두 결국 같은 분류 로직(`classifyAssetType`)을 거쳐 `useAssetLibrary`의 등록 함수를 호출한다. 이 패널은 `useAssetLibrary`를 직접 호출하지 않고, 바깥에서 만들어진 결과를 건네받아 쓴다 — 그래야 나중에 배경 설정 화면과 폰트 목록 화면이 같은 등록 창구(같은 목록 상태)를 공유할 수 있다.

폴더를 통째로 드래그해서 떨어뜨리는 경우(폴더 자체를 드롭)까지는 이번 단계에서 다루지 않는다 — 브라우저가 폴더 내부를 재귀적으로 읽어내는 처리가 꽤 복잡해서, 여러 파일을 한 번에 등록하고 싶으면 "폴더 선택" 버튼을 쓰면 된다는 합의로 범위를 좁혔다.

## 데이터가 어떻게 흐르나?

```
사용자 입력
  ├─ 파일 선택 버튼 클릭 → 파일 여러 개 고름
  ├─ 폴더 선택 버튼 클릭 → 폴더 안 파일 전부(하위 폴더 포함) 고름
  └─ 패널 위로 파일을 끌어다 놓음(드래그앤드롭)
        │
        ▼
   파일 하나씩 확장자 확인 (classifyAssetType)
        │
        ├─ 이미지 확장자 ──▶ registerImage(file)
        ├─ 폰트 확장자   ──▶ registerFont(file)
        └─ 그 외         ──▶ 무시 (등록 안 함)
        │
        ▼
   assetStorage.saveAsset(...)로 창고에 저장, 번호(id) 발급
        │
        ▼
   목록 새로고침 → 패널에 "이미지 (N)" / "폰트 (N)" 개수와 파일명 나열
```

예를 들어 사용자가 `cat.png`, `MyFont.ttf`, `notes.txt` 3개 파일을 한꺼번에 드롭하면: `cat.png`는 이미지로 창고에 저장되고, `MyFont.ttf`는 폰트로 저장되며(폰트 이름 `MyFont`), `notes.txt`는 조용히 무시되어 창고에 아무것도 남지 않는다. 패널에는 "이미지 (1)"과 "폰트 (1)" 아래 각각 파일명이 하나씩 표시된다.

## 쉽게 말하면

Step 0~1이 창고와 이삿짐 트럭이었다면, 이번 step은 창고 앞에 세워둔 "접수대"다. 택배로 보내든(파일 선택), 이삿짐 박스째로 갖고 오든(폴더 선택), 그냥 손에 들고 와서 던져놓든(드래그앤드롭) — 접수대 직원(`classifyAssetType`)이 사진인지 폰트 파일인지 한눈에 보고 알아서 맞는 칸에 넣어준다. 편지봉투(.txt 같은 엉뚱한 파일)를 들고 오면 받지 않고 그냥 돌려보낸다.

## 만든 파일

- `src/hooks/useAssetLibrary.js` — 창고 목록 상태 + 등록 함수(`registerImage`, `registerFont`), 확장자 분류 함수(`classifyAssetType`)
- `src/hooks/useAssetLibrary.test.jsx` — 등록하면 목록에 반영되는지, 폰트 이름이 겹치면 구분값이 붙는지 확인하는 자동 테스트
- `src/components/AssetImportPanel/AssetImportPanel.jsx` — 파일 선택/폴더 선택/드래그앤드롭 3경로를 갖춘 등록 화면
- `src/components/AssetImportPanel/AssetImportPanel.test.jsx` — 이미지+폰트를 동시에 넣으면 각각 맞는 등록 함수가 불리는지, 지원하지 않는 확장자는 무시되는지 확인하는 자동 테스트

## 제대로 됐는지 어떻게 확인했나?

자동 테스트를 새로 6개 작성해서 전부 통과시켰다.

`useAssetLibrary.test.jsx` 3개: 이미지를 등록하면 이미지 목록에만 반영되는지, 폰트를 등록하면 폰트 목록에 반영되고 파일명이 폰트 이름 기본값으로 쓰이는지, 같은 이름 폰트를 두 번 등록하면 두 번째에 구분값이 붙어 서로 다른 이름이 되는지.

`AssetImportPanel.test.jsx` 3개: 파일 선택창에 이미지 1개+폰트 1개를 동시에 넣으면 등록 함수가 각각 정확히 1번씩, 맞는 파일과 함께 불리는지, 지원하지 않는 확장자(.txt)는 등록 함수가 아예 불리지 않는지, 등록된 파일명이 화면 목록에 표시되는지.

이번 phase 완료 기준인 `npm run lint`, `npm run build`, `npm run test`를 모두 실행해 통과를 확인했다(기존 97개 + 새로 만든 6개 = 총 103개, 전부 통과). 다만 전체 테스트를 여러 번 돌리는 과정에서 이번 step과 무관한 두 파일(`useCanvasHistory.test.js`, `useCanvasBackground.test.js`)이 실행마다 번갈아 가며 5초 제한 시간 초과로 1개씩 실패하는 걸 봤다 — 둘 다 이번 step이 건드리지 않은 파일이고, 각각 따로 떼어 실행하면 문제없이 통과했다. 여러 테스트가 동시에 도는 상황에서 컴퓨터가 잠깐 느려져 생기는 우연한 현상으로 보이며, 이번 step에서 만든 코드와는 관련이 없다.

## 전체 그림에서 지금 어디?

```
Step 0: 창고 만들기 (완료)
        │
        ▼
Step 1: 기존 데이터 이사시키기 (완료)
        │
        ▼
[Step 2: 파일 등록 화면 만들기] ← 지금 여기, 완료
        │
        ▼
Step 3: 업로드한 폰트를 실제로 쓸 수 있게 등록
        │
        ▼
Step 4: 배경 이미지 설정을 새 창고에 연결
        │
        ▼
Step 5: Ctrl+V로 이미지 붙여넣기
```

이 접수대(등록 화면)를 창고·이사 다음, 배경 이미지 연결보다 먼저 만든 이유는 다음 두 step(폰트 등록, 배경 이미지 연결)이 결국 "파일을 받아서 창고에 넣는" 이 부품을 그대로 갖다 쓰기 때문이다. 접수대가 먼저 있어야 그 뒤에 오는 화면들이 접수 로직을 새로 만들지 않고 재사용할 수 있다.

## 아직 안 된 것 (솔직하게)

- 이 패널은 아직 화면 어디에도 붙어 있지 않다. 만들기만 했고, 실제 배경 설정 화면이나 텍스트 메모 화면에 연결하는 건 Step 3(폰트), Step 4(배경 이미지) 몫이다.
- 업로드한 폰트 파일을 실제로 브라우저가 폰트로 쓸 수 있게 만드는 작업(FontFace 등록)은 이번 step에 없다 — 지금은 창고에 파일만 저장될 뿐, 아직 화면에 그 폰트로 글자가 보이지는 않는다.
- 클립보드 붙여넣기(Ctrl+V)로 이미지를 등록하는 경로는 이번 step에 없다 (Step 5 몫).
- 등록된 파일을 지우거나 관리하는 화면은 없다 — 이번 phase는 "등록"까지만 다루기로 미리 정해둔 범위다.
- 폴더 자체를 통째로 드래그해서 떨어뜨리는 경우는 지원하지 않는다. 여러 파일을 한 번에 등록하려면 "폴더 선택" 버튼을 쓰면 된다.
