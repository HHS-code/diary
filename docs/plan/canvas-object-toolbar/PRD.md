# PRD — 캔버스 오브젝트 조작 툴바

## 배경 / 왜

지금 `DiaryCanvas`는 스티커·이미지·텍스트를 캔버스에 올릴 수는 있지만, 올린 뒤 다루는 도구가 부족하다. PPT에서 도형 하나를 다룰 때처럼 자연스러운 조작(복사, 삭제, 정렬, 레이어 순서, 회전)이 필요하다.

## 목표

- 캔버스 위 오브젝트(스티커/이미지/텍스트)를 선택하면 조작용 미니 툴바가 뜬다.
- 복사, 삭제, 레이어 순서 변경이 버튼 클릭으로 가능하다.
- 여러 오브젝트를 선택하면 정렬·등간격 배치가 가능하다.
- 자주 쓰는 동작(삭제, 복사/붙여넣기)은 키보드 단축키로도 가능하다.

## 대상 사용자

기존과 동일. 다이어리 편집 화면(`DiaryCanvas`)을 쓰는 개인 사용자.

## In-scope

- **플로팅 미니 툴바**: 오브젝트를 선택하면 해당 오브젝트 근처에 뜨는 작은 버튼 묶음. 선택 해제 시 사라진다.
- **복사**: 선택된 오브젝트를 복제해 원본에서 살짝 오프셋된 위치에 추가한다.
- **삭제**: 선택된 오브젝트를 캔버스에서 제거한다.
- **레이어 순서**: 맨 앞으로, 맨 뒤로, 한 단계 앞으로, 한 단계 뒤로 — 4개 버튼.
- **회전**: 별도 버튼 없음. Fabric.js 기본 회전 핸들(오브젝트 모서리 드래그)을 그대로 사용한다.
- **정렬 (다중 선택 시)**: 좌/우/상/하/가운데 정렬 + 등간격 배치(가로/세로).
- **키보드 단축키**: `Delete`(삭제), `Ctrl+C`/`Ctrl+V`(복사/붙여넣기).
- **스타일**: `docs/design.md`에 정의된 기존 XP 버튼·패널 스펙을 그대로 재사용한다. 새 색상/그라디언트/보더 값을 정의하지 않는다.

## Out-of-scope (명시)

- 그룹핑 / 그룹 해제
- 다중 선택 시 크기 일괄 조정
- 정렬 가이드라인(스냅)
- 회전 전용 버튼(각도 직접 입력 등) — 기본 핸들로 충분
- `DiaryCanvas` 외 다른 캔버스 관련 기능 변경 (스티커 팔레트, 이미지 업로드, 텍스트 메모, export/import는 그대로)

## 성공 기준

- 캔버스에서 오브젝트 하나를 클릭하면 근처에 미니 툴바가 뜨고, 복사·삭제·레이어 순서 버튼이 동작한다.
- 오브젝트 모서리를 드래그하면 회전된다 (기존 Fabric.js 동작 그대로).
- 오브젝트를 두 개 이상 선택(Shift+클릭 또는 드래그 선택)하면 정렬/등간격 버튼이 나타나고 동작한다.
- 오브젝트 선택 후 `Delete` 키로 삭제되고, `Ctrl+C`/`Ctrl+V`로 복사·붙여넣기가 된다.
- 툴바 버튼 스타일이 기존 XP 룩(다른 버튼들)과 이질감 없다.

## 진행 방식

1. 문서(PRD/architecture) 합의 → 실행 승인
2. 툴바 컴포넌트 + 선택 상태 연동 → 레이어/복사/삭제 → 정렬/등간격 → 키보드 단축키 → 완료 시 사용자에게 결과 설명
