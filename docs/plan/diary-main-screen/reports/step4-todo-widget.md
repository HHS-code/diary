# Step 4: todo-widget

## 이 단계를 왜 했나?

메인 화면 우측 위젯 3개(할 일, 날씨, 시계) 중 마지막 남은 할 일 체크리스트 위젯을 이번 단계에서 만든다. 선택한 날짜별로 독립된 할 일 목록을 추가·체크·삭제할 수 있게 하는 부품이다. 지난 단계들과 마찬가지로 이 위젯 하나만 독립적으로 완성하고, 실제 diary 화면에 붙이는 건 하지 않는다(5단계에서 한다).

## 무엇을 만들었나?

`src/components/TodoWidget/TodoWidget.jsx`라는 새 화면 부품을 만들었다.

- 바깥에서 `selectedDate`(예: `"2026-07-11"`) 하나만 값으로 받는다. 이 값이 어떤 날짜의 할 일을 보여줄지 결정한다.
- 화면에 놓이면 그 날짜에 저장된 할 일 목록을 읽어와 보여준다. 아직 아무것도 저장된 적 없는 날짜라면 빈 목록으로 시작한다.
- 입력창에 글자를 쓰고 "추가" 버튼(또는 Enter 키)을 누르면 목록 맨 아래에 새 할 일이 생기고, 그 순간 바로 저장된다(따로 저장 버튼 없음).
- 각 할 일 앞의 체크박스를 누르면 완료 표시(취소선 + 흐린 글자색)로 바뀌고, 다시 누르면 원래대로 돌아온다. 이것도 누르는 즉시 저장된다.
- 각 할 일 옆의 "삭제" 버튼을 누르면 목록에서 바로 사라지고, 그 순간 저장된다.
- `selectedDate`가 다른 날짜로 바뀌면(예: 캘린더에서 다른 날을 고르면) 이전 날짜의 목록은 사라지고 새 날짜의 목록으로 통째로 바뀐다. 날짜마다 완전히 독립된 목록이다.
- 저장은 항상 `storage/diaryStorage.js`의 기존 함수(`getDateTodos`/`setDateTodos`)를 거쳐서만 이뤄진다 — 이 위젯이 브라우저 저장소를 직접 건드리는 부분은 없다.
- 전체를 날씨·시계 위젯과 같은 톤(진한 파란 테두리 + 베이지 배경 + 그림자)의 XP 스타일 카드로 감쌌다.

## 데이터가 어떻게 흐르나?

```
TodoWidget이 화면에 뜨거나 selectedDate가 바뀔 때
        │
        ▼
저장소 전체를 읽음 (loadAllDiaryData)
        │
        ▼
그 날짜(selectedDate)의 할 일만 꺼냄 (getDateTodos)
  예: "2026-07-11" → [ { 빨래하기, 완료아님 } ]
        │
        ▼
화면에 목록으로 표시

──────────── 사용자가 추가/체크/삭제할 때 ────────────

새 할 일 생성 또는 기존 목록 수정 (메모리 상에서)
        │
        ▼
저장소 전체를 다시 읽음 (loadAllDiaryData)
        │
        ▼
그 날짜의 할 일만 갱신한 새 저장소 데이터 만듦 (setDateTodos)
  — 같은 날짜의 다이어리/영화 데이터는 그대로 보존됨
        │
        ▼
저장소 전체를 다시 씀 (saveAllDiaryData)
```

실제로 "빨래하기"를 추가했더니 목록에 바로 나타났고, 페이지를 새로고침해도 그대로 남아있었다. 체크박스를 눌렀더니 취소선이 붙었고, 새로고침 후에도 체크된 채로 유지됐다. 삭제 버튼을 눌렀더니 목록에서 사라졌고, 새로고침 후에도 다시 나타나지 않았다. `selectedDate`를 `"2026-07-12"`로 바꿔서 띄웠더니 그 날짜엔 아무것도 없는 빈 목록이 보였다(2026-07-11에 있던 "빨래하기"와 완전히 별개).

## 쉽게 말하면

이 위젯은 "날짜별로 서랍이 나뉜 메모지 묶음"과 같다. 어떤 날짜 서랍을 열지는 바깥(`selectedDate`)에서 정해주고, 이 위젯은 그 서랍 안의 메모지들만 꺼내 보여준다. 메모를 새로 쓰거나, 줄을 그어 지우거나(완료 체크), 아예 찢어버리면(삭제) 그 즉시 서랍에 다시 넣어둔다 — 나중에 몰아서 정리하는 게 아니라 손이 닿는 순간 바로 정리하는 방식이다.

## 만든 파일

- `src/components/TodoWidget/TodoWidget.jsx` (신규) — 선택된 날짜의 할 일 목록을 추가·체크·삭제할 수 있는 위젯.

## 제대로 됐는지 어떻게 확인했나?

1. **린트(코드 스타일 자동 검사) 통과**: `npm run lint`를 돌려 에러 없이 통과했다.
2. **브라우저에서 직접 확인**: 개발 서버를 띄우고 diary 창 안에 할 일 위젯을 임시로 띄워 자동화 브라우저(Playwright)로 확인했다(확인 후 임시 코드는 원상복구해 이번 커밋에는 남기지 않았다).
   - **추가 + 새로고침 유지**: "빨래하기"를 입력하고 추가 버튼을 눌렀더니 목록에 나타났고, 페이지를 새로고침해도 그대로 있었다 — 확인됨.
   - **체크 + 새로고침 유지**: 체크박스를 눌렀더니 취소선이 붙었고, 새로고침 후에도 체크된 상태가 유지됐다 — 확인됨.
   - **삭제 + 새로고침 유지**: 삭제 버튼을 눌렀더니 목록에서 사라졌고, 새로고침 후에도 다시 나타나지 않았다(빈 목록) — 확인됨.
   - **다른 날짜는 독립된 목록**: `selectedDate`를 다른 날짜로 바꿔 띄웠더니 그 날짜엔 아무 항목도 없는 빈 목록이 보였다 — 확인됨.
   - 모든 경로에서 콘솔 에러 없었다.

## 전체 그림에서 지금 어디?

```
[0] storage-todos (완료)
[1] xp-calendar (완료)
[2] analog-clock-widget (완료)
[3] weather-widget (완료)
[4] todo-widget (완료)   <- 지금 여기
 └─ 5  main-screen-wiring    <- 여기서 캘린더 + 위젯 3개를 한 화면에 모아 조립한다
```

우측 위젯 3개(시계, 날씨, 할 일)가 모두 독립적으로 완성됐다. 다음 단계(5단계)에서 이 셋과 XP 캘린더를 하나의 메인 화면으로 조립하고, 날짜 클릭 시 편집 화면으로 넘어가는 화면 전환까지 연결한다. 지금 diary 창을 열어도 아직 이 할 일 위젯은 보이지 않는 게 정상이다.

## 아직 안 된 것 (솔직하게)

- 실제 diary 화면에는 아직 연결되지 않았다. 지금 diary 창을 열면 할 일 위젯이 보이지 않는다. 연결은 5단계에서 한다.
- 우선순위, 마감일, 반복 일정, 알림 같은 기능은 없다 — 계획 문서에 "추가/체크/삭제만"으로 범위가 정해져 있어서 그 범위만 지켰다.
- 항목이 아주 많아졌을 때 목록이 얼마나 길어질 수 있는지, 스크롤이 필요한지에 대한 처리는 따로 하지 않았다 — 계획 문서에 이 부분에 대한 언급이 없어서 최소 범위로만 구현했다.
