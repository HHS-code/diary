# Step 0: storage-todos

## 이 단계를 왜 했나?

다이어리 메인 화면에는 날짜별로 할 일 체크리스트가 들어갈 예정이다. 이 체크리스트가 새로고침해도 사라지지 않으려면, 브라우저에 데이터를 저장하고 다시 꺼내오는 기능이 먼저 있어야 한다. 이번 단계는 그 저장/조회 기능만 만드는 단계다. 화면에 보이는 체크박스나 입력창은 아직 없다 — 그건 다음 단계들의 몫이다.

## 무엇을 만들었나?

`src/storage/diaryStorage.js`(다이어리 데이터를 브라우저 저장소인 localStorage에 읽고 쓰는 파일)에 함수 두 개를 추가했다.

- **할 일 목록 꺼내기**: 특정 날짜의 할 일 목록을 가져온다. 그 날짜에 할 일이 하나도 없으면 빈 목록을 돌려준다 (에러가 나지 않는다).
- **할 일 목록 저장하기**: 특정 날짜의 할 일 목록을 새로 저장한다. 이때 그 날짜에 이미 있던 다이어리 글이나 영화 기록은 건드리지 않고 그대로 남겨둔다.

할 일 항목 하나는 "고유 번호, 내용 글자, 완료 여부" 세 가지로 이루어진다. 예: `{ id: "a", text: "빨래하기", done: false }`.

## 데이터가 어떻게 흐르나?

```
저장 전 (2026-07-11에 다이어리 글만 있던 상태)
{
  "2026-07-11": {
    "diary": { "canvasJSON": {...} }
  }
}

           |  setDateTodos(data, "2026-07-11", [할일 목록])
           v

저장 후
{
  "2026-07-11": {
    "diary": { "canvasJSON": {...} },   <- 그대로 유지됨
    "todos": [
      { "id": "a", "text": "빨래하기", "done": false }
    ]
  }
}
```

숫자로 예를 들면: 2026-07-11에 다이어리 글이 하나 저장되어 있는 상태에서 "빨래하기" 할 일 1개를 추가하면, 그 날짜 데이터 안에 `diary` 필드는 그대로 있고 `todos` 필드만 새로 생긴다. 다이어리 글이 사라지는 일은 없다.

## 쉽게 말하면

하루치 다이어리 데이터를 서랍 하나라고 생각하면 된다. 서랍 안에는 "다이어리 칸", "영화 칸"이 이미 있었다. 이번에 "할 일 칸"을 하나 더 만든 것이다. 할 일 칸에 물건(할 일 목록)을 새로 넣어도 옆에 있는 다이어리 칸, 영화 칸은 건드리지 않는다.

## 만든 파일

- `src/storage/diaryStorage.js` (수정) — `getDateTodos`, `setDateTodos` 두 함수 추가.

## 제대로 됐는지 어떻게 확인했나?

두 가지를 확인했다.

1. **린트(코드 스타일 자동 검사) 통과**: `npm run lint`를 돌려 에러 없이 통과했다.
2. **동작 확인**: 임시 스크립트로 다음을 직접 실행해 결과를 눈으로 확인했다 (이 프로젝트에는 아직 자동 테스트 도구가 설치되어 있지 않아, 계획 문서에 적힌 대로 Node 스크립트로 검증했다).
   - 빈 데이터에 할 일을 저장하면 `{ "2026-07-11": { "todos": [...] } }` 형태가 나온다 — 확인됨.
   - 이미 다이어리 글이 있는 날짜에 할 일을 저장해도 그 다이어리 글이 사라지지 않는다 — 확인됨.
   - 할 일이 없는 날짜를 조회하면 빈 목록 `[]`이 나온다 — 확인됨.
   - 원본 데이터는 함수 호출 후에도 변경되지 않는다(항상 새 데이터를 돌려준다) — 확인됨.

## 전체 그림에서 지금 어디?

```
[0] storage-todos (완료)   <- 지금 여기
 └─ 1  xp-calendar
 └─ 2  analog-clock-widget
 └─ 3  weather-widget
 └─ 4  todo-widget          <- 이 단계에서 만든 함수를 실제로 사용하는 곳
 └─ 5  main-screen-wiring
```

이번 단계는 "저장 창고"를 만든 것이다. 다음 단계들은 눈에 보이는 화면(캘린더, 시계, 날씨, 할 일 체크박스)을 하나씩 만들고, 그중 4번 단계(todo-widget)에서 오늘 만든 저장 기능을 가져다 실제 체크리스트 화면에 연결한다.

## 아직 안 된 것 (솔직하게)

- 화면에 보이는 할 일 체크리스트 UI는 아직 없다. 지금은 저장/조회 함수만 있고, 이걸 부르는 화면은 4단계(todo-widget)에서 만든다.
- 할 일 항목의 고유 번호(id)를 실제로 만들어주는 코드는 이 단계에 없다. 그건 체크리스트 화면(4단계)에서 항목을 추가할 때 만들기로 정해져 있다.
- 이 프로젝트에는 자동 테스트 도구(예: 코드가 바뀔 때마다 자동으로 재검증해주는 장치)가 아직 설치되어 있지 않다. 그래서 이번 확인은 자동화된 테스트가 아니라 사람이 스크립트를 돌려 눈으로 확인한 것이다.
