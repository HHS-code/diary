# Step 1: xp-calendar

## 이 단계를 왜 했나?

지금 diary 창을 열면 네모 칸이 격자로 늘어선 평범한 캘린더가 보인다. 이걸 Windows XP 시절 바탕화면에 있던 달력 위젯 스타일(파란 헤더 + 카드형 날짜 칸 + 오늘 날짜 노란 강조)로 바꾸는 작업의 첫 단계다. 이번 단계는 그 "XP 스타일 달력 부품" 하나만 독립적으로 만든다. 아직 실제 diary 화면에 끼워 넣지는 않는다 — 그건 5단계(main-screen-wiring)의 몫이다.

## 무엇을 만들었나?

`src/components/XPCalendar/XPCalendar.jsx`라는 새 화면 부품(컴포넌트, 재사용 가능한 화면 조각 단위)을 만들었다.

- 맨 위에 파란 그라데이션 헤더 바 — 가운데 "2026년 7월"처럼 연도·월을 표시하고, 양옆에 ◀▶ 버튼으로 월을 이동한다.
- 그 아래 요일 행(일~토), 그 아래 날짜가 카드 모양 칸으로 나열된 격자.
- 오늘 날짜 칸은 노란 배경으로 강조된다.
- 오늘이 아니면서 사용자가 선택한 날짜는 파란 테두리 박스로 표시된다.
- 날짜 칸을 클릭하면 그 날짜를 부모 화면에 알려준다 (실제로 무슨 일이 일어나는지는 부모가 정한다 — 5단계에서 "편집 화면으로 전환"으로 연결될 예정).
- 이 부품은 "지금 몇 년 몇 월을 보여줄지"를 스스로 기억하지 않는다. 부모 화면이 그 값을 넘겨주고, ◀▶를 누르면 "이 연도, 이 월로 바꿔달라"고 부모에게 요청만 한다. 이렇게 만든 이유는, 나중에 날짜 클릭으로 편집 화면에 들어갔다가 다시 돌아왔을 때도 보고 있던 달이 그대로 유지되게 하기 위해서다(부모가 값을 계속 들고 있으니까).

## 데이터가 어떻게 흐르나?

```
부모 화면(나중에 DiaryApp이 됨)
  │
  │  currentYear=2026, currentMonth=6(7월, 0부터 셈),
  │  selectedDate="2026-07-11"
  ▼
XPCalendar
  │
  │  화면에 "2026년 7월" 표시 + 날짜 칸 그리기
  │  (7/11 칸은 오늘이라 노란 배경)
  │
  ├─ 사용자가 ◀ 클릭
  │     └─ onChangeMonth(2026, 5) 호출  → 부모가 "6월 보여줘" 요청 받음
  │
  └─ 사용자가 날짜 칸(예: 7/18) 클릭
        └─ onSelectDate("2026-07-18") 호출 → 부모가 "이 날짜 선택됨" 통보 받음
```

숫자로 예를 들면: 화면에 "2026년 7월"이 떠 있는 상태에서 ◀를 누르면 XPCalendar는 스스로 화면을 바꾸지 않고, 그저 "2026년 6월로 바꿔줘"라는 요청(`onChangeMonth(2026, 5)`, 월은 0부터 세므로 6월=5)만 부모에게 보낸다. 부모가 그 요청을 받아 실제로 연/월 값을 6월로 바꿔주면, 그제서야 XPCalendar가 6월 화면을 다시 그린다.

## 쉽게 말하면

XPCalendar는 "달력 그림만 그리는 화가"라고 생각하면 된다. 화가는 "지금 몇 월을 그릴지", "어느 날짜에 동그라미를 칠지"를 스스로 정하지 않는다. 의뢰인(부모 화면)이 "7월을 그려줘, 11일에 동그라미 쳐줘"라고 알려주면 그대로 그린다. 대신 사용자가 화살표 버튼을 누르면 화가는 그림을 마음대로 바꾸는 대신 의뢰인에게 "8월로 바꿔달라는 요청이 왔어요"라고 전달만 한다.

## 만든 파일

- `src/components/XPCalendar/XPCalendar.jsx` (신규) — XP 스타일 월간 캘린더 부품.

## 제대로 됐는지 어떻게 확인했나?

1. **린트(코드 스타일 자동 검사) 통과**: `npm run lint`를 돌려 에러 없이 통과했다.
2. **브라우저에서 직접 확인**: 개발 서버를 띄우고 화면에 XPCalendar를 임시로 띄워 눈으로 확인했다(확인 후 임시 코드는 원상복구해 이번 커밋에는 남기지 않았다).
   - 파란 헤더 + ◀▶ 버튼, 요일 행, 카드형 날짜 격자가 레퍼런스로 준 XP 달력 위젯 스크린샷과 비슷한 레이아웃으로 보였다 — 확인됨.
   - 오늘 날짜(2026-07-11) 칸이 노란색으로 표시됐다 — 확인됨.
   - ◀ 버튼을 눌렀더니 콘솔에 `2026 5`가 찍혔다 — 7월(6)에서 한 달 전인 6월(5)로 바꿔달라는 요청이 정확한 값으로 전달됐다는 뜻 — 확인됨.
   - 날짜 칸 "11"을 클릭했더니 콘솔에 `2026-07-11`이 찍혔다 — 날짜 선택 요청이 정확한 형식으로 전달됐다는 뜻 — 확인됨.

## 전체 그림에서 지금 어디?

```
[0] storage-todos (완료)
[1] xp-calendar (완료)        <- 지금 여기
 └─ 2  analog-clock-widget
 └─ 3  weather-widget
 └─ 4  todo-widget
 └─ 5  main-screen-wiring      <- 여기서 XPCalendar를 실제 diary 화면에 끼운다
```

이번 단계는 "달력 부품 하나"만 따로 완성한 것이다. 나머지 위젯(시계, 날씨, 할 일)도 같은 방식으로 각자 독립적으로 만든 뒤, 마지막 5단계에서 한 화면에 전부 조립한다. 지금 당장 diary 창을 열어봐도 옛날 캘린더가 그대로 보이는 게 정상이다 — 아직 안 바꿨기 때문이다.

## 아직 안 된 것 (솔직하게)

- 실제 diary 화면에는 아직 연결되지 않았다. 지금 diary 창을 열면 여전히 기존의 사각 격자 캘린더가 보인다. XPCalendar로 교체하는 작업은 5단계에서 한다.
- 기존 사각 격자 캘린더(`Calendar.jsx`) 파일은 아직 지우지 않았다. 삭제도 5단계에서 한다.
- 연/월을 드롭다운으로 점프하는 기능이나 헤더를 눌러 나오는 팝업 같은 건 만들지 않았다 — 계획 문서에 "◀▶ 버튼으로만 이동"이라고 명시돼 있어서 그 범위만 지켰다.
