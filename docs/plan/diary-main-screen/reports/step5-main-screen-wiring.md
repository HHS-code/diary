# Step 5: main-screen-wiring

## 이 단계를 왜 했나?

지난 네 단계에서 부품들(할 일 위젯, 날씨 위젯, 시계 위젯, XP 캘린더)을 각각 따로 만들었다. 하지만 이것들은 서로 연결되어 있지 않아서, 지금 diary 창을 열어도 예전 그대로의 사각 캘린더+편집 화면만 보인다. 이번 단계는 그 부품들을 실제로 한 화면에 모아 조립하고, "캘린더 화면 ↔ 편집 화면"을 오가는 전환 동작을 완성하는 마지막 단계다.

## 무엇을 만들었나?

`src/components/DiaryApp/DiaryApp.jsx`(diary 창 전체를 담당하는 부품)를 고쳤다.

- diary 창이 지금 "캘린더 화면"인지 "편집 화면"인지를 기억하는 값(`screen`)을 하나 추가했다. 창을 처음 열면 항상 캘린더 화면(`'main'`)부터 보인다.
- 캘린더가 지금 몇 년 몇 월을 보여주고 있는지(`currentYear`, `currentMonth`)를 diary 창이 직접 기억하도록 했다. 캘린더 부품 자신은 이 값을 기억하지 않고, diary 창이 넘겨주는 값을 그대로 보여주기만 한다 — 그래야 편집 화면에 갔다가 돌아와도 몇 월을 보고 있었는지 잊어버리지 않는다.
- 캘린더 화면일 때: 왼쪽엔 XP 스타일 캘린더, 오른쪽엔 할 일·날씨·시계 위젯 3개를 세로로 나란히 놓았다.
- 캘린더에서 날짜를 하나 클릭하면: 그 날짜를 "선택된 날짜"로 기억하고, 화면을 편집 화면으로 바꾼다.
- 편집 화면일 때: 예전과 똑같이 날짜 제목 + 탭(다이어리/영화리뷰) + 그림판이 보인다. 다만 이번에 제목 왼쪽에 되돌아가기 버튼(◀)을 새로 붙였다.
- ◀ 버튼을 누르면: 화면만 캘린더 화면으로 되돌리고, 선택했던 날짜나 보고 있던 월은 그대로 둔다. 즉 8월을 보다가 15일을 골라 편집하고 돌아오면, 다시 8월이 보인다(7월로 리셋되지 않는다).
- 예전에 쓰던 사각형 캘린더 부품(`src/components/Calendar/Calendar.jsx`)은 이제 아무 데서도 안 쓰이게 되어 폴더째 지웠다.

## 데이터가 어떻게 흐르나?

```
diary 창 열림
    │
    ▼
screen = 'main' (기본값)
    │
    ▼
[캘린더 화면]  XP캘린더(좌) + 할일/날씨/시계 위젯(우)
    │
    │  날짜 클릭 (예: "2026-08-15")
    ▼
selectedDate = "2026-08-15"
screen = 'edit'
    │
    ▼
[편집 화면]  ◀ 2026-08-15
             [다이어리] [영화리뷰]
             (그림판)
    │
    │  ◀ 버튼 클릭
    ▼
screen = 'main'   (selectedDate, currentYear, currentMonth는 안 건드림)
    │
    ▼
[캘린더 화면]  아까 보고 있던 월(8월) 그대로 다시 표시
```

실제로 8월로 한 칸 넘긴 뒤 15일을 클릭했더니 "2026-08-15" 편집 화면으로 넘어갔고, 하트 스티커를 하나 찍어봤더니 그림판에 정상적으로 붙었다. ◀를 눌러 돌아왔더니 캘린더는 7월이 아니라 8월을 그대로 보여줬다.

## 쉽게 말하면

지금까지 만든 위젯들이 부품 상자 안에 따로따로 들어있었다면, 이번 단계는 그 부품들을 실제 책상 위에 올려놓고 조립하는 작업이다. 그리고 "캘린더 화면"과 "편집 화면"은 마치 책의 목차 페이지와 본문 페이지 같은 관계다 — 목차(캘린더)에서 원하는 장을 짚으면 그 페이지(편집 화면)로 넘어가고, 다시 목차로 돌아와도 아까 펼쳐뒀던 페이지 근처가 그대로 열려 있는 것과 같다.

## 만든 파일

- `src/components/DiaryApp/DiaryApp.jsx` (수정) — 캘린더 화면/편집 화면 전환 로직과 위젯 조립을 담당.
- `src/components/Calendar/Calendar.jsx` (삭제) — XP 캘린더로 완전히 대체되어 더 이상 쓰이지 않던 예전 사각 캘린더.

## 제대로 됐는지 어떻게 확인했나?

1. **린트(코드 스타일 자동 검사) 통과**: `npm run lint` 에러 없이 통과.
2. **브라우저에서 직접 확인**(개발 서버 + 자동화 브라우저 Playwright):
   - diary 창을 열면 왼쪽 XP 캘린더 + 오른쪽 위젯 3개(할일/날씨/시계)가 보이는 캘린더 화면이 뜬다 — 확인됨.
   - ▶ 버튼으로 8월로 이동한 뒤 15일을 클릭하니 "2026-08-15" 편집 화면(◀ 버튼 + 탭 + 그림판)으로 전환됐다 — 확인됨.
   - 편집 화면에서 하트 스티커를 추가했더니 그림판에 정상적으로 붙었다(기존 기능 회귀 없음) — 확인됨.
   - ◀ 버튼을 누르니 캘린더 화면으로 돌아왔고, 화면엔 여전히 "2026년 8월"이 표시됐다(7월로 리셋되지 않음) — 확인됨.
   - `src/components/Calendar/Calendar.jsx`를 지운 뒤에도 프로젝트 어디에서도 이 파일을 찾는 곳이 없음을 확인(삭제 전 이 파일을 쓰던 곳은 `DiaryApp.jsx` 한 곳뿐이었다).

## 아직 안 된 것 (솔직하게)

- 날씨 위젯은 이번 단계에서 새로 검증하지 않았다(이미 3단계에서 확인됐고, 이번엔 실제 화면에 붙어서 잘 뜨는지만 확인했다). 위젯 자체의 정확도나 API 응답 지연 등은 3단계 리포트를 참고하면 된다.
- 화면 크기가 작을 때 캘린더+위젯 3개가 한 줄에 다 들어가는지(반응형 레이아웃)는 확인하지 않았다 — 계획 문서에 반응형 요구사항이 없어서 다루지 않았다.
- 편집 화면에서 ◀ 대신 브라우저 뒤로가기 버튼을 눌렀을 때의 동작은 확인하지 않았다 — 이 프로젝트는 라우터를 안 쓰므로 애초에 브라우저 주소가 안 바뀌고, 계획 문서에도 언급이 없다.
