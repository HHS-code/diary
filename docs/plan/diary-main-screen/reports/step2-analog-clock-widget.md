# Step 2: analog-clock-widget

## 이 단계를 왜 했나?

메인 화면 우측에는 위젯 3개(할 일, 날씨, 시계)가 세로로 나란히 놓인다. 그중 시계 위젯을 이번 단계에서 만든다. Taskbar(화면 아래 파란 막대)에는 이미 숫자로 시간을 보여주는 디지털 시계(`Clock.jsx`)가 떠 있는데, 이번에 만드는 건 그것과는 완전히 별개인 "바늘 있는 둥근 시계"다. 둘은 서로 참조하지 않고, 각자 독립적으로 1초마다 시간을 갱신한다.

## 무엇을 만들었나?

`src/components/AnalogClockWidget/AnalogClockWidget.jsx`라는 새 화면 부품을 만들었다.

- 흰 원판에 12시간 눈금이 찍힌 둥근 시계 모양.
- 시침(짧고 굵은 바늘), 분침(중간 길이 바늘), 초침(가늘고 빨간 바늘) 3개가 각각 실제 시/분/초에 맞는 각도로 회전한다.
- 시침은 "정각"만 보고 딱딱 튀지 않는다. 예를 들어 11시 30분이면 시침이 11과 12 사이 정확히 중간에 위치하도록, 분 값까지 반영해서 부드럽게 계산했다.
- 1초마다 시간을 다시 읽어와 바늘을 다시 그린다(Taskbar 디지털 시계와 같은 방식 — 1초 타이머).
- 이 위젯은 바깥에서 아무 값도 받지 않는다(props 없음) — 그냥 놓으면 알아서 지금 시각을 보여준다.
- 전체를 XP 스타일(진한 파란 테두리 + 베이지 배경 + 그림자)의 카드로 감싸서, 다른 위젯들과 톤이 맞도록 했다.

## 데이터가 어떻게 흐르나?

```
브라우저의 실제 시계 (예: 11시 18분 42초)
        │
        │  1초마다 한 번씩 "지금 몇 시야?" 물어봄 (setInterval 1000ms)
        ▼
AnalogClockWidget 내부 상태(state)
        │
        │  11시 18분 42초 → 각도로 변환
        │    시침 각도 = 11×30 + 18×0.5 = 339도
        │    분침 각도 = 18×6 + 42×0.1 = 112.2도
        │    초침 각도 = 42×6 = 252도
        ▼
SVG로 그 각도만큼 바늘 회전해서 그림
```

숫자로 예를 들면: 시침은 "1시간에 30도씩" 돈다(시계 원판 한 바퀴 360도를 12시간으로 나누면 30도). 여기에 "1분에 0.5도씩"을 더해서, 11시 정각이 아니라 11시 30분이면 시침이 11과 12 사이 딱 중간(339도가 아니라 345도)에 오도록 만들었다. 초침은 1초에 6도씩 돌아서 60초(1분)에 정확히 한 바퀴(360도) 돈다.

## 쉽게 말하면

이 위젯은 "손목시계 하나를 화면 안에 넣어둔 것"과 같다. Taskbar의 디지털 시계가 숫자로 "11:18"이라고 알려주는 안내판이라면, 이번에 만든 건 실제로 바늘이 째깍째깍 돌아가는 아날로그 손목시계다. 둘 다 같은 시간을 보여주지만 서로 다른 시계이고, 하나가 고장 나도 다른 하나는 영향받지 않는다.

## 만든 파일

- `src/components/AnalogClockWidget/AnalogClockWidget.jsx` (신규) — 바늘 3개가 있는 SVG 아날로그 시계 위젯.

## 제대로 됐는지 어떻게 확인했나?

1. **린트(코드 스타일 자동 검사) 통과**: `npm run lint`를 돌려 에러 없이 통과했다.
2. **브라우저에서 직접 확인**: 개발 서버를 띄우고 diary 창 안에 시계 위젯을 임시로 띄워 눈으로 확인했다(확인 후 임시 코드는 원상복구해 이번 커밋에는 남기지 않았다).
   - 시침·분침·초침이 있는 둥근 시계가 보였다 — 확인됨.
   - 첫 화면 캡처 시각(Taskbar 기준 11:18)과 5초 뒤 다시 캡처한 시각(11:19)을 비교했더니, 초침이 실제로 움직여 있었고 분이 바뀌는 순간 분침도 같이 움직인 것을 확인했다 — 확인됨.

## 전체 그림에서 지금 어디?

```
[0] storage-todos (완료)
[1] xp-calendar (완료)
[2] analog-clock-widget (완료)   <- 지금 여기
 └─ 3  weather-widget
 └─ 4  todo-widget
 └─ 5  main-screen-wiring         <- 여기서 이 시계를 실제 diary 화면에 끼운다
```

시계 위젯 하나를 독립적으로 완성했다. 다음 단계는 날씨 위젯(step 3), 그다음 할 일 위젯(step 4)을 같은 방식으로 각자 따로 만들고, 마지막 5단계에서 캘린더 + 위젯 3개를 한 화면에 모아 조립한다. 지금 diary 창을 열어도 아직 이 시계는 보이지 않는 게 정상이다.

## 아직 안 된 것 (솔직하게)

- 실제 diary 화면에는 아직 연결되지 않았다. 지금 diary 창을 열면 시계 위젯이 보이지 않는다. 연결은 5단계에서 한다.
- 알람, 타이머, 시간대 변경 같은 기능은 만들지 않았다 — 계획 문서에 "현재 시각을 보여주는 아날로그 시계"로만 범위가 정해져 있어서 그 범위만 지켰다.
