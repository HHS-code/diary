# 빌드 리포트 — diary (diary-main-screen)

각 step이 무엇을 했는지 위에서 아래로 읽으면 프로젝트가 자라는 과정이 보인다.

- [완료] **Step 0: storage-todos** — src/storage/diaryStorage.js에 getDateTodos(data, dateKey) -> Todo[] (없으면 []), setDateTodos(data, dateKey, todos) -> DiaryData (data 비변경, diary/movie 필드 보존, spread 병합) 추가. Todo = { id, text, done } — id 생성은 호출자(TodoWidget, step4) 책임. loadAllDiaryData/saveAllDiaryData는 미수정, 그대로 재사용. 프로젝트에 테스트 러너 없음 — Node 스크립트로 수동 검증(AC 그대로). 다음 step이 import할 산출 파일: src/storage/diaryStorage.js. ([리포트](step0-storage-todos.md))
- [완료] **Step 1: xp-calendar** — src/components/XPCalendar/XPCalendar.jsx 신규 생성: export function XPCalendar({ selectedDate, onSelectDate, currentYear, currentMonth, onChangeMonth }). 연/월 state는 이 컴포넌트가 갖지 않고 부모에게서 props로만 받아 표시 — ◀▶ 클릭 시 계산만 해서 onChangeMonth(year, month) 호출. 오늘=노란 배경(#ffff00), 선택된 날짜(오늘 아님)=파란 2px 테두리, buildCalendarDays/formatDateKey는 Calendar.jsx와 동일 로직으로 파일 내부에 재구현(공유 안 함). Calendar.jsx는 삭제하지 않음(step 5 범위). DiaryApp.jsx는 아직 연결 안 함(step 5 범위). 다음 step이 import할 산출 파일: src/components/XPCalendar/XPCalendar.jsx. ([리포트](step1-xp-calendar.md))
- [완료] **Step 2: analog-clock-widget** — src/components/AnalogClockWidget/AnalogClockWidget.jsx 신규 생성: export function AnalogClockWidget() — props 없음. Clock.jsx의 useEffect+setInterval(1000) 패턴을 독립적으로 재구현(import/참조 없음). 시/분/초 각도는 calculateHandPoint(angleDegrees, length)로 SVG 좌표(viewBox 0 0 100 100, 중심 50,50) 계산, 시침은 분 값 반영(hourAngle = hours*30 + minutes*0.5)해 부드럽게 움직임. XP 톤(#0a246a 테두리, #ece9d8 배경) 카드로 감쌈. DiaryApp.jsx는 미수정(연결은 step 5). 다음 step이 import할 산출 파일: src/components/AnalogClockWidget/AnalogClockWidget.jsx. ([리포트](step2-analog-clock-widget.md))
- [완료] **Step 3: weather-widget** — src/components/WeatherWidget/WeatherWidget.jsx 신규 생성: export function WeatherWidget() — props 없음. requestCurrentCoordinates()가 navigator.geolocation.getCurrentPosition을 Promise로 감싸고, 미지원/실패 시 서울(37.5665, 126.978)로 resolve(에러 throw 없음). fetchCurrentWeather(lat, lon)가 Open-Meteo current_weather를 호출, describeWeatherCode(weathercode)가 0/1~3/45,48/51~67/71~86/그외로 대략 그룹핑해 이모지+한글 라벨 반환. status state는 'loading'|'ready'|'error' 세 값, renderWeatherContent(status, weather)가 순수 함수로 분기 렌더(삼항 미사용). Playwright로 위치 허용(부산 좌표, 31°C 표시)·거부(getCurrentPosition을 error 콜백으로 오버라이드, 서울 fallback, 30°C 표시) 두 경로 모두 api.open-meteo.com 200 응답과 함께 수 초 내 렌더 확인. DiaryApp.jsx는 임시 연결 후 원복(git diff 없음), 실제 연결은 step 5. 다음 step이 import할 산출 파일: src/components/WeatherWidget/WeatherWidget.jsx. ([리포트](step3-weather-widget.md))
- [완료] **Step 4: todo-widget** — src/components/TodoWidget/TodoWidget.jsx 신규 생성: export function TodoWidget({ selectedDate }). loadAllDiaryData()→getDateTodos로 목록을 읽고, 추가(crypto.randomUUID())/체크(toggleTodoDone)/삭제(removeTodo) 시마다 loadAllDiaryData()→setDateTodos→saveAllDiaryData로 즉시 저장(낙관적 setState 병행). selectedDate가 바뀌면 useEffect가 그 날짜의 todos로 목록 교체. localStorage 직접 접근 없이 diaryStorage.js 함수만 사용. done 항목은 취소선(#888). WeatherWidget/AnalogClockWidget과 동일한 XP 카드 스타일(160px, #0a246a 테두리, #ece9d8 배경). Playwright로 추가/체크/삭제 각각 새로고침 후에도 유지/삭제됨을 확인, 다른 selectedDate에서 독립된 빈 목록도 확인. DiaryApp.jsx는 검증 후 원복(git diff 없음), 실제 연결은 step 5. 다음 step이 import할 산출 파일: src/components/TodoWidget/TodoWidget.jsx. ([리포트](step4-todo-widget.md))
