# PRD — 다이어리 메인 화면 (XP 캘린더 + 위젯)

## 배경 / 왜

지금 "diary" 창을 열면 사각 그리드 캘린더(`Calendar.jsx`) + 탭 + 캔버스 에디터가 한 화면에 바로 나온다. 이를 Windows XP 데스크톱 캘린더 가젯(`reference/화면 캡처 2026-07-11 103520.png`) 스타일의 메인 화면으로 교체한다. 좌측엔 XP 스타일 월간 캘린더, 우측엔 레트로 스타일 위젯 3종(할 일 체크리스트, 날씨, 아날로그 시계)을 두고, 날짜를 클릭해야 그 날짜의 다이어리 편집 화면(탭+캔버스)으로 들어가는 흐름으로 바꾼다.

## 목표

- diary 창을 열면 캘린더+위젯으로 구성된 메인 화면이 먼저 보인다.
- 메인 화면의 캘린더에서 날짜를 클릭하면 같은 창 안에서 그 날짜의 다이어리 편집 화면(기존 탭+캔버스)으로 전환된다.
- 편집 화면에서 뒤로가기(◀)를 누르면 메인 화면으로 돌아간다.
- 우측 위젯(할 일 체크리스트, 날씨, 아날로그 시계)은 모두 레트로(XP) 스타일로 보인다.

## 대상 사용자

기존과 동일(혼자 쓰는 개인 다이어리 사용자). 다이어리 편집 기능(캔버스, 스티커, 텍스트, PNG 내보내기 등) 자체는 변경 없음 — 진입 방식과 메인 화면 구성만 바뀐다.

## In-scope

### XP 스타일 캘린더 (좌측)

- 레퍼런스 이미지 스타일: 상단 파란 헤더에 "YYYY년 M월" 표시 + ◀▶ 버튼으로 월 이동, 요일 행, 카드형 날짜 셀 그리드.
- 오늘 날짜 = 노란색 배경 하이라이트.
- 선택된 날짜(오늘이 아닌 경우) = 파란색 테두리 박스로 표시.
- 날짜 셀 클릭 → 해당 날짜를 선택하고 다이어리 편집 화면으로 전환.
- 기존 `Calendar.jsx`(사각 그리드 스타일)는 삭제하고 이 컴포넌트로 완전히 대체한다.

### 우측 위젯 영역

세 위젯을 레트로(XP) 스타일 카드로 세로로 배치한다.

1. **할 일 체크리스트**
   - 선택된 날짜 기준으로 그 날의 할 일 목록을 보여준다 (날짜별 독립 데이터).
   - 항목 추가, 완료 체크(취소선/체크 표시), 삭제 가능.
   - localStorage에 날짜별로 저장되어 새로고침해도 유지된다.

2. **날씨 위젯**
   - 무료 날씨 API(예: Open-Meteo, API 키 불필요)로 현재 위치의 실시간 날씨(기온, 날씨 상태)를 보여준다.
   - 위치는 브라우저 Geolocation API로 요청 — 사용자가 허용하면 해당 좌표, 거부/실패 시 서울 좌표로 fallback.

3. **아날로그 시계**
   - 시침/분침/초침이 있는 SVG 기반 아날로그 시계. 1초 단위로 실시간 갱신.
   - Taskbar에 이미 있는 디지털 시계(`Clock.jsx`)와는 별개의 새 컴포넌트.

### 화면 전환

- diary 창 안에서 "메인 화면"(캘린더+위젯)과 "편집 화면"(탭+캔버스, 기존 구조)을 상태로 전환한다.
- 메인 화면 → 날짜 클릭 → 편집 화면 (선택된 날짜로 진입).
- 편집 화면 → 제목(`{selectedDate}`) 왼쪽 ◀ 버튼 클릭 → 메인 화면으로 복귀.
- 새 npm 패키지(라우터 등) 추가 없음 — 기존 프로젝트 관례대로 컴포넌트 내부 state로만 전환한다.

### 데이터 저장

- 할 일 데이터는 `storage/diaryStorage.js`의 기존 `DiaryData` 구조(날짜를 키로 하는 localStorage JSON, 같은 `STORAGE_KEY`)를 확장해 날짜별 `todos` 배열을 추가하는 방식으로 저장한다.

## Out-of-scope (명시)

- 다이어리 편집 기능 자체(캔버스, 스티커, 텍스트, 이미지, PNG/JSON 내보내기, 영화 탭) — 로직 변경 없음.
- Desktop 화면, Window 창 껍데기, Taskbar, 전원 버튼 — `desktop-ui` 범위 그대로, 변경 없음.
- 날씨 위젯의 도시 검색/변경 UI — 자동 감지 또는 서울 고정만, 수동 지역 설정 기능 없음.
- 할 일 체크리스트의 반복 일정, 알림, 우선순위 등 고급 기능 — 추가/체크/삭제만.
- 위젯 개수/종류 확장을 위한 범용 위젯 시스템(플러그인 구조 등) — 지금은 정해진 3개 위젯만 하드코딩.
- 캘린더 연/월 드롭다운 점프 — ◀▶ 버튼으로만 이동.

## 성공 기준

- diary 창을 열면 좌측 XP 스타일 캘린더 + 우측 위젯 3개(할일/날씨/시계)가 보인다.
- 캘린더에서 날짜를 클릭하면 그 날짜의 다이어리 편집 화면(탭+캔버스)으로 전환되고, 기존과 동일하게 편집·자동저장이 동작한다.
- 편집 화면에서 ◀ 버튼을 누르면 메인 화면으로 돌아가고, 캘린더 상태(현재 보던 월)가 유지된다.
- 할 일 항목을 추가/체크/삭제하면 그 날짜에 저장되고 새로고침해도 유지된다. 다른 날짜를 선택하면 그 날짜의 할 일로 바뀐다.
- 날씨 위젯이 실제(또는 fallback) 위치의 날씨 정보를 보여준다.
- 아날로그 시계가 실제 시간과 일치해 초 단위로 움직인다.

## 진행 방식

1. 문서(PRD/architecture) 합의 → 실행 승인
2. storage 확장(todos) → XP 캘린더 컴포넌트 → 위젯 3종 → 메인/편집 화면 전환 → DiaryApp 조합 → 완료 시 사용자에게 결과 설명
