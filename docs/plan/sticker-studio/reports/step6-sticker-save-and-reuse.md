# Step 6: sticker-save-and-reuse

## 이 단계를 왜 했나?

지난 다섯 단계에서 그리기, 이미지 업로드, 크롭, 배경 제거(누끼), 흰 테두리까지 스티커를 "만드는" 모든 도구가 갖춰졌다. 하지만 그동안 만든 결과물은 화면(캔버스) 위에만 존재했다 — 창을 닫으면 사라지고, 다이어리 캔버스에서 골라 쓸 방법도 없었다. 이번 단계는 완성한 스티커를 실제로 저장하고, 다이어리에서 꺼내 쓰고, 나중에 다시 불러와 고칠 수 있게 만드는 "마지막 연결 고리"다.

## 무엇을 만들었나?

세 가지를 만들었다.

1. **"완성" 버튼** — 스티커 스튜디오 캔버스에서 그린/편집한 결과를 투명 배경 PNG 파일로 저장한다.
2. **"내 스티커" 패널** — 저장했던 스티커들을 썸네일(작은 미리보기 그림) 목록으로 보여주고, 클릭하면 캔버스에 불러와 다시 편집할 수 있다.
3. **다이어리 쪽 "스티커" 목록** — 다이어리 캔버스의 에셋(그림·폰트 등 첨부 자료) 패널에 "스티커" 항목이 추가되어, 저장한 스티커를 다이어리에도 바로 붙일 수 있다.

**용어 하나만 짚고 가면**: "Blob"은 이미지·동영상 같은 이진 데이터 덩어리를 자바스크립트에서 다루는 형식이다. 여기서는 "완성된 스티커 그림 파일 자체"라고 이해하면 된다.

## 데이터가 어떻게 흐르나?

```
[사용자가 "완성" 버튼 클릭]
        |
        v
[테두리를 확정했으면 그 그림을, 안 했으면 지금 캔버스 그대로를 픽셀 이미지로 뜸]
        |
        v
[그 픽셀 이미지를 PNG 파일(Blob)로 변환]
        |
        v
[registerSticker(파일, "sticker-1752838800000.png") 호출]
        |
        v
[저장소(assetStorage)에 type: 'sticker'로 기록 — 항상 새 항목]
        |
        v
[버튼 글자가 1.5초간 "저장됨"으로 바뀜]
```

```
["내 스티커" 패널에서 썸네일 클릭]
        |
        v
[저장소에서 그 스티커의 원본 파일을 불러옴]
        |
        v
[지금 캔버스에 있던 그림을 전부 지우고, 불러온 스티커 하나로 채움]
        |
        v
[사용자가 이어서 편집 → "완성"을 다시 누르면 또 새 항목으로 저장]
        (원본은 그대로, 편집본만 새로 추가 — 절대 원본을 덮어쓰지 않음)
```

```
[다이어리 캔버스의 에셋 패널]
  이미지 (3)  ← 기존
  폰트 (1)    ← 기존
  스티커 (2)  ← 이번 단계에서 추가
        |
        v (스티커 썸네일 클릭)
[다이어리 캔버스에 그 스티커를 그림으로 배치]
  (기존 이미지를 붙이는 방식과 완전히 동일한 경로를 그대로 탄다)
```

**숫자로 보면**: 테스트에서 "완성" 버튼을 누르면 `registerSticker`가 정확히 1번, PNG 타입(`image/png`)의 파일과 함께, `sticker-` 뒤에 숫자(현재 시각)가 붙고 `.png`로 끝나는 파일명(예: `sticker-1752838800000.png`)으로 호출되는 것을 확인했다. 저장 목록 화면에서는 3열 그리드로 스티커가 나열되고, 스티커가 1개 저장돼 있으면 "내 스티커 (1)", "스티커 (1)"처럼 개수가 정확히 표시된다.

## 쉽게 말하면

지금까지 작업대(캔버스) 위에서만 만들던 스티커를, 이번 단계에서 처음으로 "사진첩에 저장"하는 것과 같다. 저장 버튼을 누르면 새 사진 한 장이 사진첩에 추가되고(항상 새 사진, 기존 사진을 덮어쓰지 않음), 사진첩 썸네일을 눌러 다시 작업대에 올려 손볼 수 있으며, 다이어리라는 다른 앱에서도 같은 사진첩을 열어 그 사진을 골라 쓸 수 있게 된 것이다.

## 전체 그림에서 지금 어디?

```
[0] 캔버스 크기 파라미터화 ✅
[1] 스티커 저장 타입(assetStorage) ✅
[2] 데스크톱 진입점 + 빈 캔버스 ✅
[3] 이미지 업로드 + 크롭 ✅
[4] 올가미로 배경 제거(누끼) ✅
[5] 흰 테두리 만들기 ✅
[6] 완성 스티커 저장 + 다이어리에서 재사용 ✅  ← 지금 여기, 마지막 단계
```

PRD에 정의된 "In-scope" 항목(데스크톱 진입점, 캔버스, 그리기, 업로드/크롭, 배경 제거, 흰 테두리, 저장 및 재사용)이 모두 구현되어, 스티커 스튜디오 phase의 계획된 개발 단계가 여기서 끝난다.

## 아직 안 된 것 (솔직하게)

- **스티커 삭제·이름 변경 UI가 없다.** "내 스티커" 목록은 계속 늘어나기만 하고, 잘못 저장한 스티커를 지우거나 이름을 바꿀 방법이 없다 — 이번 phase의 계획(PRD Out-of-scope)에서 처음부터 제외한 부분이다.
- **파일명을 사용자가 직접 정할 수 없다.** 저장할 때마다 `sticker-현재시각.png` 형태의 자동 파일명이 붙는다. 다이어리 에셋 목록에서 스티커를 구분할 때 이 자동 생성된 이름만 보이므로, 스티커가 많아지면 어떤 게 어떤 그림인지 이름만으로는 알아보기 어려울 수 있다.
- **테두리가 캔버스 가장자리에서 잘리는 문제(5단계에서 확인된 것)가 그대로 남아 있다.** 이번 단계는 저장 로직만 다뤘고, 그림을 캔버스 가장자리에 딱 붙여 그렸을 때 테두리가 잘리는 현상 자체는 고치지 않았다.
- **누끼 이미지가 배치될 때 위치가 살짝 치우치는 문제(4단계에서 확인된 것)도 그대로다.** 실제 사용에 큰 지장은 없지만, 정중앙이 아니라 좌상단 쪽으로 치우쳐 보일 수 있다.
