# Step 4: 올가미로 배경 제거(누끼따기)

## 이 단계를 왜 했나?

이전 단계(step 3)까지는 이미지 전체를 올리거나, 사각형 영역으로만 잘라낼 수 있었다. 그런데 진짜 "나만의 스티커"를 만들려면 사진 속 인물이나 물건만 자유로운 모양으로 오려내고, 배경은 투명하게 지워야 한다(PRD가 "배경 제거(누끼따기)"라고 부르는 기능). 이번 단계는 그 기능을 만들었다 — 마우스로 자유곡선을 그리면 그 안쪽만 남고 바깥은 투명해진다.

## 무엇을 만들었나?

1. **올가미 도구** — 그리기 도구 목록에 "올가미" 버튼이 추가됐다. 다른 도구(연필, 브러시 등)와 구분되도록 얇은 파란 점선으로 표시된다. 이 점선은 실제로 캔버스에 남는 그림이 아니라, "여기부터 여기까지가 선택 영역이다"라는 표시선일 뿐이다 — 선을 다 그리는 순간 화면에서 사라진다.
2. **미리보기** — 올가미로 곡선을 다 그리면, 그 곡선 안쪽에 있던 이미지만 즉시 화면에 남고 바깥쪽은 투명하게 사라져 보인다. 아직 "확정"은 아니라서, 마음에 안 들면 올가미를 다시 그려서 다른 모양으로 덮어쓸 수 있다(몇 번이든 다시 시도 가능).
3. **누끼 적용 버튼** — 미리보기가 마음에 들면 "누끼 적용" 버튼을 누른다. 그러면 지금 보이는 모습 그대로 실제 그림 픽셀로 확정되고, 이후에는 다시 올가미를 그려도 이미 확정된 부분은 영향받지 않는다.

곡선을 완전히 한 바퀴 닫지 않고(시작점과 끝점이 정확히 안 만나고) 끝내도, 자동으로 시작점과 끝점을 직선으로 이어서 닫힌 모양처럼 처리한다 — 별도로 "곡선을 다시 그리세요" 같은 경고는 띄우지 않는다(ADR-5에서 이미 정한 방식).

## 데이터가 어떻게 흐르나?

```
사용자가 "올가미" 도구 선택
  → 캔버스에 마우스로 곡선을 그림 (파란 점선으로 표시됨)
  → 마우스를 떼는 순간, 그 점선은 "그림"이 아니라 "영역 표시"이므로
    캔버스에서 곧바로 지워짐 (사라진 것처럼 보이지만, 그 모양 정보는 다음 단계로 넘어감)
  → 이미지에 "이 모양 바깥은 가려라"는 마스크로 지정 (미리보기)
  → 화면에는 곡선 안쪽 이미지만 보이고, 바깥쪽은 투명해짐

"누끼 적용" 클릭
  → 지금 화면에 보이는 모습(마스크 적용된 상태)을 실제 픽셀로 다시 그림
  → 원래 이미지는 지우고, 새로 그려진 픽셀 그림으로 교체
  → 이제 마스크 정보는 필요 없어짐 (이미 픽셀에 반영됐으므로)
```

숫자로 예시를 들면: 100×100 크기의 빨간 사각형 이미지가 캔버스에 있을 때, 그 왼쪽 절반만 감싸는 올가미(가로 56픽셀, 세로 100픽셀 정도의 영역)를 그리면, 미리보기 화면에는 왼쪽 절반(56×100)만 빨갛게 보이고 오른쪽 절반은 투명해진다. "누끼 적용"을 누르면 캔버스에 남는 오브젝트는 여전히 1개(교체됐을 뿐 개수는 그대로)이고, 이 오브젝트는 왼쪽 절반만 빨간 픽셀을 가진 순수한 이미지가 된다.

## 쉽게 말하면

사진 위에 트레이싱지를 놓고 오리고 싶은 모양을 손으로 따라 그리는 것과 같다. 손으로 따라 그리는 동안에는 "이 선 안쪽만 남길 거야"라는 예고일 뿐이고(미리보기), 가위로 실제로 오려서 트레이싱지를 걷어내는 순간(누끼 적용) 비로소 그 모양대로 사진이 잘려나간다. 마음에 안 들면 가위질하기 전까지는 몇 번이고 다시 선을 그어볼 수 있다.

## 제대로 됐는지 어떻게 확인했나?

- 코드 검사(lint), 빌드(build), 자동 테스트(test) 모두 통과(테스트 30개 파일, 176개 테스트 전부 성공).
- 새로 만든 자동 테스트로 확인한 것:
  - 올가미로 그린 곡선을 이미지에 지정하면 미리보기용 마스크가 설정된다.
  - 다시 그리면 이전 마스크를 새 것으로 덮어쓴다.
  - 대상 이미지가 없거나 곡선이 점 2개 미만으로 너무 짧으면 아무 일도 일어나지 않는다.
  - "누끼 적용" 후에는 캔버스 오브젝트 개수가 그대로 1개이되(교체됨) 새 오브젝트는 더 이상 마스크를 갖지 않는(이미 픽셀로 확정된) 순수 이미지다.
  - 확정된 새 이미지는 원래 이미지가 있던 자리에 그대로 남는다.
  - 확정 시 저장소 식별자(assetId) 같은 기존 정보가 새 이미지로 그대로 이어진다.
  - 올가미로 그린 선은 다른 그리기 도구의 획과 달리 "그림"으로 남지 않고 즉시 지워진다.
- 실제 브라우저에서도 직접 확인(자동화된 화면 조작 도구 사용): 스티커 스튜디오에서 빨간 사각형 이미지를 올리고, 그 왼쪽 절반만 감싸는 올가미를 그렸더니 오른쪽 절반이 즉시 투명해지는 것을 화면으로 확인했고, "누끼 적용"을 누른 뒤에도 왼쪽 절반만 남은 이미지 1개가 정확히 같은 자리에 남는 것까지 확인했다.

## 전체 그림에서 지금 어디?

```
[완료] 0. 캔버스 크기 조절 가능하게
[완료] 1. 스티커 저장 형식 준비
[완료] 2. 스티커 스튜디오 창 열기 + 그리기 도구
[완료] 3. 이미지 업로드 + 사각형 크롭
[완료] 4. 올가미로 배경 제거(누끼따기)   ← 지금 여기
[남음] 5. 흰색 테두리 추가
[남음] 6. 저장하고 다이어리에서 꺼내 쓰기
```

다음 단계는 "흰색 테두리 추가"다. 지금까지 만든 결과물은 "배경이 투명한 이미지"인데, 다음 단계는 그 실루엣을 따라 흰 테두리를 둘러서 스티커다운 느낌을 낸다. 이번 단계에서 확정한 결과물(더 이상 마스크가 아니라 순수 픽셀 이미지)이 바로 다음 단계가 입력으로 받을 재료라서, 순서상 자연스럽게 이어진다.

## 아직 안 된 것 (솔직하게)

- 결과는 아직 저장되지 않는다 — 브라우저를 새로고침하면 사라진다. 저장은 6단계("완성" 버튼)에서 만든다.
- 흰색 테두리는 이번 단계 범위가 아니라서 아직 없다(다음 단계 몫).
- 이번 조사 과정에서 사용 중인 그림 라이브러리(Fabric.js) 버전의 기본 동작이 예전 버전과 달라진 것을 하나 확인했다: 이미지를 캔버스 "정중앙"에 놓으려는 기존 계산식(2·3단계에서 만든 이미지 업로드·크롭 기능)이 실제로는 이미지를 살짝 왼쪽 위로 치우치게 배치하고 있었다(라이브러리의 좌표 기준점 기본값이 바뀐 영향으로 보인다). 이번 단계에서 새로 만든 코드(누끼 확정 시 이미지 위치 유지)는 이 문제와 무관하게 정확히 동작하도록 만들었지만, 기존 업로드·크롭 기능의 "정중앙 배치"가 실제로는 정중앙이 아닐 수 있다는 점은 이번 단계 범위 밖이라 고치지 않았다 — 참고로만 남겨둔다.
