# Step 5: sticker-outline

## 이 단계를 왜 했나?

스티커 스튜디오에서 그림을 그리거나 사진을 오려내면(누끼 따기) 스티커 모양이 만들어진다. 하지만 카카오톡·라인 등에서 흔히 보는 스티커들은 실루엣(그림의 바깥 윤곽) 바로 바깥에 흰 테두리가 둘러져 있다. 이 테두리가 있어야 어떤 배경(다이어리의 색깔 있는 종이, 어두운 배경 등) 위에 스티커를 붙여도 경계가 또렷하게 보인다. 이번 단계는 이 흰 테두리를 만드는 기능을 추가했다.

## 무엇을 만들었나?

두 가지를 만들었다.

1. **테두리를 실제로 그리는 계산 로직**(`stickerOutline.js`) — 그림(스티커) 캔버스와 "몇 픽셀 두께로 할지" 숫자를 받으면, 흰 테두리가 둘러진 새 그림을 돌려주는 함수 하나.
2. **스티커 스튜디오 화면의 UI** — "테두리 추가" 버튼을 누르면 두께 조절 슬라이더와 미리보기 화면이 나타나고, 마음에 들면 "적용" 버튼으로 확정한다.

**용어 하나만 짚고 가면**: "래스터라이즈(rasterize)"는 도형이나 선처럼 수학적으로 표현된 그림을 실제 화면 픽셀(점 하나하나의 색깔 정보)로 "구워내는" 작업을 말한다. 이 단계에서는 스티커 스튜디오 편집 캔버스를 통째로 이 방식으로 픽셀 이미지로 떠서 테두리 계산에 넘긴다.

## 데이터가 어떻게 흐르나?

```
[사용자가 슬라이더를 움직임: 두께 5px]
        |
        v  (100밀리초 동안 더 움직이지 않으면 실행 — "디바운스")
[편집 중인 캔버스를 픽셀 이미지로 떠냄]
        |
        v
[createOutlinedSticker(이미지, 5)]
        |
        +-- 1) 이미지의 불투명한 부분만 골라 하얗게 칠한 "실루엣" 이미지를 만든다
        |
        +-- 2) 그 실루엣을 원 둘레를 따라 여러 방향(예: 10방향)으로
        |      5픽셀씩 옮겨가며 겹쳐 그린다
        |        ↳ 방향이 적으면 테두리가 다각형처럼 각져 보이므로,
        |          두께가 두꺼워질수록 방향 수를 8~32개 사이로 늘린다
        |
        +-- 3) 그 위에 원본 그림을 다시 그린다 (테두리가 그림을 가리지 않도록)
        |
        v
[테두리 둘린 새 이미지 완성]
        |
        v
[사이드바 미리보기 칸에 표시]
        |
        v  (사용자가 "적용" 클릭)
[확정 — 다음 단계(저장)가 이 결과를 쓸 준비 완료]
```

**숫자로 보면**: 20x20 픽셀짜리 테스트 이미지 중앙에 8x8 크기의 파란 사각형만 그려 넣고 확인했다. 테두리 두께 0이면 사각형 바로 바깥(예: 왼쪽으로 1픽셀 떨어진 지점)은 그대로 투명했다. 두께 4를 주자 그 지점이 불투명(흰색)으로 바뀌었다. 두께를 2에서 6으로 올리자, 사각형에서 6픽셀 떨어진 지점은 두께 6일 때만 불투명해지고 두께 2일 때는 여전히 투명했다 — 두께 값이 실제로 "몇 픽셀 밖까지 번지는지"를 그대로 결정한다는 뜻이다.

실제 화면에서도 확인했다: 브러시로 굵기 8짜리 사각형 테두리 선을 그린 뒤, 테두리 두께를 0으로 두면 원래 그린 선(약 8픽셀 폭 두 줄)만 보였고, 두께를 5로 올리자 그 두 줄 사이 간격이 통째로 메워져 하나의 넓은 흰 띠로 이어지는 것을 픽셀 값으로 직접 확인했다.

## 쉽게 말하면

스티커를 도장처럼 생각하면 쉽다. 원본 그림을 하얗게 칠한 "도장"을 만든 다음, 시계 방향으로 뺑 돌아가며 여러 번 살짝씩 옮겨 찍는다. 그러면 도장이 찍힌 자국들이 서로 겹치면서 원래 그림 바깥으로 부풀어 오른 하얀 테두리가 생긴다. 마지막에 원래 색깔 그림을 그 위에 다시 얹으면, 흰 테두리 안에 원본 그림이 온전히 살아있는 스티커가 완성된다.

## 만든 파일

- `src/fabric/stickerOutline.js` — 테두리를 계산하는 함수(`createOutlinedSticker`).
- `src/fabric/stickerOutline.test.js` — 이 함수가 두께 0/음수/양수 각각에서 올바르게 동작하는지 확인하는 자동 테스트 6개.
- `src/components/StickerStudio/StickerStudio.jsx` — "테두리 추가" 버튼, 두께 슬라이더, 미리보기, "적용" 버튼을 화면에 연결.
- `src/components/StickerStudio/StickerStudio.test.jsx` — 버튼을 누르면 슬라이더가 나타나고, 다시 누르면 사라지는지 확인하는 테스트 추가.

## 제대로 됐는지 어떻게 확인했나?

- `npm run lint` — 통과 (코드 스타일 문제 없음).
- `npm run build` — 통과 (배포용 빌드가 에러 없이 만들어짐).
- `npm run test` — 전체 183개 자동 테스트 모두 통과 (이번 단계에서 추가한 7개 포함).
- 실제 브라우저(Playwright로 자동 조작)로 직접 확인:
  - 스티커 스튜디오를 열고 브러시로 파란 사각형 테두리를 그림.
  - "테두리 추가" → 슬라이더 → "적용" 순서로 눌러봄.
  - 슬라이더 값을 바꿀 때마다 미리보기 캔버스의 픽셀 값을 직접 읽어, 두께가 커질수록 불투명 영역이 넓어지는 것을 확인.
  - "적용"을 누르면 편집 패널이 정상적으로 닫히는 것을 확인.

## 전체 그림에서 지금 어디?

```
[0] 캔버스 크기 파라미터화 ✅
[1] 스티커 저장 타입(assetStorage) ✅
[2] 데스크톱 진입점 + 빈 캔버스 ✅
[3] 이미지 업로드 + 크롭 ✅
[4] 올가미로 배경 제거(누끼) ✅
[5] 흰 테두리 만들기 ✅  ← 지금 여기
[6] 완성 스티커 저장 + 다이어리에서 재사용
```

지금까지 "스티커를 어떻게 그리고 다듬는지"(그리기, 이미지 편집, 배경 제거, 테두리)가 모두 갖춰졌다. 다음 단계는 이 결과물을 실제로 "저장"해서, 다이어리 캔버스에서 골라 쓸 수 있게 만드는 마지막 단계다. 이번 단계에서 만든 "확정된 테두리 이미지"(또는 테두리 없이 원본)를 그대로 넘겨받아 파일로 저장하기만 하면 되도록 미리 준비해뒀다.

## 아직 안 된 것 (솔직하게)

- **아직 저장되지 않는다.** "적용" 버튼을 눌러도 스티커가 파일로 저장되거나 다이어리에서 보이지는 않는다. 이번 단계는 "테두리 만들기"까지만 하고, 실제 저장은 다음 단계(6번)의 몫이다.
- **테두리가 캔버스 가장자리에서 잘릴 수 있다.** 그림이 캔버스(512x512) 가장자리에 딱 붙어 있으면, 테두리가 캔버스 밖으로 나가는 부분은 잘려서 안 보인다. 이번 단계는 이 경우를 별도로 처리하지 않았다 — 스티커를 그릴 때 가장자리에 여유를 두고 그리는 것을 권장한다.
- **"취소" 버튼이 없다.** "테두리 추가"를 다시 눌러 패널을 닫을 수는 있지만, 이미 "적용"으로 확정한 테두리를 되돌리는 별도 버튼은 없다(다시 "테두리 추가"를 눌러 새로 만들면 이전 확정 값을 덮어쓴다).
