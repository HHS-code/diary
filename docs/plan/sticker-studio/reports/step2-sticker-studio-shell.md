# Step 2: sticker-studio-shell

## 이 단계를 왜 했나?

지난 단계까지는 스티커를 "어디에 저장할지"(창고 쪽)만 준비돼 있었고, 스티커를 실제로 만드는 화면은 전혀 없었다. 이번 단계는 데스크톱에 아이콘을 하나 새로 만들고, 그걸 클릭하면 빈 캔버스가 있는 창이 열리도록 만드는 작업이다. 이 창 안에서는 아직 그림을 그리고 지우고 되돌리는(undo) 것만 되고, 사진 업로드나 배경 지우기 같은 기능은 다음 단계들에서 하나씩 붙는다.

## 무엇을 만들었나?

세 가지를 만들었다.

첫째, 데스크톱 화면에 "sticker studio"라는 이름의 새 아이콘을 추가했다. 클릭하면 다이어리 아이콘을 눌렀을 때와 똑같은 방식으로 새 창이 뜬다(코드 구조를 그대로 복사해 만들었다 — 다이어리 앱과 스티커 스튜디오는 서로 완전히 독립된 별개의 창이다).

둘째, "스티커 스튜디오"라는 새 화면(컴포넌트)을 만들었다. 이 화면을 열면 512×512 크기의 정사각형 투명 캔버스가 나타난다(다이어리 캔버스는 1600×1000의 큰 직사각형이라 서로 다르다). 캔버스 옆에는 다이어리에서 쓰던 것과 똑같은 그리기 도구 패널(연필, 브러시, 에어브러시, 지우개, 색상, 굵기)이 그대로 붙어 있다.

셋째, 그리기·되돌리기(undo/redo)·복사삭제(Ctrl+C, Delete) 기능을 다이어리에서 쓰던 코드를 그대로 가져다 연결했다. 새로 만든 코드가 아니라 기존에 검증된 코드를 재사용한 것이라, 이 부분에서 새로운 버그가 생길 가능성이 적다.

## 데이터가 어떻게 흐르나?

```
데스크톱 화면
  [diary 아이콘]  [sticker studio 아이콘]  <- 이번에 추가
        |                  |
   클릭하면              클릭하면
        |                  |
   다이어리 창 열림    스티커 스튜디오 창 열림 (신규)
                            |
                  빈 투명 캔버스(512x512) 표시
                            |
              그리기 도구 패널(연필/브러시/에어브러시/지우개)
                            |
              Ctrl+Z(되돌리기), Ctrl+C+Delete(복사/삭제) 동작
```

숫자 예시: 다이어리 캔버스는 가로 1600 x 세로 1000 픽셀 크기의 큰 도화지였다. 스티커 스튜디오 캔버스는 512 x 512, 정사각형에 훨씬 작은 크기다 — 다이어리는 편지지 한 장 크기라면, 스티커 스튜디오는 명함 한 장 정도의 작업 공간이라고 생각하면 된다. 그리고 배경색도 다르다. 다이어리는 흰 종이(#ffffff)지만, 스티커 스튜디오는 처음부터 투명하다 — 나중에 다이어리에 스티커를 붙였을 때 캔버스 배경이 네모난 흰 박스로 따라오지 않게 하기 위해서다.

## 쉽게 말하면

다이어리 앱을 지을 때 썼던 "설계 도면"(그리기 도구, 되돌리기 기능 등)을 그대로 복사해서, 이번에는 크기가 다른 작은 건물(스티커 스튜디오)을 하나 더 지은 것과 비슷하다. 문(아이콘)만 새로 하나 달았을 뿐, 안에 들어가는 가구(그리기 도구, undo 버튼)는 원래 있던 것을 그대로 옮겨왔다.

## 만든 파일

- `src/assets/icons.js`, `src/assets/sticker-studio-icon.png` — 데스크톱에 쓸 스티커 스튜디오 아이콘 이미지 추가 (원본: `reference/Windows XP Icons/Paint.png`)
- `src/components/Desktop/Desktop.jsx`, `src/components/Desktop/Desktop.test.jsx` — 데스크톱에 스티커 스튜디오 아이콘 버튼 추가, 클릭 시 콜백이 호출되는지 확인하는 테스트
- `src/App.jsx` — 스티커 스튜디오 창을 여닫는 상태(state)와 창 연결 추가
- `src/components/StickerStudio/StickerStudio.jsx`, `src/components/StickerStudio/StickerStudio.test.jsx` — 신규 스티커 스튜디오 화면. 512x512 투명 캔버스 + 그리기 도구 패널 + 되돌리기/복사삭제 연결. 에러 없이 화면이 뜨고 캔버스가 나타나는지 확인하는 테스트

## 제대로 됐는지 어떻게 확인했나?

- `npm run lint`: 통과 (경고 없음)
- `npm run build`: 통과 (정상 빌드됨)
- `npm run test`: 전체 27개 테스트 파일, 156개 테스트 모두 통과
  - 신규 테스트 3개 모두 통과:
    - 데스크톱에서 diary 아이콘을 클릭하면 다이어리 여는 콜백만 호출되고, 스티커 스튜디오 콜백은 호출되지 않는지
    - 데스크톱에서 스티커 스튜디오 아이콘을 클릭하면 그 반대로 정확히 동작하는지
    - 스티커 스튜디오 화면이 에러 없이 뜨고, 캔버스 엘리먼트가 화면에 나타나는지
  - 기존 테스트(다이어리 캔버스, 창고, 그리기 도구 등)가 수정 없이 그대로 통과 — 다이어리 쪽 진입 경로나 동작이 이번 변경으로 전혀 흔들리지 않았다는 뜻

## 전체 그림에서 지금 어디?

```
[0] canvas-size-param        <- 완료
[1] sticker-asset-type       <- 완료
[2] sticker-studio-shell     <- 지금 여기 (완료)
[3] sticker-image-upload-crop
[4] sticker-lasso-cutout
[5] sticker-outline
[6] sticker-save-and-reuse
```

이번 단계로 "빈 방"(스티커 스튜디오 창)이 만들어졌다. 다음 단계(`sticker-image-upload-crop`)에서는 이 방에 "사진을 가져와 붙이고 필요한 부분만 잘라내는" 기능을 넣는다. 그다음(`sticker-lasso-cutout`)에는 손으로 그린 곡선을 따라 배경을 지우는 기능, 그다음(`sticker-outline`)에는 흰 테두리를 두르는 기능이 차례로 붙는다. 마지막(`sticker-save-and-reuse`)에서 "완성" 버튼을 눌러 저장하고, 다이어리 화면에서 그 스티커를 꺼내 쓸 수 있게 연결한다.

## 아직 안 된 것 (솔직하게)

- 사진 업로드, 자르기(크롭), 배경 지우기(누끼), 흰 테두리 추가, 저장 버튼 — 전부 아직 없다. 지금은 흰 도화지(투명 캔버스) 위에 그림만 그릴 수 있는 상태다.
- Ctrl+V(붙여넣기)로 이미지 파일을 붙였을 때 그걸 "에셋"으로 정식 등록하는 기능은 아직 연결하지 않았다 — 이 단계에서는 그 연결 고리를 비워뒀다. 클립보드에 이미지가 있어도 이번 단계에서는 별도로 등록되지 않고, 기존에 캔버스에 있던 도형을 복사/붙여넣기 하는 것만 확실히 동작한다.
- "완성한 스티커를 저장" 버튼 자체가 없다 — 그림을 그려도 저장할 방법이 아직 없고, 창을 닫으면 그린 내용은 사라진다. 이건 의도된 것으로, 마지막 단계에서 저장 기능이 들어간다.
- 저장된 스티커를 불러와 다시 편집하는 "내 스티커" 목록도 아직 없다.
