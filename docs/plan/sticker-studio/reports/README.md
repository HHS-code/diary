# 빌드 리포트 — diary (sticker-studio)

각 step이 무엇을 했는지 위에서 아래로 읽으면 프로젝트가 자라는 과정이 보인다.

- [완료] **Step 0: canvas-size-param** — useFabricCanvas(canvasElementRef, initialCanvasJSON, onSave, options) 시그니처 유지, options에 logicalSize?: {width,height}와 backgroundColor?: string 추가. 내부에서 const logicalSize = options?.logicalSize ?? LOGICAL_CANVAS 계산해 Canvas 생성 width/height와 onSave 두 번째 인자(canvasSize)에 사용. backgroundColor는 options?.backgroundColor ?? '#ffffff'. LOGICAL_CANVAS export(1600x1000)는 이름/값 그대로 유지, 옵션 미전달 시 기존 다이어리 동작 100% 동일(회귀 테스트로 확인). 다음 step(sticker-studio-shell)은 STICKER_LOGICAL_CANVAS={width:512,height:512}와 backgroundColor:'transparent'를 이 options로 넘겨 사용하면 됨. 산출 파일: src/hooks/useFabricCanvas.js, src/hooks/useFabricCanvas.test.js. ([리포트](step0-canvas-size-param.md))
- [완료] **Step 1: sticker-asset-type** — assetStorage.js AssetRecord.type / saveAsset / listAssets JSDoc을 'image' | 'font' | 'sticker'로 확장(구현 로직 변경 없음 — 문자열 그대로 저장/필터). useAssetLibrary()가 stickers: AssetRecord[]와 registerSticker(blob: Blob, filename: string) -> Promise<string>을 반환하도록 확장 — registerImage/registerFont와 달리 File이 아니라 이미 만들어진 Blob을 받는다(mimeType은 blob.type || 'image/png'). classifyAssetType은 변경하지 않음(스티커는 확장자 분류 대상이 아님). AssetImportPanel.jsx는 아직 stickers를 노출하지 않음 — 그건 step 6 범위. 산출 파일: src/storage/assetStorage.js, src/storage/assetStorage.test.js, src/hooks/useAssetLibrary.js, src/hooks/useAssetLibrary.test.jsx. ([리포트](step1-sticker-asset-type.md))
- [완료] **Step 2: sticker-studio-shell** — Desktop.jsx가 onOpenStickerStudio prop을 받아 diary 아이콘 옆에 스티커 스튜디오 아이콘 버튼(aria-label="sticker studio", diary 버튼도 aria-label="diary" 추가)을 렌더링. App.jsx에 isStickerStudioOpen state를 isDiaryOpen과 나란히 추가해 Window(title="sticker studio")로 <StickerStudio />를 연다. 신규 src/components/StickerStudio/StickerStudio.jsx: 모듈 상수 STICKER_LOGICAL_CANVAS={width:512,height:512}로 useFabricCanvas(el, null, null, {logicalSize, backgroundColor:'transparent'}) 호출(onSave 없음=오토세이브 비활성, 저장은 이후 step), usePaintTools/useCanvasHistory/useCanvasKeyboardShortcuts(등록 콜백 없이 호출 — Ctrl+C/V 객체 복제·Delete는 동작, 이미지 파일 붙여넣기 등록은 이후 step 범위)를 그대로 연결하고 PaintToolbox + <canvas>를 렌더링. src/assets/icons.js에 stickerStudioIcon export 추가(소스: reference/Windows XP Icons/Paint.png → src/assets/sticker-studio-icon.png). 다음 step(sticker-image-upload-crop)은 StickerStudio.jsx에 이미지 업로드 버튼과 크롭 UI를 추가하면 됨 — 캔버스/그리기 도구 뼈대는 이미 갖춰져 있음. 산출 파일: src/assets/icons.js, src/assets/sticker-studio-icon.png, src/components/Desktop/Desktop.jsx, src/components/Desktop/Desktop.test.jsx, src/App.jsx, src/components/StickerStudio/StickerStudio.jsx, src/components/StickerStudio/StickerStudio.test.jsx. ([리포트](step2-sticker-studio-shell.md))
- [완료] **Step 3: sticker-image-upload-crop** — 신규 src/components/StickerStudio/StickerImageUpload.jsx: <input type="file">로 고른 이미지를 FabricImage.fromURL(createAssetObjectURL(file))로 캔버스에 추가, fc.getWidth()/getHeight() 기준(하드코딩 상수 대신 실제 캔버스 크기 사용)으로 스케일 다운. assetStorage 저장/assetId 부여 없음(완성 버튼 전까지 임시 오브젝트, ADR-4). 신규 src/hooks/useStickerCropTool.js: useStickerCropTool(fabricCanvasRef) -> { isCropping, startCropping, cancelCropping, applyCrop }. 크롭 모드 진입 시 canvas.selection/skipTargetFind를 잠시 끄고 mouse:down/move/up(scenePoint)으로 점선 Rect를 그리며, applyCrop은 그 Rect를 제거한 뒤 canvas.toCanvasElement(1, {left,top,width,height})로 rasterize하고 기존 오브젝트를 모두 제거해 new FabricImage(croppedElement) 하나만 캔버스 중앙에 남긴다(assetStorage 저장 없음). StickerStudio.jsx가 두 기능을 사이드바에 연결(자르기/크롭 적용/취소 버튼). 산출 파일: src/components/StickerStudio/StickerImageUpload.jsx, src/components/StickerStudio/StickerImageUpload.test.jsx, src/hooks/useStickerCropTool.js, src/hooks/useStickerCropTool.test.jsx, src/components/StickerStudio/StickerStudio.jsx. 다음 step(sticker-lasso-cutout)은 usePaintTools에 'lasso' 도구를 추가하고 신규 src/fabric/stickerCutout.js에서 path:created로 얻은 Path를 clipPath로 미리보기 후 rasterize하면 됨 — 캔버스 위 임시 오브젝트를 다루는 패턴(스케일/좌표 변환 없이 논리 좌표 그대로)은 이번 step과 동일. ([리포트](step3-sticker-image-upload-crop.md))
