# 빌드 리포트 — diary (sticker-studio)

각 step이 무엇을 했는지 위에서 아래로 읽으면 프로젝트가 자라는 과정이 보인다.

- [완료] **Step 0: canvas-size-param** — useFabricCanvas(canvasElementRef, initialCanvasJSON, onSave, options) 시그니처 유지, options에 logicalSize?: {width,height}와 backgroundColor?: string 추가. 내부에서 const logicalSize = options?.logicalSize ?? LOGICAL_CANVAS 계산해 Canvas 생성 width/height와 onSave 두 번째 인자(canvasSize)에 사용. backgroundColor는 options?.backgroundColor ?? '#ffffff'. LOGICAL_CANVAS export(1600x1000)는 이름/값 그대로 유지, 옵션 미전달 시 기존 다이어리 동작 100% 동일(회귀 테스트로 확인). 다음 step(sticker-studio-shell)은 STICKER_LOGICAL_CANVAS={width:512,height:512}와 backgroundColor:'transparent'를 이 options로 넘겨 사용하면 됨. 산출 파일: src/hooks/useFabricCanvas.js, src/hooks/useFabricCanvas.test.js. ([리포트](step0-canvas-size-param.md))
- [완료] **Step 1: sticker-asset-type** — assetStorage.js AssetRecord.type / saveAsset / listAssets JSDoc을 'image' | 'font' | 'sticker'로 확장(구현 로직 변경 없음 — 문자열 그대로 저장/필터). useAssetLibrary()가 stickers: AssetRecord[]와 registerSticker(blob: Blob, filename: string) -> Promise<string>을 반환하도록 확장 — registerImage/registerFont와 달리 File이 아니라 이미 만들어진 Blob을 받는다(mimeType은 blob.type || 'image/png'). classifyAssetType은 변경하지 않음(스티커는 확장자 분류 대상이 아님). AssetImportPanel.jsx는 아직 stickers를 노출하지 않음 — 그건 step 6 범위. 산출 파일: src/storage/assetStorage.js, src/storage/assetStorage.test.js, src/hooks/useAssetLibrary.js, src/hooks/useAssetLibrary.test.jsx. ([리포트](step1-sticker-asset-type.md))

App.jsx에 isStickerStudioOpen state를 isDiaryOpen과 나란히 추가해 Window(title="sticker studio")로 <StickerStudio />를 연다. 신규 src/components/StickerStudio/StickerStudio.jsx: 모듈 상수 STICKER_LOGICAL_CANVAS={width:512,height:512}로 useFabricCanvas(el, null, null, {logicalSize, backgroundColor:'transparent'}) 호출(onSave 없음=오토세이브 비활성, 저장은 이후 step), usePaintTools/useCanvasHistory/useCanvasKeyboardShortcuts(등록 콜백 없이 호출 — Ctrl+C/V 객체 복제·Delete는 동작, 이미지 파일 붙여넣기 등록은 이후 step 범위)를 그대로 연결하고 PaintToolbox + <canvas>를 렌더링. src/assets/icons.js에 stickerStudioIcon export 추가(소스: reference/Windows XP Icons/Paint.png → src/assets/sticker-studio-icon.png). 다음 step(sticker-image-upload-crop)은 StickerStudio.jsx에 이미지 업로드 버튼과 크롭 UI를 추가하면 됨 — 캔버스/그리기 도구 뼈대는 이미 갖춰져 있음. 산출 파일: src/assets/icons.js, src/assets/sticker-studio-icon.png, src/components/Desktop/Desktop.jsx, src/components/Desktop/Desktop.test.jsx, src/App.jsx, src/components/StickerStudio/StickerStudio.jsx, src/components/StickerStudio/StickerStudio.test.jsx. - [완료] **Step 3: sticker-image-upload-crop** — 신규 src/components/StickerStudio/StickerImageUpload.jsx: <input type="file">로 고른 이미지를 FabricImage.fromURL(createAssetObjectURL(file))로 캔버스에 추가, fc.getWidth()/getHeight() 기준(하드코딩 상수 대신 실제 캔버스 크기 사용)으로 스케일 다운. assetStorage 저장/assetId 부여 없음(완성 버튼 전까지 임시 오브젝트, ADR-4). 신규 src/hooks/useStickerCropTool.js: useStickerCropTool(fabricCanvasRef) -> { isCropping, startCropping, cancelCropping, applyCrop }. 크롭 모드 진입 시 canvas.selection/skipTargetFind를 잠시 끄고 mouse:down/move/up(scenePoint)으로 점선 Rect를 그리며, applyCrop은 그 Rect를 제거한 뒤 canvas.toCanvasElement(1, {left,top,width,height})로 rasterize하고 기존 오브젝트를 모두 제거해 new FabricImage(croppedElement) 하나만 캔버스 중앙에 남긴다(assetStorage 저장 없음). StickerStudio.jsx가 두 기능을 사이드바에 연결(자르기/크롭 적용/취소 버튼). 산출 파일: src/components/StickerStudio/StickerImageUpload.jsx, src/components/StickerStudio/StickerImageUpload.test.jsx, src/hooks/useStickerCropTool.js, src/hooks/useStickerCropTool.test.jsx, src/components/StickerStudio/StickerStudio.jsx. 다음 step(sticker-lasso-cutout)은 usePaintTools에 'lasso' 도구를 추가하고 신규 src/fabric/stickerCutout.js에서 path:created로 얻은 Path를 clipPath로 미리보기 후 rasterize하면 됨 — 캔버스 위 임시 오브젝트를 다루는 패턴(스케일/좌표 변환 없이 논리 좌표 그대로)은 이번 step과 동일. ([리포트](step3-sticker-image-upload-crop.md))
- [완료] **Step 4: sticker-lasso-cutout** — usePaintTools.js에 'lasso' 도구 추가: buildBrush가 얇은 파란 점선(#0080ff, width 1, strokeDashArray [4,4]) PencilBrush를 반환하고, path:created 구독(pinCreatedStroke)이 tool==='lasso'일 때 isFreeDrawing/erasable 태그를 붙이지 않고 canvas.remove(path)로 즉시 제거한다(스테일 클로저 방지를 위해 이 effect의 deps에 tool 추가 — 매 tool 변경마다 재구독). 신규 src/fabric/stickerCutout.js: previewLassoCutout(targetImage, lassoPath)는 util.sendObjectToPlane(lassoPath, undefined, targetImage.calcTransformMatrix())로 lassoPath를 targetImage 로컬 좌표계로 옮겨 clipPath로 지정(주의: sendObjectToPlane은 'fabric'의 top-level export가 아니라 util 네임스페이스에만 있음 — import { util } from 'fabric'). commitLassoCutout(canvas, targetImage)는 targetImage.getBoundingRect()로 화면 절대좌표 경계를 구해 canvas.toCanvasElement(1, {left,top,width,height,filter})로 rasterize한 뒤 새 FabricImage로 교체 — 이 Fabric 버전(v7)은 FabricObject 기본 originX/originY가 'center'라 새 이미지에 originX:'left'/originY:'top'을 명시하고 getBoundingRect()의 left/top을 그대로 써야 원래 시각적 위치가 보존된다(기존 StickerImageUpload.jsx/useStickerCropTool.js의 `left:(canvasWidth-imgWidth)/2` 중앙 배치 공식은 이 center-origin 기본값과 맞지 않아 실제로는 이미지가 중앙이 아니라 좌상단 쪽으로 치우쳐 배치되는 것을 Playwright로 실측 확인함 — 이번 step 범위 밖이라 수정하지 않았고, 다음 관련 step에서 다룰 필요가 있으면 참고). StickerStudio.jsx가 path:created를 별도 구독(캔버스 폴링 패턴 재사용)해 tool==='lasso'일 때 resolveLassoCutoutTarget(canvas)(이미지 1개면 그것, 아니면 getActiveObject())로 previewLassoCutout 호출, '누끼 적용' 버튼이 commitLassoCutout 호출. PaintToolbox.jsx에 '올가미'(MdGesture) 버튼 추가. Playwright로 실제 브라우저에서 lasso 미리보기(부분 클립)와 누끼 적용(rasterize 후 clipPath 없는 단일 이미지로 교체) 모두 실측 확인 완료. 산출 파일: src/hooks/usePaintTools.js, src/hooks/usePaintTools.test.js, src/fabric/stickerCutout.js, src/fabric/stickerCutout.test.js, src/components/PaintToolbox/PaintToolbox.jsx, src/components/PaintToolbox/PaintToolbox.test.jsx, src/components/StickerStudio/StickerStudio.jsx. 다음 step(sticker-outline)은 commitLassoCutout이 만든 clipPath 없는 순수 rasterized FabricImage(또는 크롭/원본 이미지)를 입력으로 받아 src/fabric/stickerOutline.js에서 오프스크린 다방향 스탬프로 흰 테두리를 생성하면 됨. ([리포트](step4-sticker-lasso-cutout.md))
- [완료] **Step 5: sticker-outline** — 신규 src/fabric/stickerOutline.js: createOutlinedSticker(sourceCanvasElement: HTMLCanvasElement, thicknessPx: number) -> HTMLCanvasElement — Fabric/DOM 상태 비의존 순수 함수. thicknessPx<=0이면 원본을 복사한 캔버스 반환. 절차: source-in 컴포지트로 흰 실루엣 생성 → resolveStampDirectionCount(thicknessPx)(8~32, thicknessPx*2로 스케일)만큼 원 둘레 방향으로 실루엣을 drawImage 반복 스탬핑 → 그 위에 원본을 다시 그려 합성(ADR-6). 캔버스 크기는 소스와 동일하게 유지(확장 안 함) — 소스 이미지가 가장자리에 여백이 없으면 테두리가 잘릴 수 있음(이번 step 범위 밖, 다음 저장 step에서 필요시 고려). StickerStudio.jsx에 '테두리 추가' 토글 버튼 + 두께 슬라이더(0~20px, 기본 5) + 100ms 디바운스 미리보기(별도 <canvas> 컨테이너, replaceChildren로 부착) + '적용' 버튼 추가. '적용'을 누르면 미리보기 결과가 outlinedResultRef(useRef)에 저장되고 편집기가 닫힌다 — 누르지 않으면 outlinedResultRef.current는 null로 남아 '테두리 없이 저장' 경로가 자동 성립. Playwright로 실제 브라우저에서 두께 0(원본 스트로크 폭만) vs 5(스트로크 두 변이 하나의 넓은 불투명 띠로 합쳐짐)의 alpha 프로파일 차이를 getImageData로 실측 확인. 다음 step(sticker-save-and-reuse)은 outlinedResultRef.current가 있으면 그 캔버스를, 없으면 fabricCanvas.toCanvasElement() 결과를 toBlob('image/png')해 assetStorage.saveAsset({type:'sticker', ...})로 저장하면 됨. 산출 파일: src/fabric/stickerOutline.js, src/fabric/stickerOutline.test.js, src/components/StickerStudio/StickerStudio.jsx, src/components/StickerStudio/StickerStudio.test.jsx. ([리포트](step5-sticker-outline.md))
