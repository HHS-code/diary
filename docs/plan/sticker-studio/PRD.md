# PRD — 스티커 스튜디오 (sticker-studio)

## 배경 / 왜

현재 다이어리 캔버스의 `StickerPalette`는 고정된 이모지 30종(유니코드 문자)을 붙이는 기능만 제공한다. 사용자가 직접 그리거나, 사진을 업로드해 배경을 제거한 뒤 "나만의 스티커"로 등록해 다이어리에 재사용할 방법이 없다.

`asset-import-pipeline`(IndexedDB 에셋 저장소, 공용 파일 등록 UI)과 `animated-gif-support`(GIF 프레임 처리 인프라)가 완료되어, 스티커를 이미지 에셋으로 등록·재사용할 기반은 이미 갖춰져 있다. 이 phase는 그 위에 "스티커를 만드는" 전용 편집 화면을 추가한다.

## 목표

- 데스크톱에서 독립적으로 열리는 "스티커 스튜디오" 앱을 만든다.
- 사용자가 직접 그려서, 또는 이미지를 업로드해 편집(그리기·크롭·배경 제거·테두리 추가)해서 스티커를 만들 수 있다.
- 완성한 스티커는 투명 배경 PNG로 저장되어 다이어리 캔버스(`AssetImportPanel`)에서 바로 꺼내 쓸 수 있다.
- 저장된 스티커를 다시 열어 편집할 수 있다(단, 원본은 보존되고 편집 결과는 항상 새 스티커로 저장된다).

## 대상 사용자

기존과 동일. 혼자 쓰는 개인 다이어리 사용자.

## In-scope

### 1. 데스크톱 진입점

- 데스크톱에 "스티커 스튜디오" 아이콘을 새로 추가한다(`reference/Windows XP Icons/Paint.png` 사용).
- 클릭하면 `App.jsx`의 `isStickerStudioOpen` 상태로 독립 `Window`가 열린다 — 다이어리 앱(`isDiaryOpen`)과 동일한 패턴.
- 열리면 곧바로 빈 투명 캔버스가 표시된다(그리기/업로드를 고르는 시작 화면 없음).

### 2. 캔버스

- 작은 정사각형 고정 크기(예: 512×512), 배경은 투명.
- 다이어리와 동일하게 확대/축소 없이 고정 크기로 표시한다.

### 3. 스티커 제작 — 그리기

- 다이어리의 그리기 도구(`usePaintTools`: 연필/브러시/에어브러시/지우개)를 그대로 재사용한다.
- Undo/Redo(Ctrl+Z/Y)를 지원한다(`useCanvasHistory` 재사용).
- 오브젝트 복사/붙여넣기(Ctrl+C/V)와 Delete를 지원한다(`useCanvasKeyboardShortcuts` 재사용). 클립보드에 이미지 파일이 있으면 우선 이 스튜디오의 에셋 등록 경로로 처리한다.

### 4. 스티커 제작 — 이미지 업로드 편집

- 사이드바 "이미지 업로드" 버튼으로 언제든 이미지를 불러와 캔버스에 올릴 수 있다.
- 불러온 이미지 위에 그리기 도구로 그리거나 덧칠할 수 있다.
- **크롭**: 사각형 영역을 지정해 이미지를 잘라낸다.
- **배경 제거(누끼따기)**: 올가미(라소) 도구로 자유곡선을 그려 폐곡선을 만들면, 그 안쪽만 남고 바깥은 투명해진다.

### 5. 흰색 테두리

- 저장 직전 단계에서 "테두리 추가" 버튼을 눌러 스티커 실루엣(불투명 픽셀 경계)을 따라 흰색 테두리를 생성한다.
- 두께는 슬라이더로 조절하며, 적용 결과를 미리보기하면서 결정한다.

### 6. 저장 및 재사용

- 완성한 스티커는 `assetStorage`에 `type: 'sticker'`로 저장한다(PNG, 알파 채널로 투명 배경 유지).
- `AssetImportPanel`(다이어리 캔버스 사이드바)에 스티커 목록이 노출되어 다이어리 캔버스에 배치할 수 있다.
- 스티커 스튜디오 사이드바에 "내 스티커" 패널을 두어, 저장된 스티커 목록을 썸네일로 보여준다. 클릭하면 해당 스티커가 현재 캔버스에 불러와져 편집을 이어갈 수 있다.
- 저장은 항상 "다른 이름으로 저장" 방식이다 — 기존 스티커를 불러와 편집 후 저장해도 원본은 그대로 남고, 편집 결과는 새 스티커(새 assetId)로 추가된다. 덮어쓰기는 없다.

## Out-of-scope (명시)

- 스티커 삭제/이름 변경 등 관리 UI — 저장·재편집(다른 이름으로 저장)까지만.
- 자동/반자동 배경 제거(예: 원클릭 배경 감지) — 이번엔 올가미 수동 선택만 지원.
- 흰색 외 다른 색상의 테두리, 테두리 스타일(점선 등) — 흰색 단색, 두께 조절만.
- 스티커를 애니메이션(GIF)으로 만드는 기능 — 정지 이미지만.
- 캔버스 크기를 사용자가 바꾸는 기능 — 고정 크기만.
- `AssetImportPanel`/`useAssetLibrary`의 이미지·폰트 목록 UI 자체의 재설계 — 스티커 타입을 추가로 노출하는 것 이상은 하지 않는다.
- `youtube-embed`, `desktop-custom`, `movie-review-standalone` 등 로드맵의 다른 phase 범위.
- `ui-polish` 범위의 스타일 정비.

## 성공 기준

- 데스크톱 아이콘을 클릭하면 스티커 스튜디오 창이 열리고, 빈 투명 캔버스가 보인다.
- 그리기 도구로 그림을 그려 스티커를 만들고 저장할 수 있다.
- 이미지를 업로드해 크롭하고, 올가미로 배경을 제거해(누끼따기) 원하는 영역만 남길 수 있다.
- 흰색 테두리를 두께 조절하며 미리보기 후 추가할 수 있다.
- 완성한 스티커는 투명 배경 PNG로 저장되고, 다이어리 캔버스의 `AssetImportPanel`에서 선택해 배치할 수 있다.
- 스티커 스튜디오의 "내 스티커" 패널에서 기존 스티커를 불러와 다시 편집하고, "다른 이름으로 저장"하면 원본은 그대로 남고 편집본이 새로 추가된다.
- Ctrl+Z/Y(undo/redo), Ctrl+C/V(복사/붙여넣기), Delete가 다이어리와 동일하게 동작한다.
- 위 변경 전체에 걸쳐 기존 다이어리 캔버스 기능(저장/불러오기, export, 오토세이브, 오브젝트 조작)이 이전과 동일하게 동작한다.

## 진행 방식

1. 문서(PRD/ADR/architecture) 합의 → 실행 승인
2. useFabricCanvas 캔버스 크기 파라미터화 → 데스크톱 진입점 → 그리기/업로드 도구 재사용 연결 → 크롭 → 누끼따기(올가미) → 흰색 테두리 → assetStorage sticker 타입 저장 → AssetImportPanel/내 스티커 패널 연동 → 완료 시 사용자에게 결과 설명
