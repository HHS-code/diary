# PRD — 고정 캔버스 해상도 + 표시 스케일

## 배경 / 왜

지금은 다이어리 창을 열 때마다 그 기기의 가용 공간을 측정해 캔버스를 그 크기로 생성한다. 그래서 맥북(작은 화면)에서는 캔버스가 작게 만들어지고, 기기를 오갈 때마다 오브젝트 좌표가 기기 크기에 맞춰 반복 변환된다 — 화면 비율이 다르면 가로/세로 배율이 달라 스티커·글자가 미세하게 찌그러질 수 있고, PNG 출력물의 크기·비율도 기기마다 달라진다.

## 목표

- 캔버스의 논리 크기를 **1600×1000(16:10)으로 고정**한다. 저장 데이터는 어느 기기에서나 이 좌표계 기준.
- 화면에는 각 기기 창에 맞게 **비율 유지 축소/확대 표시**만 한다 (데이터 변환 없음).
- **어느 기기에서 PNG를 내보내도 항상 1600×1000의 동일한 출력물**이 나온다.

## 대상 사용자

기존과 동일. 데스크톱과 맥북을 오가며 같은 다이어리(JSON export/import)를 쓰는 사용자.

## In-scope

- 캔버스 논리 크기 1600×1000 고정 (상수).
- 표시 스케일: 창에서 확보한 공간에 맞춰 `scale = min(가용폭/1600, 가용높이/1000)`으로 축소/확대 표시. Fabric.js의 zoom + 엘리먼트 크기 조정 방식 — 오브젝트 좌표는 항상 1600×1000 평면에 존재하고, 마우스 조작(드래그·크기조절·클릭)은 fabric이 zoom을 반영해 정확히 동작한다.
- PNG 내보내기: 표시 스케일과 무관하게 항상 1600×1000 픽셀로 출력.
- 기존 저장 데이터(기기별 크기로 저장됨) 1회 변환: 로드 시점에 **비율 유지 축소/확대 + 가운데 정렬**로 1600×1000 좌표계에 옮긴다. 찌그러짐 없음, 비율이 다르면 상하/좌우에 빈 공간(흰 배경) 허용.
- 저장 시 `canvasSize`는 항상 1600×1000으로 기록 (스키마 유지, 변환 완료 표시 역할).

## Out-of-scope (명시)

- 창 리사이즈 실시간 반응 — 기존대로 편집 화면 진입 시 1회 측정.
- 캔버스 크기/비율을 사용자가 선택하는 기능 — 1600×1000 하나로 고정.
- 갤러리 썸네일 로직 변경 — 저장된 canvasSize대로 렌더링하는 기존 동작 유지 (아직 안 열어본 구버전 데이터는 이전 비율로 보이다가, 한 번 열면 1600×1000으로 변환됨).

## 성공 기준

- 데스크톱과 맥북(뷰포트 크기가 다른 두 환경)에서 같은 다이어리의 PNG를 내보내면 두 파일 모두 정확히 1600×1000 픽셀이다.
- 작은 화면에서 캔버스가 잘리지 않고 축소 표시되며, 오브젝트 드래그/크기조절/클릭 위치가 정확하다.
- 기기를 오가며 열고 저장해도 오브젝트가 찌그러지거나 위치가 밀리지 않는다 (변환은 구버전 데이터당 최초 1회뿐).
- 구버전 데이터(예: 800×600)를 열면 내용물이 비율 그대로 가운데 배치된다.

## 진행 방식

1. 문서(PRD/architecture) 합의 → 실행 승인
2. 고정 좌표계 + 표시 스케일 → 기존 데이터 fit 변환 → PNG 고정 출력 → 완료 시 결과 설명
