# PRD — 애니메이션 GIF 지원 (animated-gif-support)

## 배경 / 왜

`asset-import-pipeline` phase(완료)에서 이미지 파일을 캔버스에 올릴 수 있게 됐지만, GIF 파일을 올려도 첫 프레임에서 멈춘 정지 이미지로만 표시된다. 이는 버그가 아니라 Fabric.js의 `FabricImage.fromURL()`이 `<img>` 엘리먼트를 `drawImage`로 캔버스에 한 프레임만 캡처하는 근본적인 동작 방식 때문이다.

애니메이션 GIF를 실제로 움직이는 상태로 캔버스에서 재생하려면, GIF 프레임을 직접 디코딩해서 캔버스에 주기적으로 다시 그리는 별도 구현이 필요하다.

## 목표

- 캔버스에 올라간 GIF가 원본과 동일하게 애니메이션 재생된다.
- 기존 오브젝트 기능(회전, 크기조절, 이동, 복사, 삭제, 레이어 순서, 다중선택)이 애니메이션 GIF 오브젝트에도 동일하게 동작한다.
- 캔버스를 저장했다가 다시 불러와도 애니메이션이 유지된다.
- GIF를 여러 개 동시에 올려도 렌더링 성능이 눈에 띄게 저하되지 않는다.

## 대상 사용자

기존과 동일. 혼자 쓰는 개인 다이어리 사용자.

## In-scope

### 1. GIF 프레임 디코딩

- `gifuct-js` 등 경량 라이브러리로 GIF 바이너리를 파싱해 프레임 이미지 배열과 각 프레임의 표시 시간(delay)을 추출한다.
- 프레임이 1개뿐인 정적 GIF는 애니메이션 처리 없이 기존 정지 이미지(`FabricImage`) 경로로 그대로 처리한다.

### 2. 캔버스 애니메이션 오브젝트

- 프레임이 2개 이상인 GIF는 새 애니메이션 전용 Fabric 오브젝트로 캔버스에 추가한다.
- 이 오브젝트는 기존 오브젝트 툴바(회전, 크기조절, 삭제, 복사, 레이어 순서)와 다중선택 이동을 포함한 표준 오브젝트 기능을 동일하게 지원한다.

### 3. 공유 렌더 루프

- 모든 애니메이션 GIF 오브젝트는 캔버스당 하나의 공유 `requestAnimationFrame` 루프에 등록된다.
- 매 틱마다 "다음 프레임으로 넘어갈 시각"이 된 오브젝트만 프레임을 갱신하고, `canvas.renderAll()`은 틱당 최대 1회만 호출한다.
- 개별 GIF 개수에 따른 별도 상한을 두지 않는다 — 렌더 호출 빈도가 GIF 개수와 무관하게 고정되도록 설계해 성능 저하를 구조적으로 방지한다.

### 4. 등록 경로 통합

- `AssetImportPanel`을 통해 등록된 이미지를 캔버스에 배치하는 경로(`canvasAssetPlacement.js`)에서 GIF를 지원한다.
- 사이드바 `ImageUploadButton`(현재 base64 dataUrl을 직접 쓰며 IndexedDB를 거치지 않음)을 `assetStorage`(IndexedDB) 경로로 전환해, 이 버튼으로 올리는 이미지도 `assetId`를 부여받도록 한다. GIF뿐 아니라 이 버튼으로 올리는 모든 이미지에 적용되는 변경이다.

### 5. 저장 및 재로드

- 애니메이션 GIF 오브젝트는 `assetId`(IndexedDB 참조)를 직렬화 데이터에 포함한다. GIF 프레임 배열 자체는 저장하지 않는다.
- 캔버스를 다시 불러올 때 `assetId`로 IndexedDB에서 원본 GIF blob을 다시 조회해 프레임을 재파싱하고, 애니메이션 오브젝트로 복원한다.

## Out-of-scope (명시)

- 캔버스 배경(`isBackground` 태그, `useCanvasBackground.js`)으로 설정하는 GIF의 애니메이션 재생 — 배경은 계속 정지 이미지로 처리한다.
- PNG export(모아보기 갤러리, 내보내기 이미지)에서 GIF 애니메이션 반영 — PNG는 태생적으로 정지 포맷이므로 export 시점의 현재 프레임을 그대로 캡처한다. 추가 구현 없음.
- 동시 재생 GIF 개수의 인위적인 제한이나 그에 따른 경고 UI.
- GIF 편집(트리밍, 프레임 삭제, 재생 속도 조절) — 원본 그대로 재생만 한다.
- `sticker-studio`, `youtube-embed` 등 로드맵의 다른 phase 범위.
- `ui-polish` 범위의 스타일 정비.

## 성공 기준

- 애니메이션 GIF를 캔버스에 올리면 원본과 동일한 속도로 움직인다.
- 애니메이션 GIF 오브젝트를 회전·크기조절·이동·복사·삭제·레이어 순서 변경·다중선택 이동해도 정상 동작한다.
- 캔버스를 저장 후 새로고침해도 애니메이션 GIF가 계속 움직인다.
- 프레임이 1개뿐인 정적 GIF는 기존과 동일하게 정지 이미지로 표시된다.
- GIF를 여러 개(예: 5개 이상) 동시에 캔버스에 올려도 다른 오브젝트 조작(드래그, 회전 등)이 눈에 띄게 끊기지 않는다.
- 위 변경 전체에 걸쳐 기존 캔버스 저장/불러오기, JSON/PNG export, 오토세이브, 오브젝트 복사/삭제 기능이 이전과 동일하게 동작한다.

## 진행 방식

1. 문서(PRD/ADR/architecture) 합의 → 실행 승인
2. GIF 프레임 디코딩 → 애니메이션 오브젝트 클래스 → 공유 렌더 루프 → 등록 경로(AssetImportPanel, ImageUploadButton) 통합 → 저장/재로드 → 완료 시 사용자에게 결과 설명
