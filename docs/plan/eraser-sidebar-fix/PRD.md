# PRD — 사이드바 패널 잘림 수정 + 지우개 실시간화

2026-07-17 합의 (coplan OQ-1~5).

## 문제

1. 창 높이가 부족하면 사이드바 패널(그리기/스티커/배경/오브젝트)이 압축되어 내용이 잘린다.
   원인: 세로 flex 컨테이너의 자식 패널들이 `overflow: hidden`이라 최소 높이 보호 없이 줄어든다.
2. 지우개가 드래그 중에는 가는 선만 그려지고, 마우스를 떼야 한꺼번에 지워진다.
   원인: @erase2d의 실시간 지우기 미리보기가 이 앱 환경(fabric 7.4 + 표시 배율 zoom)에서 깨져 있다.

## 목표 (성공 기준)

1. **패널 잘림 없음**: 어떤 창 높이에서도 패널 내용이 잘리지 않는다.
   공간이 부족하면 사이드바가 세로 스크롤된다.
2. **실시간 지우기**: 지우개 드래그 중 지나간 자리가 그 즉시 지워져 보인다 (마우스를 떼기 전에).
3. **지우개 커서**: 지우개 도구 선택 시 마우스 위치에 지우개 크기만큼의 사각형 커서가 보인다.
4. **지우개 전용 굵기**: 지우개 선택 시 굵기 버튼이 8/16/24/32로 바뀌고 기본 16.
   브러시·에어브러시는 기존 2/4/6/8 그대로.

## In-scope

- 사이드바 패널의 flexShrink 수정 (DiaryCanvas 사이드바)
- 지우개 실시간 미리보기 수정 — erase2d 미리보기 합성이 깨진 원인을 디버그해
  브러시 설정/서브클래스로 고치고, 불가하면 미리보기만 자체 구현
- 지우개 전용 굵기 체계 (usePaintTools + PaintToolbox)
- 지우개 크기 커서 표시

## Out-of-scope

- 지우기 **데이터 커밋 타이밍** 변경 — 실제 오브젝트 클리핑은 지금처럼 mouseup에 커밋
  (화면만 실시간, 체감 동일)
- 연필/브러시/에어브러시 동작 변경, undo/redo, 저장 포맷 변경

## 검증

- 창 높이를 줄인 상태의 스크린샷에서 패널 잘림 없음 + 스크롤 확인
- 지우개 드래그 **중간** 스크린샷에서 경로가 이미 지워져 보임 (Playwright 재현)
- 기존 테스트(usePaintTools.test.js 등) 통과
