# Step 5: 캘린더와 탭 — 날짜별로 이동하기

## 이 단계를 왜 했나?

지금까지 만든 기능(스티커, 이미지, 텍스트)은 화면 하나에만 존재했다.  
날짜마다 다른 내용을 적으려면 "오늘의 페이지", "어제의 페이지"를 따로 구분해 불러올 방법이 필요하다.  
이 단계에서는 캘린더로 날짜를 고르면 그 날짜에 저장된 내용을 꺼내서 캔버스에 보여주는 구조를 완성했다.  
또한 같은 날짜 안에서 "다이어리" 탭과 "영화리뷰" 탭을 나눠 쓸 수 있도록 탭 UI도 만들었다.

## 무엇을 만들었나?

**Calendar 컴포넌트** (`src/components/Calendar/Calendar.jsx`)  
외부 라이브러리 없이 직접 구현한 월별 달력. 이전/다음 달 이동 버튼과 날짜 그리드로 이루어져 있다.  
날짜를 클릭하면 `"2026-07-10"` 같은 문자열 형태로 선택된 날짜를 상위 컴포넌트에 알린다.

**Tabs 컴포넌트** (`src/components/Tabs/Tabs.jsx`)  
"다이어리"와 "영화리뷰" 두 탭을 보여주는 간단한 탭 바. 이번 단계에서 영화리뷰 탭을 클릭하면 "영화 리뷰는 곧 추가됩니다."라는 안내 문구만 보인다.

**기존 파일 수정**  
- `useFabricCanvas.js`: 훅이 캔버스를 처음 만들 때 "이전에 저장한 내용"(canvasJSON)을 받아서 바로 복원할 수 있게 두 번째 인자를 추가했다.  
- `DiaryCanvas.jsx`: 위에서 추가한 기능을 쓸 수 있도록 `canvasJSON` prop을 받아 훅에 전달하도록 변경했다.  
- `App.jsx`: 캘린더 + 탭 + 캔버스를 한 화면에 조합하고, 날짜·탭이 바뀔 때마다 저장소에서 데이터를 꺼내 캔버스로 전달하는 흐름을 연결했다.

## 데이터가 어떻게 흐르나?

```
사용자가 달력에서 7월 10일 클릭
        |
        v
App.jsx — selectedDate = "2026-07-10"
        |
        v
loadAllDiaryData()  ←  localStorage "diary-app-data"
        |
        v
getDatePageData(allData, "2026-07-10", "diary")
        |
        +-- 저장된 내용 있음 → canvasJSON = { objects: [...], background: ... }
        +-- 없음            → canvasJSON = null
        |
        v
<DiaryCanvas key="2026-07-10-diary" canvasJSON={canvasJSON} />
        |
        v
useFabricCanvas(ref, canvasJSON)
        |
        +-- canvasJSON 있음 → canvas.loadFromJSON(canvasJSON)  : 이전 내용 복원
        +-- canvasJSON 없음 → 빈 캔버스로 시작
```

숫자 예시: 7월 10일 저장된 스티커 3개(이모지 😊, 🎉, 🌟)가 있다면,  
`canvasJSON.objects` 배열에 3개의 항목이 담겨 있고,  
`loadFromJSON`이 이를 읽어 캔버스에 그대로 그려낸다.  
7월 5일처럼 아무것도 저장하지 않은 날짜를 클릭하면 `canvasJSON = null`이므로 빈 흰 캔버스가 열린다.

## 쉽게 말하면

종이 일기장 책을 생각하면 된다.  
캘린더는 날짜 목차이고, 날짜를 클릭하는 것은 그 쪽을 펼치는 것이다.  
저장된 내용이 있는 날에는 이전에 꾸몄던 페이지가 그대로 나타나고,  
비어 있는 날에는 새 백지 페이지가 열린다.  
탭은 한 날짜 안에 "오늘의 일기" 쪽과 "오늘 본 영화" 쪽을 따로 두는 것과 같다.

## 만든 파일

## 제대로 됐는지 어떻게 확인했나?

**AC 1 — `npm run build` 성공**  
빌드 완료, 478 kB (이전 step 4의 475 kB에서 3 kB 증가).

**AC 2 — 브라우저 동작**  
- 캘린더에서 다른 날짜를 클릭하면 위에 표시된 날짜 문자열이 바뀌고 캔버스가 그 날짜 데이터로 교체된다.  
- "영화리뷰" 탭을 클릭하면 "영화 리뷰는 곧 추가됩니다." 문구가 나타난다.  
- "다이어리" 탭으로 돌아오면 캔버스가 다시 보인다.

```
------------------------------------------------------------
Step 6  자동 저장 (캔버스 변경 감지 → localStorage 저장)
Step 7  JSON / PNG 내보내기·불러오기
```

다음 Step 6이 "저장"인 이유: 이번 단계에서 불러오기 방향(storage → canvas)은 완성됐지만,  
"캔버스에서 그린 내용 → localStorage 저장" 방향은 아직 연결하지 않았다.  
날짜를 바꾸거나 새로고침하면 방금 그린 내용이 사라진다.  
Step 6에서 캔버스 변경 이벤트를 감지해 자동으로 저장하면, 이 루프가 완성된다.

## 아직 안 된 것 (솔직하게)

- **저장 안 됨**: 날짜를 바꾸거나 새로고침하면 그린 내용이 사라진다 — step 6에서 해결 예정.
- **영화리뷰 탭**: placeholder 문구만 있고 실제 기능 없음 — PRD 2순위(TMDB 연동)로 이후 단계 예정.
- **캘린더 스타일**: 기능은 되지만 폰트·색상·여백 등 비주얼 다듬기는 하지 않았다.
