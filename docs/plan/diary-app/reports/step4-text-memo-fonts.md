# Step 4: 손글씨 텍스트 메모 추가

## 이 단계를 왜 했나?

다이어리를 꾸미려면 이모지 스티커와 사진만으로는 부족하다. 날짜 옆에 한 줄 메모, 감상, 짧은 일기 문장을 손글씨처럼 써넣을 수 있어야 진짜 "꾸민 다이어리" 느낌이 난다. 이 단계에서는 손글씨 느낌의 폰트 8종 중 하나를 골라 텍스트를 캔버스에 추가하는 기능을 만들었다.

## 무엇을 만들었나?

`TextMemoButton` 컴포넌트 하나를 새로 만들었다. 이 컴포넌트는 왼쪽 도구 패널에 "텍스트 추가" 버튼으로 표시된다.

버튼을 누르면:

1. 폰트를 고르는 드롭다운 메뉴가 나타난다 (8종).
2. 텍스트 입력창이 나타난다.
3. 입력 후 Enter를 누르거나 "추가" 버튼을 클릭하면 선택한 폰트로 캔버스 중앙에 글자가 나타난다.

나타난 텍스트는 마우스로 드래그해 옮기거나, 모서리를 잡아 크기를 바꾸거나, 돌릴 수 있다. 더블클릭하면 글자를 직접 수정할 수도 있다(Fabric.js가 기본으로 제공하는 기능이라 별도로 구현할 것이 없었다).

지원 폰트 목록:

| 이름 | 특징 |
|------|------|
| Nanum Pen Script | 한글 지원, 펜 느낌 |
| Gaegu | 한글 지원, 동글동글한 체 |
| Hi Melody | 한글 지원, 귀여운 체 |
| Poor Story | 한글 지원, 빈티지 느낌 |
| Caveat | 영문 중심, 자연스러운 필기체 |
| Kalam | 영문 중심, 깔끔한 손글씨 |
| Gochi Hand | 영문 중심, 팝한 손글씨 |
| Shadows Into Light | 영문 중심, 가느다란 손글씨 |

폰트 파일은 새로 추가하지 않았다. 이전 단계에서 `index.html`에 이미 Google Fonts 링크로 8종이 연결되어 있었다.

## 데이터가 어떻게 흐르나?

```
사용자 클릭 "텍스트 추가"
        │
        ▼
[TextMemoButton] 드롭다운 + 입력창 표시
        │
   폰트 선택 + 텍스트 입력 + Enter
        │
        ▼
IText 오브젝트 생성
  fontFamily: "Gaegu" (예시)
  fontSize: 32
  left: 400, top: 300  (캔버스 중앙)
        │
        ▼
fabricCanvasRef.current.add(text)
        │
        ▼
Fabric.js 캔버스에 렌더링
(이후 드래그/크기조절/회전/더블클릭 편집은 Fabric.js 자체 처리)
```

숫자 예시: "안녕 diary" 를 입력하고 Gaegu 폰트를 선택하면, 32px 크기의 Gaegu 글씨가 캔버스 중앙(x=400, y=300)에 생긴다. 이를 드래그하면 위치가 바뀌고, 모서리 핸들을 잡으면 크기가 바뀐다.

## 쉽게 말하면

포토샵에서 텍스트 도구(T)를 클릭하면 글자를 쓸 수 있는 것처럼, 이 기능도 같은 역할을 한다. 차이는 폰트가 손글씨체 8종으로 미리 골라져 있다는 것. 원고지 위에 손으로 쓴 것처럼 보이지만 실제로는 웹 폰트가 그렇게 생겼을 뿐이다.

## 제대로 됐는지 어떻게 확인했나?

**AC 결과:**

- `npm run build` — 에러 없이 성공 (475 kB, 98ms)
- 브라우저 동작:
  - "텍스트 추가" 버튼 클릭 → 드롭다운 + 입력창 표시 확인
  - 8종 폰트 각각 선택 후 텍스트 입력 → 캔버스에 서로 다른 손글씨체로 표시됨 확인
  - 추가된 텍스트를 드래그로 이동, 핸들로 크기조절, 회전 핸들로 회전 확인
  - 더블클릭 시 Fabric.js 기본 텍스트 편집 모드 진입 확인

## 전체 그림에서 지금 어디?

```
[0] storage-layer    완료  ← localStorage 읽기/쓰기 기반
[1] canvas-shell     완료  ← Fabric.js 캔버스 뼈대
[2] sticker-palette  완료  ← 이모지 스티커 추가
[3] image-upload     완료  ← 사진/이미지 추가
[4] text-memo-fonts  완료  ← 지금 여기: 손글씨 텍스트 추가
[5] calendar-tabs    다음  ← 날짜별 페이지, 다이어리/영화리뷰 탭
[6] autosave              ← 변경 시 localStorage 자동 저장
[7] json-png-io           ← JSON 내보내기/불러오기, PNG 내보내기
```

다음 단계(calendar-tabs)에서는 지금까지 만든 캔버스를 날짜와 연결한다. 캘린더에서 날짜를 클릭하면 그 날짜의 다이어리 페이지가 열리는 구조를 만드는 것이다. 이것이 완성돼야 "날짜별 다이어리"로서 의미가 생기기 때문에 다음 순서로 정해져 있다.

## 아직 안 된 것 (솔직하게)

- 입력창 preview에서 폰트가 미리 보이기는 하지만, `<option>` 태그 안의 폰트는 운영체제에 따라 적용되지 않을 수 있다. 드롭다운 항목 자체가 선택한 폰트로 표시되는 것은 브라우저 제한사항.
- 한글을 지원하지 않는 영문 폰트(Caveat 등)로 한글을 입력하면 브라우저 기본 폰트로 fallback된다. PRD에서 "별도 처리 불필요"로 확정된 사항이므로 현재 의도된 동작.
- 저장 기능은 아직 없다(step 6에서 연결). 새로고침하면 캔버스 내용이 사라진다.
