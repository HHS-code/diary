# Step 6: 자동 저장 (autosave)

## 이 단계를 왜 했나?

캔버스에 스티커를 붙이고 브라우저를 새로고침하면 모든 내용이 사라졌다. 그림을 그려도 기억이 없는 다이어리는 의미가 없으므로, 편집 내용을 자동으로 기기에 저장하는 연결고리가 필요했다. 이 단계에서 "캔버스 편집 → 자동 저장 → 새로고침 후 복원"의 전체 흐름을 완성한다.

## 무엇을 만들었나?

이미 만들어진 세 파일을 수술적으로 수정했다. 새 파일은 없다.

- **`src/hooks/useFabricCanvas.js`**: 캔버스에 오브젝트가 추가·이동·삭제될 때마다 저장 예약을 건다. 단, 매번 즉시 저장하면 마우스를 조금만 움직여도 수백 번 쓰기가 발생하므로, 마지막 변경 후 500ms가 지나야 실제로 저장하는 "디바운스"(debounce, 연속 입력을 모아 한 번만 처리하는 기법)를 적용했다.
- **`src/components/DiaryCanvas/DiaryCanvas.jsx`**: 저장 콜백(callback, 특정 시점에 호출해달라고 넘겨주는 함수)을 받아 훅으로 전달하도록 인터페이스를 추가했다.
- **`src/App.jsx`**: 실제 저장 로직을 구현했다. 현재 날짜·탭 정보를 알고 있는 유일한 곳이기 때문에 여기에 위치한다.

## 데이터가 어떻게 흐르나?

```
사용자가 스티커를 캔버스에 추가
        |
        v
Fabric.js "object:added" 이벤트 발생
        |
        v
500ms 대기 (그 사이 추가 이벤트가 오면 타이머 재시작)
        |
        v
canvas.toJSON() — 캔버스 전체 상태를 JSON 객체로 직렬화
        |
        v
App.jsx의 handleSaveCanvas(canvasJSON) 호출
        |
        v
loadAllDiaryData()  ← localStorage에서 전체 데이터 읽기
        |
        v
setDatePageData(전체, "2026-07-10", "diary", canvasJSON)
  → 해당 날짜·탭만 교체한 새 객체 반환 (원본 불변)
        |
        v
saveAllDiaryData(새 객체)  ← localStorage에 덮어쓰기
```

숫자 예시: 오전 10시에 스티커 3개를 1초 안에 연속으로 추가했다면 이벤트는 3번 발생하지만 저장은 마지막 스티커 추가 후 500ms가 지난 시점에 딱 1번만 실행된다.

## 쉽게 말하면

일종의 "자동 저장 워드"와 같다. 글을 쓰는 도중 매 글자마다 저장하지 않고, 타이핑이 잠깐 멈추면 조용히 저장된다. 다이어리에서는 스티커 배치가 멈추면 0.5초 후 저장된다. 사용자는 저장 버튼을 누를 필요가 없다.

## 만든 파일

수정:
- `src/hooks/useFabricCanvas.js` — `onSave` 파라미터 추가, 이벤트 구독 + 디바운스 로직
- `src/components/DiaryCanvas/DiaryCanvas.jsx` — `onSave` prop 추가
- `src/App.jsx` — `handleSaveCanvas` 구현, `DiaryCanvas`에 전달

## 제대로 됐는지 어떻게 확인했나?

**AC 1. npm run build 성공**
```
✓ built in 92ms  (479 kB)
```
에러 없이 통과.

**AC 2. 브라우저에서 새로고침 후 복원**
수동 검증 절차:
1. 브라우저에서 앱 실행
2. 캔버스에 이모지 스티커 추가, 드래그로 이동, 크기 조절
3. 0.5초 대기 (저장 완료)
4. F5 새로고침
5. 같은 날짜에 동일한 스티커가 동일한 위치에 복원됨을 확인
6. 다른 날짜(캘린더 클릭) → 빈 캔버스 → 원래 날짜 복귀 → 내용 유지 확인

초기 로드 시 `loadFromJSON`이 트리거하는 이벤트는 `isLoading` 플래그로 무시하여, 불러오는 행위가 저장을 다시 일으키지 않는다.

## 전체 그림에서 지금 어디?

```
[0] 저장소(storage)  완료
[1] 캔버스 껍데기     완료
[2] 스티커 팔레트     완료
[3] 이미지 업로드     완료
[4] 텍스트 메모       완료
[5] 캘린더 + 탭       완료
[6] 자동 저장  ◀ 지금 여기
[7] JSON/PNG 내보내기·불러오기  ← 다음
[8] 영화 리뷰(TMDB)   2순위
```

다음 단계(json-png-io)는 전체 다이어리 데이터를 JSON 파일로 내려받고(기기 이동용), 현재 캔버스를 PNG 이미지로 저장하는(공유용) 기능이다. 자동 저장이 완성된 지금, localStorage에 온전한 데이터가 쌓이기 때문에 JSON export가 의미 있는 상태가 됐다.

## 아직 안 된 것 (솔직하게)

- **localStorage 용량 초과**: 이미지를 수십 장 업로드하면 브라우저 localStorage 한도(보통 5~10 MB)를 초과할 수 있다. 현재 에러 처리가 없으며, PRD 범위 밖이라 의도적으로 구현하지 않았다.
- **영화리뷰 탭 저장**: 현재 영화리뷰 탭은 placeholder 상태라 저장 연결이 없다. 영화리뷰 기능 구현(2순위) 시 같은 패턴으로 연결하면 된다.
- **이미지 업로드 후 PNG 저장 시 CORS 제한**: 외부 URL 이미지를 캔버스에 올리면 PNG 내보내기가 실패할 수 있는데, 이는 다음 step에서 다룬다.
