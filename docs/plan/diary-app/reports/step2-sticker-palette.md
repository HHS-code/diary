# Step 2: 이모지 스티커 팔레트

## 이 단계를 왜 했나?

다이어리 꾸미기의 핵심은 이모지나 스티커를 원하는 자리에 붙이는 것이다. 앞 단계(step 1)에서 캔버스 도화지 자체는 만들었지만, 실제로 무언가를 올릴 수 있는 수단이 없었다. 이번 단계에서 "팔레트 → 클릭 → 캔버스에 추가" 흐름을 처음으로 완성한다.

## 무엇을 만들었나?

`StickerPalette`라는 새 컴포넌트(화면 조각)를 만들었다. 하트·별·꽃·동물·음식 등 30개의 이모지를 버튼 그리드(격자 배열)로 보여준다. 버튼을 클릭하면 지금 열려 있는 캔버스 위의 가운데 근처 랜덤 위치에 해당 이모지가 텍스트 오브젝트로 추가된다.

이모지는 이미지 파일이 아니라 유니코드 문자(운영체제가 기본 제공하는 이모지 글자)로 렌더링된다. 별도 이미지 파일을 관리할 필요 없이, 브라우저가 문자를 그리는 방식 그대로다.

기존 `DiaryCanvas`(캔버스 컴포넌트)에서 임시로 넣었던 "테스트용 네모 추가" 버튼을 제거하고, `StickerPalette`를 캔버스 왼쪽에 배치했다.

## 데이터가 어떻게 흐르나?

```
사용자
  │
  │ 이모지 버튼 클릭 (예: ❤️)
  ▼
StickerPalette
  │
  │ fabricCanvasRef.current에 접근
  │ (이 ref는 DiaryCanvas가 생성한 Fabric.js 캔버스 인스턴스를 가리킨다)
  │
  │ new FabricText("❤️", { left: 400 + 랜덤, top: 300 + 랜덤, fontSize: 48 })
  │ canvas.add(sticker)
  │ canvas.renderAll()
  ▼
캔버스 화면에 이모지 등장
```

**숫자 예시**: 캔버스는 800×600 픽셀. 중앙은 (400, 300). 클릭할 때마다 ±100px 범위에서 무작위로 위치가 달라지므로, 10번 클릭하면 10개의 이모지가 각각 다른 자리에 놓인다.

`fabricCanvasRef`는 React의 "레퍼런스(ref)" — 화면을 다시 그리지 않고 캔버스 인스턴스에 직접 접근하는 창구다.

## 쉽게 말하면

팔레트 버튼이 "스탬프"고, Fabric.js 캔버스가 "도장판"이다. 팔레트에서 원하는 이모지 스탬프를 골라 찍으면, 도장판 위 랜덤한 자리에 찍힌다. 찍힌 스탬프는 손가락(마우스)으로 자유롭게 옮기고, 크기를 키우거나 줄이고, 돌릴 수 있다.

## 만든 파일

```
src/
  components/
    StickerPalette/
      StickerPalette.jsx   (신규) 이모지 팔레트 컴포넌트
    DiaryCanvas/
      DiaryCanvas.jsx      (수정) 테스트 버튼 제거 + StickerPalette 통합
```

## 제대로 됐는지 어떻게 확인했나?

**AC 1 — 빌드 성공**
`npm run build` 실행 결과: 에러 없음, 472 kB 번들 생성 완료.

**AC 2 — 브라우저 동작** (빌드된 결과로 확인 가능)
- 이모지 팔레트가 캔버스 왼쪽에 표시된다.
- 이모지 버튼 클릭 → 캔버스에 해당 이모지가 추가된다.
- 같은 버튼을 여러 번 클릭하면 매번 다른 위치에 새 오브젝트가 추가된다.
- 추가된 이모지 오브젝트를 클릭하면 Fabric.js의 기본 컨트롤(이동·크기조절·회전 핸들)이 나타난다.

## 전체 그림에서 지금 어디?

```
[0] storage-layer    완료 — localStorage 저장/불러오기 기반
[1] fabric-canvas-shell 완료 — Fabric.js 캔버스 도화지
[2] sticker-palette  완료 ← 지금 여기
[3] image-upload     다음 — 사용자가 올린 사진/이미지를 캔버스에 추가
[4] text-memo-fonts  대기 — 손글씨 느낌 텍스트 메모
[5] calendar-tabs    대기 — 날짜 선택 캘린더 + 탭 전환
[6] autosave         대기 — 편집할 때마다 자동 저장
[7] json-png-io      대기 — JSON 내보내기/불러오기, PNG 저장
```

다음이 `image-upload`인 이유: 이모지(텍스트)는 됐으니, 이제 사용자가 직접 찍은 사진이나 이미지 파일을 캔버스에 올릴 수 있어야 다이어리 꾸미기가 실질적으로 완성된다. 기술 구조는 이번과 동일하게 `fabricCanvasRef`를 받아 캔버스에 오브젝트를 추가하는 방식이다.

## 아직 안 된 것 (솔직하게)

- **저장 없음**: 이모지를 추가해도 새로고침하면 사라진다. 저장 연결은 step 6(autosave)에서 한다.
- **날짜 없음**: 지금은 날짜 개념 없이 단일 캔버스만 있다. 날짜별 페이지는 step 5에서 만든다.
- **팔레트 스크롤 없음**: 30개 이모지가 한 번에 보이지 않으면 잘리는 레이아웃이다. 실용 수준이면 충분하지만, 이모지를 더 추가하려면 스크롤이나 탭 분류가 필요하다.
