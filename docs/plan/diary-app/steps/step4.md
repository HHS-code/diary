# Step 4: text-memo-fonts

## 읽을 파일
- `docs/plan/diary-app/PRD.md` — "손글씨 느낌 폰트의 텍스트 메모 추가"
- `index.html` (레포 루트) — 이미 연결된 8종 손글씨 구글 폰트 `<link>` 확인
- `src/components/DiaryCanvas/DiaryCanvas.jsx`

## 작업

텍스트 메모 추가 기능을 만든다.

- "텍스트 추가" 버튼 → 폰트 선택 UI(드롭다운 등)로 아래 8종 중 하나를 고르게 한다. `index.html`에 이미 연결된 CSS `font-family` 이름을 그대로 쓴다:
  - 한글: `Nanum Pen Script`, `Gaegu`, `Hi Melody`, `Poor Story`
  - 영문: `Caveat`, `Kalam`, `Gochi Hand`, `Shadows Into Light`
- 폰트를 고르고 나면 텍스트 입력을 받아(간단한 prompt든 인라인 입력창이든 재량) 그 폰트가 적용된 Fabric 텍스트 오브젝트를 캔버스에 추가한다.
- 추가된 텍스트 오브젝트는 Fabric 기본 컨트롤로 이동·크기조절·회전이 가능해야 하며, 더블클릭 등으로 내용 편집도 가능하면 좋다(Fabric의 `IText`/`Textbox` 편집 기능을 그대로 활용 — 별도 구현 불필요).
- 한글과 영문을 섞어 입력해도 정상적으로 렌더링되어야 한다 (폰트가 한글 미지원이어도 브라우저 기본 폴백 폰트로 표시되면 됨 — 별도 처리 불필요).

## AC

- `npm run build`가 에러 없이 성공한다.
- 브라우저에서: 텍스트 추가 버튼 → 폰트 하나 선택 → 텍스트 입력 → 캔버스에 해당 폰트로 텍스트가 나타난다. 8종 폰트 각각 선택해 추가했을 때 실제로 서로 다른 손글씨체로 렌더링됨을 눈으로 확인한다. 추가된 텍스트는 이동·크기조절·회전이 된다.

## 금지
- 새 폰트 파일이나 새 구글 폰트 `<link>`를 추가하지 마라. 이유: index.html에 이미 8종이 연결되어 있고 PRD/합의된 범위가 이 8종으로 확정됨.
- 리치 텍스트 서식(굵기, 색상 팔레트, 정렬 등) 기능을 만들지 마라. 이유: PRD에 "손글씨 느낌 폰트의 텍스트 메모"만 명시되어 있고 리치 에디터는 요청 범위 밖.
