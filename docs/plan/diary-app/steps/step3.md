# Step 3: image-upload

## 읽을 파일
- `docs/plan/diary-app/PRD.md` — "사용자 이미지 업로드", "사진 업로드"
- `src/components/DiaryCanvas/DiaryCanvas.jsx`, `src/hooks/useFabricCanvas.js`

## 작업

PRD에서 "이미지 업로드해서 자기만의 스티커로 추가"와 "사진 업로드해서 캔버스에 배치"는 사실상 같은 동작(파일 선택 → 캔버스에 이미지 오브젝트로 추가)이다. 이 step에서 하나의 업로드 기능으로 구현한다.

`DiaryCanvas.jsx` 또는 별도의 작은 컴포넌트(`ImageUploadButton.jsx` 등, 위치는 재량)에 파일 업로드 버튼(`<input type="file" accept="image/*">`)을 추가한다.

- 사용자가 이미지 파일을 선택하면, `FileReader`로 Data URL로 읽어서 Fabric 이미지 오브젝트로 캔버스에 추가한다.
- 추가된 이미지는 캔버스보다 크면 적당한 크기로 스케일 다운해 처음부터 캔버스 안에 보이게 한다 (전체가 안 보이거나 캔버스를 넘어가면 안 됨).
- 추가 후에는 Fabric 기본 컨트롤로 이동·크기조절·회전이 가능해야 한다.

## AC

- `npm run build`가 에러 없이 성공한다.
- 브라우저에서: 이미지 파일(jpg/png)을 선택하면 캔버스 안에 이미지가 나타나고, 이동·크기조절·회전이 가능하다. 원본이 큰 이미지(예: 2000px 이상)를 올려도 캔버스 안에서 시작 크기가 합리적으로 보인다.

## 금지
- 이미지를 서버나 외부 스토리지에 업로드하지 마라. 이유: PRD의 "서버/백엔드 없음" — 클라이언트에만 존재해야 함.
- 이미지 압축·리사이즈를 위해 별도 라이브러리를 설치하지 마라. 이유: Fabric의 스케일 조정만으로 "처음부터 캔버스 안에 보이게" 요구사항을 충족할 수 있다 — 불필요한 의존성 추가 금지.
