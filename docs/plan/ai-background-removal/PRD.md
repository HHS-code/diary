# PRD — AI 배경 제거 (ai-background-removal)

## 배경 / 왜

`sticker-studio`(완료)는 올가미(라소) 도구로 사용자가 직접 곡선을 그려 배경을 제거하는 "누끼따기"를 제공한다. 이 방식은 사각형에 가까운 단순한 피사체는 충분하지만, 머리카락·털처럼 정교한 경계는 마우스로 따라 그리기 어렵다.

브라우저에서 완전히 클라이언트 사이드로 동작하는 AI 세그멘테이션 모델을 이용하면, 서버나 API 키 없이도 훨씬 정확한 자동 배경 제거가 가능하다. 이 phase는 그 자동 제거 기능과, 결과를 다듬을 수 있는 보정 도구를 `sticker-studio`에 추가한다.

## 목표

- "AI 배경제거" 버튼 한 번으로 업로드/그려진 이미지의 배경을 자동으로 투명하게 만든다.
- 자동 제거 결과가 완벽하지 않을 때, 새 "AI 보정" 도구로 특정 영역을 다시 보이게(복원) 하거나 추가로 지울(삭제) 수 있다.
- 모델은 완전히 클라이언트에서 로드·실행되며(서버 없음, API 키 없음), MIT 라이선스 라이브러리만 사용한다.

## 대상 사용자

기존과 동일. 혼자 쓰는 개인 다이어리 사용자.

## 전제 조건

이 phase는 `sticker-studio`(완료)의 산출물에 의존한다: `StickerStudio.jsx`(캔버스 뼈대), `usePaintTools.js`(도구 추가 패턴), `stickerCutout.js`(clipPath 미리보기 → rasterize 패턴), `assetStorage.js`의 `sticker` 타입.

## In-scope

### 1. 자동 배경 제거

- 스티커 스튜디오 사이드바에 "AI 배경제거" 버튼을 추가한다.
- 클릭하면 현재 캔버스의 대상 이미지(업로드했거나 그린 이미지)에 대해 세그멘테이션 모델을 실행해, 배경으로 판단된 영역을 투명하게 만든다.
- 모델(`u2netp`, 약 4.7MB)은 이 버튼을 **처음 클릭했을 때만** 다운로드한다. 이후에는 브라우저 캐시에서 즉시 로드된다.
- 처리 중에는 버튼이 비활성화되고 "처리 중..." 텍스트/스피너가 표시된다. 다른 화면 조작은 막지 않는다.

### 2. AI 보정 도구

- `sticker-studio`의 기존 올가미(누끼따기, "안쪽만 남기고 바깥 전부 삭제")와는 별개의 새 도구다.
- 동작 순서: 사용자가 올가미로 보정하고 싶은 영역을 그린다 → "복원" 또는 "삭제" 버튼을 눌러 그 영역에 적용한다.
  - **복원**: 선택 영역 안에서, 자동 제거로 투명해진 부분을 다시 보이게(원본 픽셀 복구) 한다.
  - **삭제**: 선택 영역 안에서, 불투명하게 남아있는 부분을 투명하게 만든다.
- 이 과정을 여러 번 반복해(그리고→복원 또는 삭제) 원하는 결과가 나올 때까지 다듬을 수 있다.

## Out-of-scope (명시)

- 서버 API(remove.bg 등) 연동 — 완전히 로컬 모델만 사용한다.
- AGPL 등 카피레프트 라이선스 라이브러리 — MIT/Apache 라이선스만 허용한다(ADR-1).
- 더 정확하지만 무거운 모델(예: isnet, ~178MB) 선택 옵션 — 이번엔 경량 모델(`u2netp`) 고정.
- 인물 특화 모델(BodyPix, MediaPipe Selfie Segmentation 등) — 스티커는 인물 외 사물도 대상이라 범용 세그멘테이션 모델을 쓴다.
- 자동 제거 결과의 배치 처리(여러 이미지 한 번에 처리) — 한 번에 캔버스 위 이미지 하나만 대상으로 한다.
- `sticker-studio`의 기존 올가미(누끼따기) 도구 자체의 수정 — AI 보정은 완전히 별개의 새 도구다.

## 성공 기준

- "AI 배경제거" 버튼을 처음 누르면 모델이 다운로드되고(진행 중 표시), 완료되면 자동으로 배경이 제거된다.
- 같은 세션에서 다시 누르면 재다운로드 없이 즉시 처리된다.
- AI 보정 도구로 영역을 그린 뒤 "복원"을 누르면 그 영역이 다시 보이고, "삭제"를 누르면 그 영역이 투명해진다.
- 자동 제거 + AI 보정을 거친 결과도 기존 `sticker-studio`의 흰색 테두리 추가, 저장(assetStorage `sticker` 타입) 흐름에 그대로 이어진다.
- 이 프로젝트에 새로 추가되는 의존성(`@bunnio/rembg-web`, `onnxruntime-web`)이 MIT 라이선스임을 확인한다.
- 위 변경 전체에 걸쳐 `sticker-studio`의 기존 기능(그리기, 크롭, 올가미 누끼따기, 테두리, 저장/재사용)이 이전과 동일하게 동작한다.

## 진행 방식

1. 문서(PRD/ADR/architecture) 합의 → 실행 승인
2. 모델 로드/추론 래퍼 → "AI 배경제거" 버튼 연결 → AI 보정 도구(복원/삭제) → 완료 시 사용자에게 결과 설명
