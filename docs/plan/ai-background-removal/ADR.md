# ADR — AI 배경 제거 주요 결정

## ADR-1. 라이브러리는 @imgly/background-removal 대신 @bunnio/rembg-web(MIT) 사용

**맥락**: 완전 로컬 AI 배경 제거를 위해 가장 널리 알려진 패키지는 `@imgly/background-removal`이지만, 조사 결과 **AGPL 라이선스**임이 확인됐다. AGPL은 네트워크로 서비스되는 소프트웨어에도 소스 공개 의무를 발생시키는 강한 카피레프트 라이선스라, 이 다이어리 앱을 나중에 웹으로 배포할 때(로드맵의 "배포" 보류 항목) 앱 전체 소스 공개 의무가 걸릴 수 있다.

**선택지**:
- (A) `@bunnio/rembg-web`(MIT, onnxruntime-web 기반, 2025-10 최신 유지) 사용
- (B) `modern-rembg`(MIT, 동일 계열이나 2024-05 이후 업데이트 정지) 사용
- (C) U-2-Net(Apache-2.0) 모델을 직접 onnxruntime-web으로 통합(패키지 없이 직접 구현)
- (D) `@imgly/background-removal` 그대로 사용(AGPL 감수)

**결정**: (A) `@bunnio/rembg-web`.

**결과(트레이드오프)**: MIT 라이선스라 향후 배포 방식과 무관하게 소스 공개 의무가 없고, 가장 최근까지 유지보수되고 있어 안정성도 상대적으로 높다. `remove(file) -> Promise<Blob>` 형태의 단순한 API로 통합 난이도도 낮다. 대신 (C)만큼 세부 커스터마이징(전처리/후처리 직접 제어)은 어렵고, 패키지 자체가 향후 유지보수 중단될 가능성은 (D)의 IMG.LY(상용 지원 가능)보다 낮게 보장된다 — 이는 오픈소스 소규모 패키지의 일반적 리스크로 감수한다.

## ADR-2. 모델은 경량판(u2netp, ~4.7MB) 고정

**맥락**: `@bunnio/rembg-web`은 여러 모델(u2netp ~4.7MB, u2net/isnet ~170MB대 등)을 선택할 수 있다. 모델이 클수록 정확도는 높아지지만 첫 다운로드 부담이 커진다.

**선택지**:
- (A) `u2netp`(경량, ~4.7MB) 고정 사용
- (B) `isnet-general-use`(대형, ~178MB) 사용
- (C) 사용자가 선택하는 옵션 제공

**결정**: (A) `u2netp` 고정.

**결과(트레이드오프)**: 다운로드 부담이 작아(4.7MB) "버튼 첫 클릭 시 다운로드"라는 UX가 실용적으로 성립한다. 대신 정확도는 대형 모델보다 낮아, 복잡한 경계(가는 털 등)에서는 결과가 완벽하지 않을 수 있다 — 이 한계를 AI 보정 도구(복원/삭제)로 보완하는 것이 이번 phase의 설계 전제다. (C)는 이번 phase 범위에서는 불필요한 복잡도로 판단해 제외했다(PRD Out-of-scope).

## ADR-3. AI 보정은 기존 올가미(누끼따기)와 별개의 새 도구로 추가

**맥락**: `sticker-studio`의 기존 올가미 도구는 "그린 영역 안쪽만 남기고 바깥 전부를 삭제"하는 단일 동작(`stickerCutout.js`의 `previewLassoCutout`/`commitLassoCutout`)이다. AI 보정은 이미 부분적으로 투명해진 이미지 위에 "그린 영역에 대해 복원 또는 삭제를 선택 적용"하는, 동작 방식이 다른 기능이다.

**선택지**:
- (A) 완전히 별개의 새 도구("AI 보정")를 추가해, 기존 올가미 도구와 관련 로직(`stickerCutout.js`)을 전혀 건드리지 않음
- (B) 기존 올가미 도구를 확장해 모드(전체 컷아웃/부분 보정)를 전환하는 옵션을 추가

**결정**: (A) 별개의 새 도구.

**결과(트레이드오프)**: `sticker-studio`에서 이미 검증된 누끼따기 동작에 회귀 위험을 주지 않는다. 대신 캔버스에 유사해 보이는 올가미 그리기 도구가 두 개(누끼따기용, AI 보정용) 존재하게 되어, UI상 두 도구를 명확히 구분하는 라벨/아이콘이 필요하다.

## ADR-4. 모델은 "AI 배경제거" 버튼 첫 클릭 시 lazy 로드

**맥락**: 모델(4.7MB)을 언제 받을지 정해야 한다 — 스티커 스튜디오 진입 시 미리 받을지, 기능을 실제로 쓸 때만 받을지.

**선택지**:
- (A) "AI 배경제거" 버튼 첫 클릭 시에만 다운로드(lazy load)
- (B) 스티커 스튜디오 진입 시 미리 다운로드(preload)

**결정**: (A) lazy load.

**결과(트레이드오프)**: 스티커 스튜디오 진입 자체는 가볍게 유지된다 — 그리기/크롭/올가미 누끼따기만 쓰는 사용자는 이 기능의 존재로 인한 부담(다운로드, 초기화 시간)을 전혀 지지 않는다. 대신 AI 배경제거를 처음 쓰는 순간에는 모델 다운로드 시간(네트워크 상황에 따라 몇 초)만큼 대기가 발생한다 — 이 대기는 PRD의 "처리 중" UX로 사용자에게 명시적으로 보여준다.
