# Step 2: ai-correction-tool

## 이 단계를 왜 했나?

step 0에서 AI가 배경을 지우는 엔진을 만들었고, step 1에서 "AI 배경제거" 버튼으로 그 엔진을 실제로 쓸 수 있게 연결했다. 그런데 AI는 완벽하지 않다 — 머리카락 끝처럼 정교한 경계는 잘못 지우거나(있어야 할 부분이 사라짐) 덜 지운다(없어야 할 배경이 남음). 이번 단계는 그 실수를 사람이 직접 손볼 수 있는 도구, "AI 보정"을 추가하는 작업이다.

## 무엇을 만들었나?

그리기 도구 목록에 "AI 보정"이라는 새 도구를 하나 추가했다. 기존에 있던 "올가미"(전부 오려내는 도구)와는 선을 완전히 다르게 그어(주황색 점선 — 올가미는 파란색 점선) 헷갈리지 않게 했다.

사용 흐름은 이렇다:

1. "AI 보정" 도구를 선택하고, 손보고 싶은 부분을 마우스로 대충 둘러 그린다(동그라미든 네모든 상관없다).
2. 그린 선은 화면에 남지 않고 바로 사라진다 — 대신 "방금 그린 영역"으로 기억해둔다.
3. "복원" 버튼을 누르면, 그 영역 안에서 AI가 지워버린 부분이 원래 모습으로 되돌아온다.
4. "삭제" 버튼을 누르면, 그 영역 안에서 남아있는 부분이 마저 투명하게 지워진다.
5. 이 과정(그리기 → 복원 또는 삭제)을 원하는 만큼 반복해서 다듬을 수 있다.

"복원"은 "AI 배경제거"를 한 번도 안 써서 되돌릴 원본이 없으면 눌러지지 않는다(버튼이 흐리게 비활성화된다). "삭제"는 원본이 없어도 항상 쓸 수 있다 — 지금 보이는 그림만 있으면 되기 때문이다. 두 버튼 모두 영역을 아직 안 그렸으면 눌리지 않는다.

## 데이터가 어떻게 흐르나?

```
사용자가 "AI 보정" 도구로 영역을 그림
        │
        ▼
그린 선은 화면에서 즉시 지워지고, "방금 그린 영역"으로만 기억됨
        │
        ├── "복원" 버튼 클릭 ──────────────┐   "삭제" 버튼 클릭
        │                                   │        │
        ▼                                   ▼        ▼
  되돌릴 원본이 있는가?                 (원본 필요 없음)
        │ 없으면: 아무 일도 안 일어남        │
        │ 있으면:                           │
        ▼                                   ▼
  기억해둔 영역 안에서만,              기억해둔 영역 안에서만,
  "AI 배경제거 이전" 원본 그림을       지금 보이는 그림을
  다시 가져와 덮어씌움                투명하게 지움
        │                                   │
        └───────────────┬───────────────────┘
                         ▼
        캔버스의 그림을 그 결과로 바꿔치기(위치는 그대로)
                         │
                         ▼

```

숫자로 보면: 512×512 스티커 캔버스에서 사람 얼굴 사진의 귀 옆 머리카락이 AI 배경제거로 잘못 지워졌다고 하자. 사용자가 귀 주변을 20×20픽셀 정도 크기로 대충 둘러 그리고 "복원"을 누르면, 그 20×20 영역 안의 픽셀만 원본으로 되돌아오고 나머지 캔버스는 전혀 손대지 않는다 — 전체 이미지를 다시 계산하는 게 아니라 그린 영역만 정확히 바꾸는 방식이다.

## 쉽게 말하면

사진관 로봇(AI)이 오려낸 사진에 붓 하나를 쥐어준 것과 같다. 로봇이 실수로 지운 자리를 손가락으로 대충 동그라미 치고 "복원" 도장을 찍으면 원래 사진이 그 자리에만 되살아나고, 반대로 로봇이 덜 지운 자리를 동그라미 치고 "삭제" 도장을 찍으면 그 자리만 마저 지워진다. 동그라미는 도장을 찍는 순간 화면에서 사라지므로, 다음 실수를 고치려면 또 새로 동그라미를 쳐야 한다.

## 만든 파일

- `src/fabric/stickerAiCorrection.js` (신규) — "복원"(`restoreRegion`)과 "삭제"(`eraseRegion`) 두 동작의 실제 픽셀 계산을 담당. 화면(Fabric.js)과 무관하게 순수하게 그림 데이터만 다루는 파일이다.
- `src/fabric/stickerAiCorrection.test.js` (신규) — 영역 안쪽만 정확히 복원/삭제되고 바깥은 그대로인지, 크롭된 좌표계로도 정확히 정렬되는지, 너무 짧게(점 하나만) 그린 경우 안전하게 무시되는지 확인하는 자동 테스트.
- `src/hooks/usePaintTools.js` (수정) — "AI 보정" 도구 추가(주황 점선 붓, 그림 획으로 영구히 남지 않음).
- `src/hooks/usePaintTools.test.js` (수정) — 새 도구의 붓 스타일과 임시성(영구히 안 남는지) 확인 테스트 추가.
- `src/components/PaintToolbox/PaintToolbox.jsx` (수정) — 도구 목록에 "AI 보정" 버튼 추가.
- `src/components/PaintToolbox/PaintToolbox.test.jsx` (수정) — 도구 개수·목록이 바뀐 것을 반영해 기존 테스트 갱신.
- `src/components/StickerStudio/StickerStudio.jsx` (수정) — "복원"/"삭제" 버튼 추가, 영역 그리기 구독, step 1에서 만든 "되돌리기용 원본" 저장 방식을 살짝 보강(아래 "제대로 됐는지" 항목 참고).
- `src/components/StickerStudio/StickerStudio.test.jsx` (수정) — 원본 없을 때 "복원" 비활성화, 영역을 그리고 각 버튼을 눌렀을 때 그림이 실제로 바뀌고 버튼이 다시 잠기는지 확인하는 테스트 추가.

## 제대로 됐는지 어떻게 확인했나?

- `npm run lint`, `npm run build`, `npm run test`(테스트 파일 34개·테스트 209개 전부) 모두 통과 확인.
- 계획 문서(architecture.md)에 미리 적혀 있던 대로, "복원"/"삭제" 함수는 화면과 분리된 순수 계산 함수로 만들고, 실제 픽셀(투명한지/불투명한지)까지 확인하는 테스트로 검증했다 — 스텁으로 대충 넘기지 않고 진짜 그림 데이터로 확인했다.
- **계획과 다르게 결정한 부분(솔직하게 밝힘)**: 계획 문서는 `restoreRegion`/`eraseRegion` 함수가 그림 데이터 3~4개만 받는 것으로 가정했는데, 실제로 구현해보니 "사용자가 그린 영역"과 "손보려는 그림 조각"의 위치를 정확히 맞추려면 그 조각이 캔버스의 어느 위치(왼쪽에서 몇 픽셀, 위에서 몇 픽셀)에서 잘렸는지도 함께 알아야 했다. 그래서 함수에 그 위치 정보(left, top) 두 개를 추가로 넘기도록 바꿨다 — 이 조정이 없으면 사용자가 그린 영역과 실제로 지워지는/복원되는 위치가 어긋난다. (step 0에서도 실제 라이브러리를 설치해보니 계획 문서와 API가 달라 조정했던 것과 같은 종류의, "실제로 만들어보니 필요했던" 조정이다.)
- 또한 step 1에서 "되돌리기용 원본"을 그림 데이터 하나로만 저장했던 것을, 위치 정보(left, top)까지 함께 저장하도록 바꿨다 — 이 정보 없이는 "복원"이 정확한 위치를 찾을 수 없기 때문이다. 이 변경이 step 1에서 만든 다른 기능(AI 배경제거 버튼)에 영향을 주지 않는지 기존 테스트로 재확인했고, 전부 그대로 통과했다.

## 아직 안 된 것 (솔직하게)

- 실제 브라우저에서 사람이 마우스로 직접 그려보며 눈으로 확인하는 절차는 이번 단계에서 하지 않았다 — 자동 테스트로 픽셀 단위까지 확인했지만, 실제 손맛(그리기 느낌, 점선이 눈에 잘 띄는지 등)은 사람이 직접 스티커 스튜디오를 열어봐야 알 수 있다.
- 만약 "AI 배경제거"를 한 뒤에 그림을 마우스로 옮기거나 크기를 바꾼 다음 "AI 보정"을 쓰면, 복원 위치가 어긋날 수 있다 — 이 단계는 "배경제거 이후 그림을 움직이지 않는다"는 일반적인 사용 흐름을 전제로 만들었고, 그림을 옮기는 특수한 경우까지는 다루지 않았다.
- PRD에 적힌 진행 방식은 이번 단계로 3단계가 모두 끝났지만, "완성" 버튼을 눌러 저장한 뒤에도 여전히 잘 동작하는지, 저장된 스티커를 다시 불러와 이어서 보정할 수 있는지는 이번 단계에서 새로 확인하지 않았다(이 흐름 자체는 step 0 이전부터 있던 기존 기능이라 별도 변경이 없었다).
