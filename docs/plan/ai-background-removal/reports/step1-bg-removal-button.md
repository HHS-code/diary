# Step 1: bg-removal-button

## 이 단계를 왜 했나?

이전 단계(step 0)에서 "AI 배경제거 엔진"(`removeBackgroundFromImage` 함수)을 만들었지만, 화면에는 아무것도 보이지 않는 상태였다. 이번 단계는 그 엔진을 실제로 사용자가 누를 수 있는 버튼으로 스티커 스튜디오 화면에 연결하는 작업이다.

## 무엇을 만들었나?

스티커 스튜디오 사이드바에 "AI 배경제거" 버튼을 추가했다. 버튼을 누르면:

1. 지금 캔버스 위에 있는 이미지(스티커로 만들고 있는 그림/사진)를 대상으로 고른다.
2. 지우기 전 원본 모습을 따로 기억해둔다 — 나중에 "이 부분은 지우지 말았어야 했는데" 싶을 때 되돌릴 수 있게 하기 위한 준비다(다음 단계인 "AI 보정"에서 쓰인다).
3. 버튼이 잠시 눌리지 않게 바뀌고, "처리 중..." 표시가 뜬다.
4. AI 엔진이 배경을 지운 결과를 받아 캔버스의 그림을 그걸로 바꿔치기한다.
5. 끝나면 버튼이 다시 눌릴 수 있는 상태로 돌아온다.
6. 만약 중간에 문제가 생기면("실패"라고 잠깐 표시된 뒤) 버튼은 다시 정상적으로 눌릴 수 있는 상태로 복구된다 — 화면이 멈추거나 먹통이 되지 않는다.

## 데이터가 어떻게 흐르나?

```
사용자가 "AI 배경제거" 버튼 클릭
        │
        ▼
지금 캔버스에 있는 그림을 골라(이미지가 하나뿐이면 그걸, 여러 개면
지금 선택된 것) 그 부분만 잘라낸 그림 데이터를 만듦
        │
        ├─ 이 잘라낸 원본을 "되돌리기용"으로 따로 보관
        │
        ▼
버튼 비활성화 + "처리 중..." 표시
        │
        ▼
step 0에서 만든 엔진(removeBackgroundFromImage) 호출
        │
        ▼
배경이 지워진 새 그림을 받음
        │
        ▼
캔버스의 원래 그림 자리에 새 그림으로 교체(위치는 그대로 유지)
        │
        ▼
버튼 다시 활성화
```

숫자로 보면: 512×512 크기의 스티커 캔버스에서 사람 사진 하나를 배경 제거하면, step0에서 확인한 것처럼 첫 처리는 모델 다운로드까지 포함해 몇 초, 이후엔 AI 판단 시간만큼(대략 3초 안팎, 컴퓨터 성능에 따라 다름) 걸린다. 이 대기 시간 동안 버튼에는 "처리 중..." 글자가 보인다.

## 쉽게 말하면

사진관 로봇(step0에서 설치한 AI 엔진) 앞에 드디어 "배경 지우기" 버튼을 달았다. 버튼을 누르면 로봇이 일하는 동안 버튼에 불이 들어와("처리 중") 다시 누를 수 없게 되고, 로봇이 사진을 다 오려내면 그 결과로 화면 속 사진이 바뀐다. 혹시 로봇이 고장 나면("실패") 버튼은 잠깐 그렇게 표시됐다가 다시 정상으로 돌아와, 사용자가 다시 시도할 수 있다.

## 만든 파일

- `src/components/StickerStudio/StickerStudio.jsx` — "AI 배경제거" 버튼과 그 동작 로직 추가.
- `src/components/StickerStudio/StickerStudio.test.jsx` — 버튼 클릭 시 비활성화되는지, 성공하면 그림이 바뀌고 버튼이 되돌아오는지, 실패해도 버튼이 되돌아오는지 확인하는 자동 테스트 추가.

## 제대로 됐는지 어떻게 확인했나?

- `npm run lint`, `npm run test` 모두 통과 확인(스티커 스튜디오 관련 테스트 12개 포함 전부 통과).
- 코드를 직접 검토해 명세대로 구현됐는지 확인: 대상 그림을 고르는 방식, 원본을 되돌리기용으로 보관하는 방식, 버튼 상태 관리, 실패 시 복구 로직 모두 계획대로 되어 있음을 확인했다.
- 처리 중에는 버튼만 잠기고 화면 전체가 잠기지 않는다는 것도 코드로 확인했다(요구사항).
- 배경 제거 결과를 스티커로 곧바로 저장하지는 않는다는 것도 확인했다 — 저장은 "완성" 버튼을 눌렀을 때만 일어나는 별개의 동작이다.

**참고**: 이 단계는 작업을 담당한 프로그램(AI 세션)이 실제 구현을 코드로 완성해 저장(커밋)까지는 마쳤지만, 작업 완료 기록을 남기는 마지막 절차 도중 자원(토큰) 문제로 멈췄다. 그래서 사람이 직접 코드와 테스트 결과를 다시 확인해 이 보고서를 작성했다 — 실제 코드 품질이나 동작에는 문제가 없음을 재확인했다.

## 전체 그림에서 지금 어디?

```
[step 0: 엔진 만들기] (완료)
        │
        ▼
[step 1: "AI 배경제거" 버튼 연결] ← 지금 여기 (완료)
        │
        ▼
[step 2: AI 보정 도구(복원/삭제)]
        │  자동으로 지운 결과가 완벽하지 않을 때, 사람이 영역을 그려
        │  다시 보이게 하거나 더 지우는 도구. 이 단계에서 보관해둔
        │  "지우기 전 원본"을 그대로 사용한다.
```

## 아직 안 된 것 (솔직하게)

- 자동으로 배경이 지워진 결과가 완벽하지 않을 때(머리카락 등 섬세한 부분) 다시 손보는 기능은 아직 없다 — 다음 단계(step 2)의 일이다.
- 실제 사람 사진, 복잡한 배경의 사진으로 브라우저에서 직접 눈으로 확인하는 절차는 이번 단계에서 다시 하지 않았다(step0에서 이미 실제 추론 성공을 확인했고, 이번 단계는 그 결과를 화면에 연결하는 배선 작업이라 코드/테스트 검증으로 충분하다고 판단했다).
