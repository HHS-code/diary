# Step 0: youtube-url-detection

## 이 단계를 왜 했나?

유튜브 카드 기능의 첫걸음은 "사용자가 붙여넣은 텍스트가 유튜브 링크인지, 맞다면 어떤 영상인지"를 알아내는 일이다. 이 판단이 틀리면 뒤에 이어지는 모든 단계(썸네일 카드 생성, 재생, 저장)가 잘못된 영상을 가리키게 된다. 그래서 이 판단만 따로 떼어내 가장 먼저 만들고, 확실히 검증해두기로 했다.

이번 단계는 그 판단 로직 하나만 만든다. 화면에 보이는 카드나 재생 기능은 아직 없다.

## 무엇을 만들었나?

`extractYoutubeVideoId`라는 함수 하나를 만들었다. 텍스트를 하나 넣으면:

- 유튜브 링크면 그 영상의 고유 ID(videoId, 유튜브 링크에 있는 11자리 코드)를 돌려준다.
- 유튜브 링크가 아니거나 그냥 텍스트면 "해당 없음"(null, 프로그래밍에서 "값이 없다"는 뜻)을 돌려준다.
- 잘못된 입력(빈 문자열, 값이 아예 없는 경우)이 들어와도 프로그램이 멈추거나 에러를 내지 않고 조용히 "해당 없음"을 돌려준다.

지원하는 유튜브 링크 형태는 세 가지다.

1. 일반 재생 링크: `youtube.com/watch?v=영상ID`
2. 단축 링크: `youtu.be/영상ID`
3. 쇼츠 링크: `youtube.com/shorts/영상ID`

각 형태에 `www.`가 붙거나 `m.`(모바일용)이 붙거나, `http`/`https`든 상관없이 인식한다. 또한 링크 뒤에 재생 시작 시각 같은 부가 정보(`&t=30s`)가 덧붙어도 영상 ID만 정확히 골라낸다.

## 데이터가 어떻게 흐르나?

```
사용자가 붙여넣은 텍스트
        |
        v
extractYoutubeVideoId(text)
        |
   +----+----------------------+
   |                           |
유튜브 링크 패턴과 일치        일치하는 패턴 없음
   |                           |
   v                           v
영상 ID 문자열 반환          null 반환
```

숫자로 보면:

- 입력: `"https://www.youtube.com/watch?v=dQw4w9WgXcQ&t=30s"`
  출력: `"dQw4w9WgXcQ"` (부가 정보 `&t=30s`는 버려지고 ID만 남음)
- 입력: `"오늘 점심 뭐 먹지"`
  출력: `null`
- 입력: `"https://vimeo.com/12345678"` (유튜브가 아닌 다른 영상 사이트)
  출력: `null`

## 쉽게 말하면

우체국 직원이 편지 봉투를 보고 "이건 우리 회사로 온 소포다" 혹은 "이건 우리 게 아니다"만 판별하는 단계라고 생각하면 된다. 소포를 열어서 내용물을 확인하거나(= 썸네일 카드를 만드는 일), 배송하는 일(= 재생하는 일)은 다음 단계의 몫이다. 이번 단계는 오직 "이 주소가 유튜브 것이 맞는지, 맞다면 영상 번호가 뭔지"만 정확히 가려내는 일을 한다.

## 만든 파일

- `src/fabric/youtubeUrl.js` — 판별 함수 본체
- `src/fabric/youtubeUrl.test.js` — 검증 코드(테스트, 함수가 의도대로 동작하는지 자동으로 확인하는 코드)

## 제대로 됐는지 어떻게 확인했나?

테스트 코드로 아래 경우들을 모두 확인했고, 전부 통과했다(총 221개 테스트 중 이번 단계가 추가한 12개 포함, 전부 성공).

- 일반 재생 링크: 기본 형태 + 부가 정보 붙은 형태
- `http`, `youtube.com`, `m.youtube.com` 변형 형태
- 단축 링크: 기본 형태 + 부가 정보 붙은 형태
- 쇼츠 링크: 기본 형태 + 부가 정보 붙은 형태
- 유튜브가 아닌 링크 → "해당 없음"
- 일반 텍스트 → "해당 없음"
- 빈 문자열 → "해당 없음"
- 값이 아예 없는 경우(`null`, `undefined`) → "해당 없음"

코드 스타일 검사(lint)도 이상 없이 통과했다.

## 전체 그림에서 지금 어디?

```
[0] 유튜브 링크 판별  <- 지금 여기
[1] 카드 오브젝트 만들기 (썸네일 + 재생버튼 모양)
[2] 붙여넣기와 연결하기 (Ctrl+V 하면 카드가 캔버스에 생김)
[3] 저장/불러오기 (파일로 저장해도 카드가 남아있음)
[4] 실제 재생 (카드 클릭하면 영상이 뜸)
```

다음 단계인 [1]은 이번에 만든 판별 함수가 돌려주는 영상 ID를 가지고, 실제로 화면에 그려질 썸네일 카드 모양을 만드는 일이다. 판별 로직이 먼저 확실해야 그 위에 카드를 올릴 수 있어서 이 순서로 진행했다.

## 아직 안 된 것 (솔직하게)

- 화면에 보이는 카드, 썸네일 이미지, 재생 버튼은 전혀 없다. 지금은 텍스트만 보고 "유튜브 링크 맞다/아니다"를 판단하는 로직뿐이다.
- 실제로 Ctrl+V 붙여넣기를 했을 때 이 함수가 호출되도록 연결하는 작업도 아직이다(다음다음 단계 몫).
- 링크 형태 판별은 정규식(문자열 패턴 매칭 규칙) 기반이라, 이번에 합의한 세 가지 형태 외의 아주 특이한 유튜브 링크 변형(예: 임베드 전용 링크 `youtube.com/embed/영상ID`)은 이번 범위에 없어서 인식하지 못한다. PRD에 명시된 세 가지 패턴만 지원한다.
