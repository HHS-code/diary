# Step 1: youtube-card-object

## 이 단계를 왜 했나?

지난 단계([0] youtube-url-detection)에서 "이 텍스트가 유튜브 링크인지, 맞다면 영상 ID가 뭔지"를 알아내는 판별 함수를 만들었다. 하지만 그 함수는 문자열 하나만 돌려줄 뿐, 캔버스 위에 실제로 보이는 그림은 아직 없다.

이번 단계는 그 영상 ID를 받아서 "캔버스 위에 올라갈 수 있는 실제 카드 모양"을 만드는 일이다. 캔버스는 이 다이어리 앱이 그림·스티커·GIF 같은 것들을 그리는 화면 영역을 담당하는 라이브러리(Fabric.js)로 동작하는데, 그 라이브러리가 이해할 수 있는 형태로 "유튜브 카드"라는 새로운 종류의 그림 조각을 정의해야 한다. 아직 붙여넣기(Ctrl+V)와 연결하지도, 실제 재생 기능을 붙이지도 않는다 — 오직 "이 모양의 카드를 어떻게 만들고, 어떻게 저장하고, 어떻게 다시 복원하는지"만 이번 단계의 몫이다.

## 무엇을 만들었나?

`YoutubeCard`라는 새로운 카드 종류(클래스)를 만들었다. 이 카드는 이 앱이 이미 갖고 있던 "이미지" 조각(기존 GIF 카드와 같은 기반)을 확장한 것이라, 이동·크기조절·삭제 같은 기존 캔버스 기능이 별도 작업 없이 그대로 적용된다.

만든 것을 요약하면:

- **영상 ID로 카드 만들기**: `YoutubeCard.create(영상ID)`를 호출하면, 유튜브가 제공하는 표준 썸네일 이미지 주소(`https://img.youtube.com/vi/영상ID/hqdefault.jpg`)에서 이미지를 내려받아 카드를 만든다. 이미지를 내려받는 데는 시간이 걸리므로, "다 되면 알려줄게"라는 뜻의 Promise(자바스크립트에서 나중에 끝나는 작업의 결과를 기다리게 해주는 객체) 형태로 결과를 돌려준다.
- **재생 버튼 그리기**: 썸네일 이미지 위에 반투명 검은 원과 흰색 삼각형(▶ 모양)을 겹쳐 그린다. 별도 그림 조각을 붙이는 대신, 카드 자신을 그리는 코드 안에서 이미지를 그린 직후 곧바로 원과 삼각형을 그려 넣는 방식이라 항상 함께 붙어 다닌다.
- **비율 고정**: 사용자가 카드 크기를 조절할 수는 있지만, 가로세로 비율(16:9, 유튜브 표준 화면비)이 어긋나지 않도록 잠갔다.
- **저장은 영상 ID 하나만**: 카드를 파일로 저장할 때 실제로 저장되는 정보는 영상 ID 하나뿐이다. 썸네일 이미지 주소나 "지금 재생 중인지" 같은 정보는 저장하지 않는다 — 어차피 영상 ID만 있으면 썸네일 주소는 언제든 같은 규칙으로 다시 만들어낼 수 있기 때문이다(파일 용량을 줄이는 효과도 있음).
- **불러오기**: 저장된 영상 ID를 다시 읽으면, 그 ID로 썸네일 주소를 재조립해서 카드를 원래 모습대로 복원한다.

## 데이터가 어떻게 흐르나?

```
영상 ID ("dQw4w9WgXcQ")
   |
   v
YoutubeCard.create(영상ID)
   |
   +-- 유튜브 썸네일 이미지 주소 조립
   |   https://img.youtube.com/vi/dQw4w9WgXcQ/hqdefault.jpg
   |
   +-- 이미지 다운로드 (시간이 걸림 → Promise로 기다림)
   |
   v
카드 완성 (이미지 + 재생버튼 그림)
   |
   v  (사용자가 저장하면)
파일에는 영상 ID만 남음: { videoId: "dQw4w9WgXcQ" }
   |
   v  (사용자가 다시 불러오면)
YoutubeCard.fromObject({ videoId: "dQw4w9WgXcQ" })
   |
   +-- 같은 규칙으로 썸네일 주소 재조립
   |
   v
카드 복원 완료 (이미지 + 재생버튼 다시 그려짐)
```

숫자로 보면: 영상 ID `dQw4w9WgXcQ`가 있으면, 저장 파일에는 딱 그 11글자만 남는다. 반면 썸네일 이미지 주소 전체(`https://img.youtube.com/vi/dQw4w9WgXcQ/hqdefault.jpg`, 약 50글자)나 카드 크기·위치 같은 다른 정보는 이미 Fabric.js가 공통으로 저장하는 값들이라 별도로 신경 쓸 필요가 없었다.

## 쉽게 말하면

이번 단계는 "즉석사진 카드 틀"을 만드는 일이라고 생각하면 된다. 사진(썸네일)을 찍어 넣고 그 위에 재생 버튼 스티커를 붙인 카드 한 장을 만드는 방법을 정의했을 뿐, 이 카드를 앨범(캔버스)의 어디에 꽂을지, 사용자가 뭘 눌러야 이 카드가 생기는지는 아직 정하지 않았다. 카드에 적는 내용도 최소한으로 줄였다 — "이 영상은 몇 번" 딱 한 줄만 적어두고, 사진이나 다른 세부사항은 그 번호만 있으면 언제든 다시 뽑아올 수 있게 해뒀다.

## 만든 파일

- `src/fabric/YoutubeCard.js` — 카드 클래스 본체
- `src/fabric/YoutubeCard.test.js` — 검증 코드(테스트)

## 제대로 됐는지 어떻게 확인했나?

테스트 코드로 아래 항목을 모두 확인했고, 전부 통과했다(전체 226개 테스트 전부 성공, 코드 스타일 검사(lint)도 이상 없음).

- 영상 ID로 카드를 만들면, 카드 종류가 정확히 "YoutubeCard"로 표시되고 이미지 주소에 그 영상 ID가 들어있는지.
- 카드를 저장 형태로 바꿨을 때(`toObject`) 영상 ID만 담기고, 썸네일 주소 전체나 재생 상태 같은 다른 정보는 담기지 않는지.
- 캔버스 라이브러리가 "YoutubeCard"라는 이름의 카드 종류를 제대로 알아보는지(카드 등록 확인).
- 저장된 영상 ID만 가지고 다시 만들었을 때(`fromObject`), 영상 ID와 썸네일 주소, 그리고 위치 값(left/top)까지 원래대로 복원되는지.
- 크기를 조절해도 16:9 비율이 깨지지 않도록 설정(`lockUniScaling`)이 켜져 있는지.

## 전체 그림에서 지금 어디?

```
[0] 유튜브 링크 판별
[1] 카드 오브젝트 만들기 (썸네일 + 재생버튼 모양)  <- 지금 여기
[2] 붙여넣기와 연결하기 (Ctrl+V 하면 카드가 캔버스에 생김)
[3] 저장/불러오기 (파일로 저장해도 카드가 남아있음)
[4] 실제 재생 (카드 클릭하면 영상이 뜸)
```

다음 단계인 [2]는 이번에 만든 `YoutubeCard.create(영상ID)`를 실제로 Ctrl+V 붙여넣기 동작과 연결해, 사용자가 유튜브 링크를 붙여넣으면 이 카드가 캔버스 위 화면에 실제로 나타나게 만드는 일이다. 카드를 "만드는 방법"이 먼저 확실해야 그걸 "언제 만들지"를 연결할 수 있어서 이 순서로 진행했다.

## 아직 안 된 것 (솔직하게)

- Ctrl+V로 붙여넣었을 때 이 카드가 실제로 캔버스에 나타나는 연결 작업은 아직이다(다음 단계 [2]의 몫).
- 카드를 클릭했을 때 실제로 영상이 재생되는 기능은 없다. 지금 만든 카드는 항상 멈춰있는 썸네일 그림일 뿐이다(재생은 [4]의 몫).
- 파일로 저장한 뒤 실제로 다시 불러왔을 때 이 앱 화면 안에서 정상 작동하는지는 이번 단계에서 다루지 않았다 — 테스트 코드로 "복원 로직 자체"는 확인했지만, 저장/불러오기 버튼과 실제로 연결되는 것은 [3] 단계다.
- 이번에 그린 재생 버튼(원+삼각형)은 크기·색 비율을 임의로 정한 것이라, 실제 화면에서 보기 좋은지는 다음 단계에서 화면에 카드가 나타난 뒤 눈으로 확인이 필요하다.
