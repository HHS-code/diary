# PRD — 다이어리 에디터/메인화면 UX 정비

## 배경 / 왜

`canvas-object-toolbar` 완료 후 실제 화면을 써보니 UX 관련 불만이 다수 나왔다. 개별적으로 하나씩 고치면 왕복이 계속 반복되므로, 이번에 전부 모아 한 번에 계획하고 한 번에 실행한다.

## 목표

- 다이어리 편집 화면(창·캔버스)이 실제 창 크기를 꽉 채우고, 배경이 있으며 사용자가 커스텀할 수 있다.
- 오브젝트 조작 툴바가 고정된 위치에서 항상 접근 가능하다.
- 주요 버튼들이 텍스트 대신 아이콘으로 표시된다.
- 작성된 다이어리를 썸네일 그리드로 모아볼 수 있다.
- 메인 화면(캘린더+위젯)이 여백 없이 꽉 차 보이고, 날짜 호버 시 바로 작성 화면으로 진입할 수 있다.

## 대상 사용자

기존과 동일. 혼자 쓰는 개인 다이어리 사용자.

## 원칙 — UI와 기능의 분리 (모든 항목에 적용)

**UI(레이아웃·색상·아이콘·위치)를 바꾸는 작업이 기존 기능(캔버스 저장/불러오기, 오브젝트 복사·삭제·레이어 순서, JSON/PNG export, 오토세이브)의 동작을 변경해서는 안 된다.** 아래 각 in-scope 항목은 "무엇을 UI로, 무엇을 로직으로 다루는지"를 architecture에서 구분해 명시한다. 기존 로직 함수(`useObjectActions.js`, `storage/diaryStorage.js`, `useFabricCanvas.js`의 저장 이벤트 처리 등)의 시그니처와 동작은 이번 범위에서 변경하지 않는다 — 그 함수들을 사용하는 방식(누가 호출하는지, 어떤 UI에서 보여주는지)만 바뀐다.

## In-scope

### 1. 에디터 레이아웃/캔버스 확대

- `Window.jsx`의 `windowBoxStyle`을 현재 90%/88%에서 더 큰 비율(95~98% 수준)로 키운다.
- `DiaryCanvas.jsx`의 캔버스 크기(현재 800×600 고정)를 창 크기에 맞춰 반응형으로 늘어나게 한다. 단, 이미 저장된 캔버스 JSON의 오브젝트 좌표가 깨지지 않아야 한다.
- 캔버스와 좌측 도구 사이드바(StickerPalette, ImageUploadButton, TextMemoButton)를 포함한 편집 화면 전체가 창을 중앙 정렬로 꽉 채운다 — 여백(공백)이 남지 않도록 한다.
- 편집 화면이 스크롤로 잘려 보이는 문제를 해결한다 (창 높이 안에 온전히 들어오도록 레이아웃 조정).
- 캔버스 기본 배경은 흰색 단색으로 지정한다.
- 사용자가 캔버스 배경을 커스텀할 수 있다: 색상 선택(컬러피커) 또는 이미지 업로드 중 선택 가능.

### 2. ObjectToolbar 고정 메뉴 재설계

- 기존 플로팅 방식(오브젝트 위치를 따라다니는 방식)을 폐기하고, 화면의 고정된 위치(예: 캔버스 상단 또는 사이드)에 항상 표시되는 메뉴로 재설계한다.
- 오브젝트가 선택되지 않았을 때는 버튼이 비활성화(disabled) 상태로 보인다.
- 기존 액션 로직(`useObjectActions.js`의 복사/삭제/레이어 순서 함수)은 변경하지 않고 그대로 재사용한다.

### 3. 아이콘 적용

- `react-icons` 라이브러리를 설치한다.
- 다음 버튼들의 텍스트 라벨을 아이콘으로 교체(또는 아이콘+텍스트 병기)한다: ObjectToolbar(복사/삭제/레이어 4종), ImageUploadButton, TextMemoButton, ExportImportControls(JSON 내보내기/불러오기, PNG 내보내기).
- 각 버튼의 `onClick` 핸들러와 로직은 변경하지 않는다.

### 4. 다이어리 갤러리

- 메인 화면(또는 메인 화면에서 진입 가능한 위치)에 작성된 다이어리들을 캔버스 썸네일 그리드로 보여주는 화면을 추가한다.
- 썸네일 클릭 시 해당 날짜의 편집 화면으로 이동한다.
- 기존 `storage/diaryStorage.js`의 저장 방식은 변경하지 않는다 — 갤러리는 저장된 데이터를 읽기 전용으로 순회해 썸네일을 생성한다.

### 5. 날짜 호버 시 작성 버튼

- XPCalendar의 날짜 셀에 마우스를 올리면 "작성" 버튼이 호버링으로 나타난다.
- 버튼 클릭 시 기존과 동일하게 해당 날짜의 편집 화면으로 진입한다 (날짜 셀 자체를 클릭하는 기존 동작은 유지).

### 6. 메인화면 여백 축소 + 날씨 로딩 스피너

- 메인 화면(XPCalendar + 위젯 3종)의 grid gap·padding을 줄이고, 위젯 내부 폰트/아이콘 크기를 키워 창을 꽉 채운 느낌을 준다.
- `WeatherWidget.jsx`의 로딩 상태(`status === 'loading'`) 표시를 텍스트에서 회전 스피너 애니메이션으로 교체한다. 로딩/에러/완료 상태 전환 로직 자체는 변경하지 않는다.

## Out-of-scope (명시)

- 캔버스 내부 오브젝트 조작 로직(복사/삭제/레이어 순서/키보드 단축키)의 동작 변경 — UI 위치·표현만 바뀜
- 정렬/등간격 배치 기능 재도입 (`canvas-object-toolbar`에서 이미 제외 결정됨)
- 갤러리에서 다이어리 삭제/일괄 관리 기능 — 보기와 이동만
- 배경 이미지에 대한 필터/편집 기능 — 업로드해서 그대로 쓰는 것만
- `free-drawing`, `asset-import-pipeline` 등 로드맵의 다른 phase 범위 — 이번 phase에 포함하지 않음
- 영화 리뷰 탭 UI — 이번 범위는 다이어리 탭 및 메인 화면만

## 성공 기준

- 다이어리 창을 열면 창 크기가 거의 전체 화면을 차지하고, 그 안의 캔버스·도구·여백이 창을 꽉 채워 빈 공백이 두드러지지 않는다.
- 편집 화면에서 스크롤 없이 캔버스와 도구들이 온전히 보인다.
- 캔버스 배경이 기본적으로 흰색이며, 사용자가 색상 또는 이미지로 배경을 바꿀 수 있고 PNG로 내보내면 그 배경이 그대로 반영된다.
- 오브젝트를 선택하고 드래그해도 조작 메뉴(고정 위치)가 계속 같은 자리에서 사용 가능하다.
- ObjectToolbar, 이미지/텍스트 추가, JSON/PNG 관련 버튼들이 아이콘으로 표시된다.
- 메인 화면(또는 별도 진입점)에서 작성된 다이어리들을 썸네일로 모아볼 수 있고, 클릭하면 해당 날짜로 이동한다.
- 캘린더 날짜에 마우스를 올리면 작성 버튼이 나타나고 클릭하면 편집 화면으로 이동한다.
- 메인 화면의 여백이 줄고 위젯 글자/아이콘이 커져 화면이 꽉 차 보인다.
- 날씨 위젯 로딩 중 스피너가 보인다.
- 위 변경 전체에 걸쳐 기존 캔버스 저장/불러오기, JSON export/import, PNG export, 오토세이브, 오브젝트 복사/삭제/레이어 순서 기능이 이전과 동일하게 동작한다.

## 진행 방식

1. 문서(PRD/architecture) 합의 → 실행 승인
2. 레이아웃/캔버스 확대 → 배경(기본+커스텀) → ObjectToolbar 고정 메뉴화 → 아이콘 적용 → 갤러리 → 날짜 호버 버튼 → 메인화면 여백/스피너 → 완료 시 사용자에게 결과 설명
