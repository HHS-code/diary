# Step 1: 캔버스 배경 커스텀 (색상 / 이미지)

## 이 단계를 왜 했나?

Step 0에서 다이어리 편집 캔버스가 1400×900으로 커지고 기본 배경이 흰색이 됐다. 하지만 배경을 바꿀 방법이 없었다 — 흰 종이에만 일기를 쓸 수 있는 셈이다. 이번 단계는 사용자가 배경을 원하는 색으로 칠하거나, 사진을 배경으로 깔 수 있게 하는 것이다.

중요한 제약이 하나 있었다: 배경을 바꿨을 때 그게 **자동으로 저장**되어야 한다. 스티커를 붙이면 0.5초 뒤 자동 저장되는 기존 구조가 이미 있는데, 배경 변경도 똑같이 저장되지 않으면 "배경을 예쁘게 바꿨는데 다시 열어보니 흰색"이 되어버린다.

## 무엇을 만들었나?

두 개의 새 파일과 한 곳의 수정이다.

1. **배경 바꾸기 로직** (`useCanvasBackground.js`, 신규)
   - "색으로 바꾸기": 캔버스의 배경색을 지정한 색(예: `#ffcc00` 노란색)으로 칠하고 화면을 다시 그린다.
   - "이미지로 바꾸기": 사용자가 고른 사진 파일을 읽어서, 캔버스 크기(1400×900)에 꽉 차도록 가로·세로 배율을 맞춘 뒤 배경으로 깐다. 예를 들어 700×450짜리 사진이면 가로 2배, 세로 2배로 늘려서 캔버스를 채운다.
   - 두 동작 모두 끝나자마자 캔버스에 "오브젝트가 수정됐다"는 신호를 쏜다. 기존 자동 저장 장치가 이 신호를 듣고 있기 때문에, 배경 변경도 스티커 추가와 똑같이 0.5초 뒤 저장된다. 기존 저장 코드는 한 줄도 고치지 않았다.

2. **배경 컨트롤 패널** (`CanvasBackgroundControl.jsx`, 신규)
   - 편집 화면 왼쪽 사이드바, 스티커 팔레트 바로 아래에 붙는 "배경"이라는 작은 패널.
   - 안에는 두 개가 있다: 색을 고르는 컬러피커(클릭하면 색상판이 뜨는 네모)와 "배경 이미지" 버튼(클릭하면 파일 선택창이 뜸).
   - 이 패널은 그림 그리는 로직을 전혀 모른다. 위의 로직 파일이 준 함수 두 개를 버튼에 연결만 한다. (UI와 로직을 분리하라는 이 프로젝트의 원칙 그대로.)

3. **편집 화면에 배치** (`DiaryCanvas.jsx`, 수정)
   - 위 패널을 사이드바에 끼워 넣는 세 줄 추가가 전부다.

## 데이터가 어떻게 흐르나?

```
사용자                컨트롤 패널              배경 로직                캔버스 / 저장
------                -----------              ---------                -------------
컬러피커에서    →     "이 색으로" 요청   →     배경색 = #ffcc00    →    화면 다시 그림
#ffcc00 선택                                    "수정됐다" 신호 발사 →   0.5초 뒤 자동 저장

사진 파일 선택  →     "이 파일로" 요청   →     파일을 읽음
                                                (700×450 사진이라면)
                                                가로 1400/700 = 2배
                                                세로  900/450 = 2배로 확대
                                          →     배경 이미지로 깔기  →    화면 다시 그림
                                                "수정됐다" 신호 발사 →   0.5초 뒤 자동 저장
```

저장되는 데이터의 모양은 바뀌지 않았다. 배경은 캔버스 상태를 통째로 저장하는 기존 JSON(캔버스의 모든 내용을 글자로 적어둔 저장 파일) 안에 원래부터 함께 들어가는 항목이라, 저장소 쪽 코드는 건드릴 필요가 없었다. 실제로 테스트에서 배경색을 바꾼 뒤 JSON으로 내보내면 `background: "#ffcc00"`이 포함되는 걸 확인했다.

## 쉽게 말하면

다이어리 꾸미기에 비유하면, 지금까지는 흰 속지에만 스티커를 붙일 수 있었는데, 이제 속지 자체를 색지로 바꾸거나 사진 인쇄된 속지로 갈아끼울 수 있게 됐다. 그리고 속지를 갈아끼우는 순간에도 "페이지가 바뀌었다"고 다이어리가 스스로 기록해 두기 때문에, 덮었다가 다시 펴도 갈아끼운 속지가 그대로 있다.

## 만든 파일

## 제대로 됐는지 어떻게 확인했나?

이번 단계의 합격 기준은 "코드 검사, 빌드, 자동 테스트가 전부 통과"였고, 셋 다 통과했다.

- 코드 검사(lint — 코드의 문법적 실수나 나쁜 습관을 잡아주는 자동 검사): 통과
- 빌드(build — 코드를 실제 배포 가능한 형태로 조립하는 과정): 통과
- 자동 테스트: 16개 전부 통과. 이 중 4개가 이번에 새로 쓴 것:
  1. 색을 바꾸면 캔버스 배경색이 실제로 그 색이 된다
  2. 바꾼 배경색이 저장용 JSON에 포함된다 (= 저장/불러오기에 반영된다)
  3. 배경을 바꾸면 자동 저장 장치가 듣는 신호가 발사된다
  4. 캔버스가 아직 준비 안 된 상태에서 눌러도 에러가 안 난다

테스트를 먼저 써서 실패하는 걸 확인한 뒤 구현해서 통과시키는 순서(TDD)로 진행했고, 기존 테스트 12개도 깨지지 않았다.

## 전체 그림에서 지금 어디?

```
[완료] Step 0  캔버스 확대 + 레이아웃
[완료] Step 1  배경 커스텀 (색상/이미지)   ← 지금 여기
[    ] Step 2  오브젝트 툴바 고정 메뉴화
[    ] Step 3  버튼 아이콘 적용
[    ] Step 4  다이어리 갤러리
[    ] Step 5  캘린더 날짜 호버 버튼
[    ] Step 6  메인화면 여백/스피너
```

다음은 Step 2(툴바 고정 메뉴화)다. 지금 오브젝트를 선택하면 그 위에 조작 메뉴가 떠다니는데, 드래그 중에 메뉴가 따라오지 않는 버그가 있다. 메뉴를 화면의 고정된 자리에 항상 두는 방식으로 바꿔 이 문제를 없앤다. 배경 패널이 자리 잡은 사이드바 구조가 이번에 정리됐으니, 툴바가 들어갈 자리를 정하기도 수월해졌다.

## 아직 안 된 것 (솔직하게)

- **이미지 로직의 자동 테스트는 없다.** 자동 테스트는 색상 쪽만 검증한다. 이미지 배경은 테스트 환경(브라우저 없이 도는 가짜 환경)이 실제 이미지 파일을 해독하지 못해서 자동화하기 어렵고, 이번 단계 요구사항도 색상 검증까지였다. 이미지 배경은 다음에 브라우저에서 직접 눌러보는 확인이 필요하다.
- **배경 이미지는 비율 무시하고 꽉 채운다.** 캔버스는 1400×900(가로가 세로의 약 1.56배)인데, 정사각형 사진을 올리면 가로로 늘어나 보인다. 비율 유지·자르기 같은 편집 기능은 계획 단계에서 이번 범위 밖으로 정해둔 것이라 넣지 않았다.
- **배경을 "다시 흰색으로" 되돌리는 전용 버튼은 없다.** 컬러피커에서 흰색을 고르면 되긴 하지만, 이미지 배경을 깐 뒤 이미지를 제거하는 버튼은 없다. 계획에 없던 항목이라 추가하지 않았다 — 필요하면 이후 단계에서 논의할 것.
