# Step 3: 버튼에 아이콘 입히기

## 이 단계를 왜 했나?

지금까지 다이어리 편집 화면의 버튼들은 전부 글자로만 되어 있었다. "복사", "삭제", "JSON 내보내기" 같은 텍스트가 줄줄이 붙어 있으면 한눈에 안 들어오고, 특히 Step 2에서 만든 오브젝트 조작 패널은 폭이 240px로 좁아서 "한 단계 앞으로" 같은 긴 글자가 공간을 많이 잡아먹었다.

그래서 이번 단계에서는 버튼마다 의미가 통하는 그림(아이콘)을 붙였다. 좁은 오브젝트 패널은 아이콘만 남기고, 나머지 버튼은 아이콘과 짧은 글자를 나란히 놓았다.

중요한 제약이 하나 있었다: **버튼이 하는 일은 절대 바꾸지 않는다.** 겉모습(라벨)만 바꾸고, 클릭했을 때 실행되는 동작은 이전과 100% 동일해야 한다.

## 무엇을 만들었나?

`react-icons`라는 아이콘 모음 라이브러리(이미 설치되어 있던 것)에서 Material 디자인 세트 하나만 골라 썼다. 여러 세트를 섞으면 그림체가 달라서 어색해 보이기 때문이다.

아이콘이 붙은 곳은 5개 컴포넌트(화면 부품), 총 13개 자리다:

- **오브젝트 조작 패널** (6개): 복사, 삭제(휴지통), 맨 앞으로, 맨 뒤로, 한 단계 앞으로(위 화살표), 한 단계 뒤로(아래 화살표). 여기는 공간이 좁아 **아이콘만** 표시하고, 마우스를 올리면 뜨는 풍선 도움말(title)과 화면 낭독기용 이름표(aria-label — 시각장애인용 프로그램이 버튼 의미를 읽을 수 있게 하는 속성)에 원래 글자를 남겼다.
- **이미지 추가 버튼** (1개): 사진 추가 아이콘 + "이미지 추가"
- **텍스트 추가 버튼** (1개): 글자 아이콘 + "텍스트 추가"
- **내보내기/불러오기 버튼 묶음** (3개): 다운로드 아이콘 + "JSON 내보내기", 업로드 아이콘 + "JSON 불러오기", 카메라 아이콘 + "PNG 내보내기"
- **배경 컨트롤** (2개): 색상 고르개 옆에 팔레트 아이콘, "배경 이미지" 버튼에 벽지 아이콘

아이콘 크기는 16~18px(픽셀 — 화면의 점 단위)로, 기존 버튼 높이에 맞췄다. 버튼의 색상·테두리·그라데이션은 하나도 건드리지 않았다.

## 데이터가 어떻게 흐르나?

이번 단계는 데이터 흐름 자체를 바꾸지 않았다. 바뀐 것은 버튼의 "겉"뿐이다.

```
[이전]                          [이후]
+--------------+               +------------------+
|    복사      |  --클릭-->    |  (복사 아이콘)    |  --클릭-->
+--------------+               +------------------+
       |                              |
       v                              v
  actions.copy(선택된 오브젝트)   actions.copy(선택된 오브젝트)
       (완전히 동일한 함수 호출)
```

예를 들어 오브젝트 패널의 삭제 버튼: 이전에는 "삭제"라는 글자 버튼을 누르면 `actions.remove`라는 함수가 실행됐다. 지금은 휴지통 그림 버튼을 누르면 **똑같은** `actions.remove`가 실행된다. 숫자로 말하면, 수정한 파일 5개에서 클릭 동작을 정의한 줄은 0줄 바뀌었고, 라벨(글자→아이콘)과 정렬용 스타일 몇 줄만 바뀌었다.

## 쉽게 말하면

가게 간판을 바꾼 것과 같다. "김밥천국"이라고 글자만 있던 간판에 김밥 그림을 그려 넣거나, 좁은 입구에는 그림만 남긴 것이다. 가게 안에서 파는 음식(버튼이 실제로 하는 일)은 하나도 안 바뀌었다. 손님이 헷갈리지 않도록, 그림만 있는 간판에는 가까이 가면 보이는 작은 설명(마우스 올리면 뜨는 풍선 도움말)을 달아 두었다.

## 제대로 됐는지 어떻게 확인했나?

이번에도 검사(테스트)를 먼저 써서 일부러 실패시킨 뒤, 아이콘을 붙여 통과시키는 순서로 진행했다(TDD — 검사부터 쓰고 코드를 나중에 쓰는 개발 방식).

- 아이콘 적용 전: 새로 쓴 검사 10개가 예상대로 실패 (아이콘이 아직 없으니까)
- 아이콘 적용 후: 전체 27개 검사 모두 통과
- `npm run lint` (코드 규칙 검사) 통과
- `npm run build` (실제 배포용으로 묶어보는 검사) 통과
- 자가 점검: 수정한 파일 5개의 변경 내역(diff)에서 `onClick`(클릭 시 실행 함수)이 걸린 줄을 전부 뽑아 비교했다. 클릭 시 호출하는 함수 표현식은 이전과 완전히 동일했고, 아이콘 추가와 정렬 스타일만 바뀌었음을 확인했다.

## 전체 그림에서 지금 어디?

```
[완료] Step 0: 창/캔버스 확대
[완료] Step 1: 배경 커스텀 (색상/이미지)
[완료] Step 2: 오브젝트 조작 패널 고정
[완료] Step 3: 버튼 아이콘  <-- 지금 여기
[    ] Step 4: 다이어리 갤러리 (썸네일 모아보기)
[    ] Step 5: 캘린더 날짜 호버 시 작성 버튼
[    ] Step 6: 메인화면 여백 축소 + 날씨 스피너
```

다음은 갤러리다. 편집 화면 쪽 정비(0~3)가 끝났으니, 이제 저장된 다이어리들을 메인 화면에서 썸네일로 모아보는 기능으로 넘어간다.

## 아직 안 된 것 (솔직하게)

- 아이콘이 "실제 화면에서 예쁘게 보이는지"는 자동 검사로 못 잡는다. 검사는 아이콘이 붙었는지/동작이 같은지만 확인한다. 크기나 간격이 어색하면 실제 화면을 보고 조정이 필요할 수 있다.
- 텍스트 추가 버튼을 누른 뒤 나오는 "추가"(입력 확정) 버튼에는 아이콘을 붙이지 않았다. 이번 지시 범위(텍스트 추가 진입 버튼)에 없어서 그대로 두었다.
- 배경 색상 고르개는 브라우저 기본 색상 선택 창(`<input type="color">`)이라 그 자체에 아이콘을 넣을 수 없어, 옆에 팔레트 아이콘을 나란히 두는 방식으로 처리했다.
