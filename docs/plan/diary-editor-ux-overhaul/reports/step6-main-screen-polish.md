# Step 6: 메인 화면 다듬기 — 여백·글자 크기 재조정과 날씨 로딩 스피너

## 이 단계를 왜 했나?

Step 0에서 다이어리 창을 화면의 97%까지 키웠는데, 메인 화면(캘린더 + 위젯 3종)은 창이 작던 시절 기준으로 만들어져 있었다. 그래서 창이 커지자 두 가지 문제가 눈에 띄었다.

첫째, 위젯 사이 간격과 바깥 여백이 4px로 너무 좁아서, 커진 화면에서 콘텐츠가 다닥다닥 붙어 보였다. 둘째, 위젯 안의 글자·아이콘이 예전 크기 그대로라 커진 화면 대비 너무 작아 보였다. 예를 들어 아날로그 시계는 최대 160px로 묶여 있어서, 시계 칸이 300px가 넘어도 가운데에 조그맣게 떠 있었다.

덤으로, 날씨 위젯이 데이터를 받아오는 동안 "날씨 정보를 불러오는 중..."이라는 밋밋한 텍스트만 보여줬는데, 이걸 돌아가는 스피너(로딩 중임을 나타내는 회전 고리 애니메이션)로 바꿨다.

## 무엇을 만들었나?

이번 단계는 전부 "보이는 것"만 바꿨다. 할 일 저장, 날씨 받아오기, 시간 계산 같은 동작 코드는 한 줄도 건드리지 않았다.

1. **메인 화면 간격 재조정**: 위젯들 사이 간격과 화면 가장자리 여백을 4px에서 10px로 늘렸다. 좁아서 답답하던 것을 풀어주면서도, 위젯 자체가 커진 화면을 채우도록 했다.
2. **할 일 위젯 글자 확대**: 제목("할 일")을 12px에서 14px로, 할 일 항목 글자를 13px에서 16px로 키웠다.
3. **날씨 위젯 확대 + 스피너**: 날씨 아이콘(이모지)을 32px에서 48px로, 온도 숫자를 20px에서 28px로 키웠다. 로딩 중 표시는 텍스트 대신 파란색(#1657d6, 이 앱의 XP 테마 파랑) 고리가 0.9초에 한 바퀴씩 도는 스피너로 교체했다.
4. **아날로그 시계 확대**: "최대 160px"라는 크기 제한을 제거해서, 시계가 자기 칸을 꽉 채우도록 했다. 시계는 SVG(크기를 늘려도 깨지지 않는 벡터 그림)라서 커져도 선명하다.

## 데이터가 어떻게 흐르나?

이번 변경의 핵심인 날씨 스피너의 흐름은 이렇다.

```
날씨 위젯 켜짐
     |
     v
상태 = "loading"  ------------------>  [ 파란 고리 스피너가 빙글빙글 ]
     |
     |  (인터넷에서 날씨 받아오기 — 기존 코드 그대로)
     |
     +-- 성공 --> 상태 = "ready" ---->  [ ☀️ 48px + 21°C 28px 표시 ]
     |
     +-- 실패 --> 상태 = "error" --->  [ "불러올 수 없습니다" 문구 ]
```

위젯은 내부적으로 "loading / ready / error" 세 가지 상태 중 하나를 가진다. 이 상태를 정하는 코드(위치 파악 → 날씨 API 요청 → 성공/실패 판정)는 그대로 두고, "loading일 때 무엇을 그릴까"라는 마지막 그리기 부분만 텍스트에서 스피너로 바꿨다.

숫자 예시: 날씨 API가 온도 21.4도를 돌려주면 화면에는 반올림된 "21°C"가 28px 크기로 표시된다 — 반올림 규칙도 기존 그대로다.

스피너 애니메이션 자체는 CSS(화면 꾸밈 규칙 파일)에 넣었다. "0도에서 360도까지 회전을 무한 반복"이라는 규칙을 `src/index.css`에 정의하고, 날씨 위젯은 그 규칙의 이름표(`weather-spinner`)만 붙인다.

## 쉽게 말하면

이사 가서 방(창)은 넓어졌는데 가구(위젯)와 가구 배치는 예전 원룸 시절 그대로였던 상황이다. 이번에 가구 사이를 적당히 벌리고, 벽시계와 글씨를 방 크기에 맞게 키웠다. 그리고 날씨 게시판에 "준비 중"이라고 종이만 붙여 놓던 것을, 빙글빙글 도는 안내 표시등으로 바꿔 달았다.

## 제대로 됐는지 어떻게 확인했나?

이번에도 테스트를 먼저 썼다(TDD — 원하는 동작을 검사하는 자동 테스트를 먼저 만들어 일부러 실패시킨 뒤, 통과하도록 코드를 고치는 방식). 새 테스트 3개:

1. 로딩 중에는 스피너가 보이고, 예전의 "불러오는 중" 텍스트는 없다 — 처음엔 실패(스피너가 아직 없으니까), 구현 후 통과.
2. 날씨를 받아오면 스피너가 사라지고 온도(21.4도 → "21°C")가 보인다.
3. 받아오기에 실패하면 스피너가 사라지고 에러 문구가 보인다.

약속된 확인 절차(AC) 결과: 코드 규칙 검사(lint) 통과, 배포용 빌드(build) 성공, 자동 테스트 전체 40개(기존 37 + 신규 3) 모두 통과. 기존 기능이 깨지지 않았다는 뜻이다.

## 전체 그림에서 지금 어디?

```
[0 캔버스 확대] -> [1 배경 커스텀] -> [2 툴바 고정] -> [3 아이콘]
      완료              완료             완료           완료
 -> [4 갤러리] -> [5 날짜 호버 버튼] -> [6 메인 화면 다듬기]
      완료              완료              완료 (지금 여기, 마지막 step)
```

이번 step이 이 phase(diary-editor-ux-overhaul)의 마지막이다. 계획했던 7개 항목 — 캔버스 확대, 배경 커스텀, 고정 툴바, 아이콘, 갤러리, 날짜 호버 버튼, 메인 화면 다듬기 — 이 전부 완료됐다.

## 아직 안 된 것 (솔직하게)

- 간격 10px, 글자 14/16px 같은 수치는 계획서가 정한 범위(8~12px 등) 안에서 고른 값이지, 실제 화면을 여러 해상도에서 보며 다듬은 값은 아니다. 직접 써 보고 어색하면 숫자만 바꾸면 되는 한 줄짜리 수정이다.
- 날씨 위젯의 상태 문구("맑음" 등)와 에러 문구는 13px 그대로다. 계획서가 아이콘과 온도만 키우라고 명시했기 때문에 그 둘만 건드렸다.
- 스피너는 날씨 위젯 전용 이름표로 만들었다. 다른 곳에서도 로딩 스피너가 필요해지면 그때 공용으로 빼면 된다 — 지금 미리 만들어 두지는 않았다.
- 영화 리뷰 탭, 갤러리에서 다이어리 삭제 같은 기능은 이 phase 범위 밖으로, 계획대로 손대지 않았다.
