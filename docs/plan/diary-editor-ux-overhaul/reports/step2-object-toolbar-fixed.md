# Step 2: 오브젝트 툴바를 고정 메뉴로 재설계

## 이 단계를 왜 했나?

지금까지 오브젝트 툴바(복사·삭제·앞뒤 순서 버튼 묶음)는 "플로팅" 방식이었다.
스티커나 사진을 클릭하면 그 오브젝트 바로 위에 작은 버튼 줄이 둥둥 떠서 나타나는 식이다.

이 방식에는 실제로 써 보니 문제가 있었다:

- 오브젝트를 드래그해서 옮기는 동안 툴바가 따라오지 않고 원래 자리에 남아 있었다.
- 오브젝트가 캔버스 가장자리에 있으면 툴바가 화면 밖으로 잘리거나 이상한 위치에 떴다.
- 아무것도 선택하지 않으면 툴바 자체가 사라져서, 어떤 조작이 가능한지 눈에 보이지 않았다.

그래서 계획 단계(PRD)에서 "따라다니는 툴바를 폐기하고, 화면의 고정된 자리에
항상 보이는 메뉴로 바꾼다"고 합의했고, 이번 단계가 그 실행이다.

## 무엇을 만들었나?

새로 만든 파일은 없고, 기존 파일 2개를 고쳤다.

**1. ObjectToolbar — 전면 재작성**

- 오브젝트 위치를 계산해서 그 근처에 띄우던 코드를 전부 지웠다.
- 대신 좌측 사이드바의 다른 패널들(스티커, 배경)과 똑같은 프레임 —
  파란 헤더바에 "오브젝트"라는 제목이 붙은 240픽셀 폭 패널 — 로 다시 만들었다.
- 버튼은 기존과 동일한 6개: 복사 / 삭제 / 맨 앞으로 / 맨 뒤로 / 한 단계 앞으로 / 한 단계 뒤로.
- 아무 오브젝트도 선택하지 않았을 때는 버튼 6개가 반투명(45% 불투명도)하게 흐려지고
  클릭해도 반응하지 않는다. 오브젝트를 클릭해 선택하는 순간 전부 진해지며 눌리게 된다.

**2. DiaryCanvas — 툴바 배치 이동**

- 툴바를 캔버스 위에 겹쳐 그리던 자리에서 빼서, 좌측 사이드바의
  "스티커 → 배경 → **오브젝트** → 이미지 추가 → 텍스트 추가" 순서 중간에 끼워 넣었다.
- 예전에는 "선택된 오브젝트가 있을 때만" 툴바를 그렸는데, 이제는 항상 그린다.
  선택 여부는 버튼의 활성/비활성으로만 표현된다.

중요한 점: **버튼을 눌렀을 때 실제로 일어나는 일(복사, 삭제, 순서 바꾸기)은
한 줄도 바꾸지 않았다.** 그 로직이 들어 있는 useObjectActions 파일은 손대지 않았고
(이번 단계의 금지 사항이기도 했다), 바뀐 건 "그 기능을 부르는 버튼이 어디에,
어떤 모습으로 있는가"뿐이다.

## 데이터가 어떻게 흐르나?

```
[캔버스에서 오브젝트 클릭]
        |
        v
useActiveSelection  --- "지금 선택된 오브젝트" 를 추적하는 훅*
        |                (선택 없으면 null**)
        v
ObjectToolbar (고정 패널, 항상 표시)
        |  선택 null  -> 버튼 6개 disabled(비활성) + 흐리게
        |  선택 있음  -> 버튼 6개 활성
        v
[버튼 클릭] --> useObjectActions의 기존 함수 호출 (복사/삭제/순서)
        |
        v
캔버스 갱신 --> 기존 오토세이브가 500ms 뒤 자동 저장 (기존 그대로)
```

\* 훅(hook): React에서 "상태를 추적하는 재사용 가능한 부품" 정도로 이해하면 된다.
\** null: "아무것도 없음"을 뜻하는 프로그래밍 값.

숫자 예시: 캔버스에 스티커가 3개 있고 아무것도 선택 안 한 상태면, 툴바는 보이지만
버튼 6개 모두 눌리지 않는다. 스티커 하나를 클릭하면 6개가 전부 활성화되고,
"복사"를 누르면 원본에서 오른쪽·아래로 10픽셀씩 어긋난 자리에 복제본이 생긴다
(이 10픽셀 규칙은 예전부터 있던 로직 그대로다).

## 쉽게 말하면

예전 툴바는 손님(오브젝트)을 졸졸 따라다니는 웨이터였다. 손님이 자리를 옮기면
못 쫓아가서 엉뚱한 테이블 옆에 서 있곤 했다. 이번에 웨이터를 없애고
벽에 고정된 주문 벨 패널을 달았다. 패널은 항상 같은 자리에 있고,
손님이 앉아 있을 때(오브젝트 선택)만 벨에 불이 들어온다.

## 만든 파일

새 파일 없음. 수정/추가는 다음과 같다.

- `src/components/ObjectToolbar/ObjectToolbar.jsx` — 전면 재작성 (플로팅 → 고정 패널)
- `src/components/ObjectToolbar/ObjectToolbar.test.jsx` — 신규 테스트 6개
- `src/components/DiaryCanvas/DiaryCanvas.jsx` — 툴바를 사이드바 고정 슬롯으로 이동

## 제대로 됐는지 어떻게 확인했나?

먼저 실패하는 테스트 6개를 작성해 옛 코드에서 전부 실패하는 것을 확인한 뒤(TDD),
새 구현으로 전부 통과시켰다:

- 선택이 없어도 버튼 6개가 모두 화면에 그려진다 — 통과
- 선택이 없으면 6개 모두 비활성(disabled) 상태다 — 통과
- 선택이 있으면 6개 모두 활성화된다 — 통과
- "복사" 클릭 시 기존 복사 함수가 선택된 오브젝트로 호출된다 — 통과
- "삭제" 클릭 시 기존 삭제 함수가 선택된 오브젝트로 호출된다 — 통과
- 화면 위에 겹쳐 띄우는 방식(absolute 배치)을 더 이상 쓰지 않는다 — 통과

AC(합격 기준) 결과:

- `npm run lint` — 통과
- `npm run build` — 통과
- `npm run test` — 전체 22개(기존 16 + 신규 6) 통과
- grep 자가 점검: ObjectToolbar.jsx에 `fabric` import 없음,
  `getBoundingRect` 호출이 src 폴더 전체에서 0건 — 확인 완료

## 전체 그림에서 지금 어디?

```
step 0  캔버스 확대/레이아웃      [완료]
step 1  배경 색상/이미지 커스텀   [완료]
step 2  툴바 고정 메뉴화          [완료]  <-- 지금 여기
step 3  버튼 아이콘 적용          [다음]
step 4  다이어리 갤러리
step 5  캘린더 호버 작성 버튼
step 6  메인화면 여백/스피너
```

다음이 step 3(아이콘 적용)인 이유: 이번에 툴바 버튼 6개가 고정 패널로 자리를
잡았으니, 이제 그 버튼들과 이미지/텍스트/내보내기 버튼의 텍스트 라벨을
아이콘으로 바꾸는 작업을 안정된 레이아웃 위에서 할 수 있다.
툴바 재배치 전에 아이콘부터 넣었다면 두 번 손댈 뻔했다.

## 아직 안 된 것 (솔직하게)

- 버튼이 아직 전부 텍스트다. "복사", "한 단계 앞으로" 같은 글자 라벨이며,
  아이콘 교체는 다음 step 3의 일이다.
- 버튼 6개가 240픽셀 폭 패널 안에서 두 줄로 감싸며 배치되는데, 라벨 길이가
  제각각이라("복사" vs "한 단계 앞으로") 줄바꿈 모양이 고르지 않다.
  step 3에서 아이콘으로 바뀌면 자연스럽게 정리될 것으로 보고 지금은 손대지 않았다.
- 여러 오브젝트를 한꺼번에 선택(다중 선택)한 경우는 예전과 동일하게 툴바가
  비활성 상태다 — 다중 선택 조작은 원래부터 이 툴바의 범위 밖이다.
- 실제 브라우저에서 마우스로 눌러 보는 수동 확인은 자동 테스트로 대체했다.
  자동 테스트가 렌더링·클릭·비활성화를 모두 검증하지만, 픽셀 단위 생김새까지
  보장하지는 않는다.
