## UI 전반 정비 (`ui-polish`, 나중에 모아서 한번에)

지금까지 만든 화면들이 기능적으론 동작하지만 "전반적으로 엉성하다"는 피드백이 있음 — 구체적으로 어디가 문제인지 아직 언어화가 안 된 상태. 개별 컴포넌트를 그때그때 고치는 대신, 위 기능 phase(1~6)를 다 끝낸 뒤 모든 화면을 한 자리에 놓고 한 번에 정비한다.

- 착수 시점에 브라우저로 전체 화면(Desktop, Window, Taskbar, 캘린더, 위젯, 캔버스 툴바 등)을 스크린샷으로 훑으며 공통 문제(여백, 그림자, 정렬 기준 등)를 먼저 언어화한다.
- 언어화된 항목은 `match-reference` 스킬로 레퍼런스 이미지 대비 실측 비교하며 고친다 — "마음에 들 때까지"가 아니라 "레퍼런스와 실측값이 일치할 때까지"를 종료 조건으로 삼는다.
- 기능 phase 중간에 사소한 스타일 어긋남(버튼 색이 스펙과 다름 등)을 발견하면 그 자리에서 바로 고쳐도 된다 — 이 phase는 "전반적으로 엉성함"처럼 언어화 안 된 큰 덩어리를 모아 처리하기 위한 것이지, 사소한 수정까지 전부 미루라는 뜻은 아니다.
