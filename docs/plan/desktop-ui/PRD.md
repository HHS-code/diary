# PRD — Windows XP 스타일 데스크톱 진입 UI

## 배경 / 왜

지금은 앱을 켜면 바로 다이어리 화면(캘린더+탭+캔버스)이 뜬다. 이를 Windows XP 데스크톱처럼 꾸며서, 앱을 켜면 먼저 바탕화면이 뜨고 아이콘을 눌러야 기능(다이어리)으로 들어가는 구조로 바꾼다.

## 목표

- 앱 시작 시 Windows XP 느낌의 데스크톱 화면(Bliss 배경 + 아이콘 + 하단 taskbar)이 먼저 보인다.
- "diary" 아이콘을 클릭하면 기존 다이어리 화면이 떠있는 창(타이틀바 + 닫기 버튼) 형태로 나타난다.
- 하단 taskbar의 전원 버튼을 클릭하면 "DOWNLOAD" 메뉴가 뜨고, 클릭하면 저장된 전체 데이터를 JSON 파일로 자동 다운로드한다.
- 하단 taskbar 우측에 날짜·시간이 실시간으로 표시된다.

## 대상 사용자

기존과 동일 (혼자 쓰는 개인 다이어리 사용자). 이번 작업은 진입 화면(껍데기)만 바꾸는 것으로, 다이어리 자체의 기능/사용자 대상에는 변화 없음.

## In-scope

- **Desktop 화면**: Bliss 배경화면 + "diary" 아이콘(이미지+텍스트) + 하단 taskbar. 앱 최초 진입 시 항상 이 화면부터 보인다.
- **Window(다이어리 창)**: "diary" 아이콘 클릭 시 화면 대부분을 채우는 고정 크기 창이 화면 중앙에 뜬다.
  - 타이틀바 + 닫기(X) 버튼 포함. 닫기를 누르면 Desktop 화면으로 돌아간다.
  - 드래그 이동 불가, 항상 화면 중앙 고정.
  - 창이 열려 있는 동안 뒤의 Desktop 아이콘 클릭은 막힌다(모달).
  - 창 안의 내용은 기존 다이어리 화면(Calendar + Tabs + DiaryCanvas + ExportImportControls)을 **로직 변경 없이 그대로** 감싼 것.
- **Taskbar**: 화면 하단 전체 폭을 덮는 바. 좌측에 전원 버튼, 우측에 실시간 시계. Desktop/Window 상태와 무관하게 항상 보인다.
- **전원 버튼 → DOWNLOAD**: 클릭 시 버튼 위에 "DOWNLOAD" 항목(아이콘+라벨) 하나짜리 메뉴가 뜬다.
  - 클릭하면 저장된 전체 다이어리 데이터(JSON)를 파일로 자동 다운로드한다. 기존 `ExportImportControls`의 JSON 내보내기와 동일한 데이터/로직을 재사용한다.
  - 메뉴 바깥을 클릭하면 메뉴가 닫힌다.
- **실시간 시계**: "H:MM a.m./p.m." + "YYYY-MM-DD" 형식(레퍼런스 목업과 동일), 1초마다 내부 갱신.
- **아이콘 에셋 정리**: reference 폴더에서 실제 쓰는 이미지 4개(배경, diary 폴더 아이콘, 전원 버튼, 다운로드 아이콘)만 `src/assets`로 옮기고, `src/assets/icons.js` 한 파일에서 모아 export한다.

## Out-of-scope (명시)

- Desktop 아이콘 확장(향후 다른 앱 아이콘 추가를 위한 배열/맵 구조) — 지금은 "diary" 아이콘 하나만, 하드코딩.
- 창 드래그 이동, 리사이즈, 최소화/최대화 등 실제 OS 창 동작.
- 전원 버튼에 "종료"·"로그오프" 등 DOWNLOAD 외 다른 메뉴 항목.
- 다이어리 자체 기능 변경(캘린더, 캔버스, 스티커, 텍스트, PNG 내보내기, 영화 탭 등) — 전부 기존 그대로.
- 새 npm 패키지 추가(라우터 등) — App.jsx 내부 state로만 화면 전환.
- 반응형/모바일 대응 — 기존 프로젝트 범위와 동일하게 데스크톱 브라우저 기준.

## 성공 기준

- 앱을 새로고침하면 항상 Desktop 화면(배경+아이콘+taskbar)부터 보인다.
- "diary" 아이콘 클릭 → 다이어리 창이 뜨고, 기존과 동일하게 캘린더/탭/캔버스가 정상 동작한다(날짜 선택, 스티커/이미지/텍스트 추가, 자동 저장, JSON/PNG 내보내기 모두 기존과 동일하게 동작).
- 창이 열린 동안 뒤 데스크톱 아이콘 클릭이 무시된다. 닫기 버튼을 누르면 Desktop으로 돌아간다.
- 전원 버튼 클릭 → DOWNLOAD 메뉴 노출 → 클릭 시 전체 데이터가 JSON 파일로 다운로드된다(내용은 기존 "JSON 내보내기" 결과와 동일).
- 하단 우측 시계가 실제 시간과 일치해 1초 단위로 최신 상태를 유지한다(화면 표시는 분 단위).
- 다이어리 관련 기존 컴포넌트/storage 코드는 한 줄도 로직이 바뀌지 않는다 (이동/래핑만 발생).

## 진행 방식

1. 문서(PRD/architecture) 합의 → 실행 승인
2. 에셋 정리 → Desktop/Window/Taskbar/PowerButton/Clock 컴포넌트 구현 → App.jsx 조합 → 완료 시 사용자에게 결과 설명
