# PRD — free-drawing (XP 그림판식 자유 그리기)

## 배경 / 왜

지금 다이어리 캔버스는 스티커·사진·텍스트를 "물건"으로 올려 꾸미는 방식만 지원한다. 손으로 낙서하고 그림을 그리는 표현이 없어서 다이어리 감성의 핵심이 빠져 있다. 그리기 경험은 **Windows XP 그림판**을 기준으로 삼는다 — 도구 팔레트, 색상 팔레트, 그리는 손맛, 지우개, 실행 취소(Ctrl+Z)까지.

## 목표

- 캔버스 위에 연필/브러시/에어브러시로 자유롭게 그릴 수 있다.
- 그린 획은 그림판처럼 종이에 "박제"된다 — 클릭해서 잡거나 옮길 수 없고, 지우개와 Ctrl+Z로만 수정한다.
- 진짜 지우개: 문지르면 지나간 자리의 획이 지워진다.
- 모든 캔버스 편집(그리기·지우기·스티커/사진/텍스트 추가·이동·삭제)이 Ctrl+Z / Ctrl+Y로 되돌리기·다시 실행된다.
- 도구 UI는 XP 그림판의 도구 팔레트 + 색상 팔레트 룩을 따른다.

## 대상 사용자

기존과 동일. 다이어리 편집 화면(`DiaryCanvas`)을 쓰는 개인 사용자.

## 핵심 설계 결정 (coplan OQ 합의, 2026-07-14)

| # | 결정 | 근거 |
|---|---|---|
| 1 | 그린 획만 그림판처럼 박제(선택 불가), 스티커/사진/텍스트는 기존대로 움직이는 오브젝트 유지 | 전체 래스터화는 기존 phase의 오브젝트 조작 기능(툴바·텍스트 수정 등)을 무력화함 |
| 2 | 지우개는 진짜 지우개(`@erase2d/fabric`, fabric ≥6 호환 확인됨) | 획 선택+Delete는 박제 결정과 모순이고 그림판 감각이 아님 |
| 3 | 지우개는 그린 획만 지운다 — 스티커/사진/텍스트 위를 지나가면 아무 일도 없음 | 반쯤 지워진 스티커를 옮기거나 반쯤 지워진 텍스트를 수정하는 건 이상한 경험. 오브젝트 제거는 기존 방법(선택→Delete)이 자연스러움 |
| 4 | undo는 캔버스 전체 편집 대상, 히스토리 30단계 제한 | 그림판의 실행 취소와 같은 감각. 30단계 제한은 스냅샷 메모리 보호 |
| 5 | 브러시는 연필(가는 고정선)/브러시(굵기 조절)/에어브러시(스프레이) 3종 | XP 그림판 도구 세트 기준 |
| 6 | 그림판 UI는 `match-reference` 스킬로 레퍼런스 스크린샷과 실측 비교해 검증 | "마음에 들 때까지"가 아니라 "실측값 일치"가 종료 조건 |

## In-scope

- **도구 팔레트 패널**: 좌측 사이드바에 XP 그림판식 패널 추가 — 2열 아이콘 격자(선택/연필/브러시/에어브러시/지우개), 그 아래 굵기 옵션, 그 아래 색상 팔레트.
- **선택 도구(화살표)**: 그리기 모드 끄기 = 기존 오브젝트 선택/이동 모드. 기본값. 그리기 도구를 고르면 그리기 모드가 켜지고, 선택 도구로 돌아오면 꺼진다 (사이드바 토글 방식).
- **연필**: 가는 고정 굵기 선.
- **브러시**: 색상·굵기 조절 가능한 선.
- **에어브러시**: 스프레이 효과.
- **지우개**: 문지르면 그린 획만 지워짐. 굵기 조절 가능.
- **색상 팔레트**: XP 그림판식 기본 색상 격자 + 현재 색 표시.
- **획 박제**: 그린 획은 선택·이동 불가. 새로고침 후에도 박제 상태 유지.
- **전체 undo/redo**: Ctrl+Z / Ctrl+Y, 모든 캔버스 편집 대상, 히스토리 30단계.
- **저장 호환**: 그린 획·지운 상태가 기존 오토세이브(localStorage)·JSON export/import·모아보기 썸네일에 그대로 포함된다.

## Out-of-scope (명시)

- 도형 그리기(직선/곡선/사각형/원/다각형) — 실현 가능성은 높으나 이번 범위가 이미 큼. **후속 phase 후보(`paint-shapes`)로 ROADMAP에 기록**
- 채우기(페인트통) — 픽셀 개념이라 오브젝트 캔버스와 성립하지 않음, 별도 설계 필요
- 돋보기 — 기존 자동 표시 배율과 충돌
- 영역 선택 — 기존 오브젝트 드래그 다중 선택과 역할 중복
- 스포이드(색 추출)
- 태블릿 필압 지원
- 텍스트 도구 — 기존 `TextMemoButton`이 이미 담당
- 기존 오브젝트(스티커/사진/텍스트)에 지우개 적용

## 성공 기준

- 도구 팔레트에서 연필/브러시/에어브러시를 고르고 캔버스에 드래그하면 선이 그려진다. 색상·굵기가 반영된다.
- 그린 획은 선택 도구로 돌아와서 클릭해도 잡히지 않는다. 새로고침 후에도 그대로다.
- 지우개로 획 위를 문지르면 지나간 부분만 지워진다. 스티커 위를 문지르면 아무 일도 없다.
- Ctrl+Z가 그리기·지우기·스티커 이동/추가/삭제를 한 단계씩 되돌리고, Ctrl+Y가 다시 실행한다.
- 그린 획과 지운 상태가 새로고침·export/import·모아보기 썸네일에 유지된다.
- 도구 팔레트 UI가 `match-reference` 실측 비교로 레퍼런스(`reference/ms-paint-on-xp-v0-lilxt1d7tbf11.webp`)와 일치한다.
- 기존 기능 무손상: 스티커/사진/텍스트 추가·이동·오브젝트 툴바·키보드 단축키가 이전과 동일하게 동작한다.

## 진행 방식

1. 문서(PRD/architecture) 합의 → 실행 승인
2. 그리기 도구(연필/브러시/에어브러시) + 획 박제 → 지우개 → 전체 undo/redo → XP 그림판 UI 실측 매칭 → 완료 시 사용자에게 결과 설명
