# 빌드 리포트 — diary (free-drawing)

각 step이 무엇을 했는지 위에서 아래로 읽으면 프로젝트가 자라는 과정이 보인다.

- [완료] **Step 0: paint-tools-drawing** — usePaintTools(fabricCanvasRef) -> { tool, color, width, setTool, setColor, setWidth } 신규 (src/hooks/usePaintTools.js). tool ∈ select|pencil|brush|airbrush(기본 select), 도구/색/굵기는 useEffect([tool,color,width])에서 applyToolToCanvas로 캔버스에 반영 — 브러시는 매번 새로 생성(PencilBrush 굵기1 고정 | PencilBrush width | SprayBrush width+density=width*5). path:created(PencilBrush의 Path·SprayBrush의 Group 모두 path 키)에서 selectable:false, evented:false, isFreeDrawing:true, erasable:true로 박제. 캔버스 준비 폴링은 useActiveSelection의 50ms setInterval 패턴 재사용. useFabricCanvas.js EXTRA_SERIALIZED_PROPS에 'isFreeDrawing','erasable' 추가됨. 테스트는 @testing-library 없이 react act+createRoot 하니스(usePaintTools.test.js 상단 renderPaintTools 참고 — step 1 지우개 테스트 시 재사용 가능). eraser 도구·@erase2d/fabric import는 아직 없음(step 1 몫), DiaryCanvas 연결도 아직 없음(step 2 몫). ([리포트](step0-paint-tools-drawing.md))
- [완료] **Step 1: eraser-erase2d** — usePaintTools tool에 'eraser' 추가 — buildEraserBrush(canvas, width)가 @erase2d/fabric EraserBrush 생성, end 이벤트에서 event.preventDefault() 후 await brush.commit(event.detail)로 수동 커밋하고 canvas.fire('object:modified')로 오토세이브 파이프라인을 태움(targets 비면 skip). erasable:true 획만 지워짐(walk가 필터) — 스티커/사진/텍스트 무영향 테스트 증명. @erase2d/fabric import는 usePaintTools.js에만 있고, 이 import 부수효과로 ClippingGroup(type 'clipping')이 classRegistry에 등록됨 — 별도 등록 코드 불필요. 직렬화 검증: src/hooks/freeDrawingSerialization.test.js — 지워진 획(clipPath=ClippingGroup) toObject→loadFromJSON 왕복·픽셀 검증·박제 속성 유지·buildThumbnail(StaticCanvas) 복원·SprayBrush Group 왕복 전부 통과 (EXTRA_SERIALIZED_PROPS 목록은 테스트에 복사돼 있음 — useFabricCanvas와 동기화 필요). 실제 포인터 문지르기는 jsdom 불가 — 브라우저 수동 검증 항목(리포트 참고). DiaryCanvas 연결은 여전히 step 2 몫. ([리포트](step1-eraser-erase2d.md))
- [완료] **Step 2: paint-toolbox-ui** — PaintToolbox({ tool, color, width, onToolChange, onColorChange, onWidthChange }) 신규 (src/components/PaintToolbox/PaintToolbox.jsx) — fabric 미import 순수 UI. 도구 2열 격자 5개(선택 MdNearMe/연필 MdCreate/브러시 MdBrush/에어브러시 MdBlurOn/지우개 MdCleaningServices, aria-label=한글명, 현재 도구 aria-pressed+sunken 스타일), 굵기 버튼 [2,4,6,8](brush/airbrush/eraser만 활성, select/pencil disabled), XP 28색 팔레트(PALETTE_COLORS 2행×14열, aria-label='색상 #HEX')+현재 색 표시 칸. 색 선택은 모든 도구에서 활성(일관성 재량 선택). DiaryCanvas의 CanvasWorkspace에 usePaintTools 호출 추가, PaintToolbox를 사이드바 최상단(StickerPalette 위) 배치 — 그 외 무수정. 테스트 src/components/PaintToolbox/PaintToolbox.test.jsx (ObjectToolbar.test.jsx의 createRoot+act+aria-label 패턴, 9개). 픽셀 정밀 매칭은 step 4(match-reference) 몫 — 스타일은 design.md 패널/버튼 스펙 재사용. ([리포트](step2-paint-toolbox-ui.md))
- [완료] **Step 3: canvas-history** — useCanvasHistory(fabricCanvasRef) -> { undo, redo, canUndo, canRedo } 신규 (src/hooks/useCanvasHistory.js). object:added/modified/removed마다 canvas.toObject(EXTRA_SERIALIZED_PROPS) 스냅샷을 past 배열에 push(30개 캡, 초과 시 shift), 새 변경 시 future 비움. undo=pop(past)→push(future)→loadFromJSON(새 past 최상단), redo는 반대. loadFromJSON 복원 중에는 isRestoringRef로 recordSnapshot을 막고, 복원 뒤 canvas.fire('object:modified')를 '가드 해제 전'에 호출해 그 fire조차 히스토리로 안 쌓이게 함(오토세이브 파이프라인은 그대로 탐). EXTRA_SERIALIZED_PROPS는 useFabricCanvas.js에서 export로 변경(값 동일, 재사용). Ctrl+Z/Y 전역 keydown 등록, IText 편집 중(getActiveObject().isEditing)엔 가로채지 않음 — useCanvasKeyboardShortcuts와 동일 판별. DiaryCanvas.jsx엔 useCanvasHistory(fabricCanvasRef) 호출 한 줄만 추가(UI 버튼 없음, PRD 범위). 알려진 한계: 초기 base 스냅샷은 '마운트 후 캔버스 발견 시점'에 잡히므로, 저장된 diary(canvasJSON)가 아직 loadFromJSON 복원 중일 때 이 훅이 붙으면 복원이 일으키는 object:added들이 히스토리에 별도 단계로 쌓일 수 있음(자기 자신의 undo/redo 왕복은 가드로 완전히 막혀 있음 — 문제는 useFabricCanvas의 '최초 1회' 복원뿐). useFabricCanvas는 onLoaded 콜백을 이미 DiaryCanvas의 fitLoadedObjects가 쓰고 있어 이번 step 금지 규칙(오토세이브 로직 무수정) 때문에 추가 배선을 하지 않음 — 실사용에서 문제가 확인되면 후속 step에서 onLoaded 신호 공유 방식 논의 필요. 테스트 src/hooks/useCanvasHistory.test.js (usePaintTools.test.js와 동일한 react act+createRoot 하니스, 7개). ([리포트](step3-canvas-history.md))
