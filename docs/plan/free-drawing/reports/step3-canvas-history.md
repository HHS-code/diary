# Step 3: canvas-history

## 이 단계를 왜 했나?

XP 그림판에서 그림을 그리다가 실수하면 Ctrl+Z 한 번으로 방금 그은 선이 사라진다. 이 다이어리 캔버스에도 같은 게 필요했다. 그냥 "그린 선"만 되돌리는 게 아니라 — 스티커를 놓고, 사진을 옮기고, 텍스트를 지우는 것까지 포함해서, 캔버스에서 일어나는 모든 편집을 Ctrl+Z(되돌리기)/Ctrl+Y(다시하기)로 오갈 수 있게 만드는 게 이번 단계의 목표였다.

이전 단계(step 0~2)까지는 "그리기 도구"와 "지우개"와 "도구 패널 UI"를 만들었다. 그런데 그것만으로는 실수했을 때 되돌릴 방법이 없었다 — 잘못 그은 선을 지우려면 지우개로 일일이 문질러 지워야 했다. 이번 단계로 그 불편함이 사라진다.

## 무엇을 만들었나?

`useCanvasHistory`라는 새 로직 파일(`src/hooks/useCanvasHistory.js`)을 만들었다. ("훅"이라는 개발 용어가 나오는데, React에서 "화면에 안 보이는 부가 기능을 캔버스에 붙여주는 부품" 정도로 이해하면 된다. 이전 단계에서 만든 `usePaintTools`, `useCanvasKeyboardShortcuts`도 같은 종류다.)

이 부품이 하는 일은 두 가지다.

1. **캔버스에서 뭔가 바뀔 때마다 스냅샷(사진)을 찍어 쌓아둔다.** 오브젝트가 추가되거나, 옮겨지거나, 지워질 때마다 "지금 캔버스가 어떤 상태인지"를 통째로 기록해서 쌓아둔다. 이 기록은 최대 30장까지만 쌓인다 — 그 이상 오래된 기록은 자동으로 버려진다. (다이어리에 사진을 많이 붙일수록 스냅샷 한 장의 용량이 커지기 때문에, 무한정 쌓아두면 브라우저 메모리를 다 먹어버릴 수 있어서 30장으로 제한했다.)
2. **Ctrl+Z를 누르면 쌓아둔 스냅샷 중 바로 이전 것을 다시 불러와서 캔버스에 되돌린다.** Ctrl+Y는 그 반대로, 되돌리기 전 상태로 다시 넘어간다.

## 데이터가 어떻게 흐르나?

```
[캔버스에서 변경 발생]              [Ctrl+Z]                    [Ctrl+Y]
       │                              │                            │
       ▼                              ▼                            ▼
 스냅샷 찍어서                   past 맨 위 스냅샷을          future 맨 위 스냅샷을
 past 더미 맨 위에 쌓음          future로 옮기고              past로 옮기고
 (future 더미는 비움)            그 아래 스냅샷을 복원          그 스냅샷을 복원
       │                              │                            │
       ▼                              ▼                            ▼
   past: [..., 최신]            past: [..., 이전]            past: [..., 이전, 최신]
   future: []                   future: [최신]                future: []
```

숫자로 예를 들면: 빈 캔버스에서 시작해서 스티커를 3번 붙였다고 하자. 이제 `past` 더미에는 [빈 캔버스, 스티커1개, 스티커2개, 스티커3개] 이렇게 4장의 스냅샷이 쌓여 있다. 여기서 Ctrl+Z를 두 번 누르면:

- 1번째 Ctrl+Z: 스티커3개 스냅샷을 `future`로 옮기고, 캔버스는 "스티커2개" 상태로 되돌아간다.
- 2번째 Ctrl+Z: 스티커2개 스냅샷을 `future`로 옮기고, 캔버스는 "스티커1개" 상태로 되돌아간다.

이때 `future`에는 [스티커2개, 스티커3개]가 쌓여 있으므로, Ctrl+Y를 누르면 순서대로 다시 스티커2개 → 스티커3개로 복원된다.

단, **되돌린 뒤에 새로 뭔가를 그리거나 붙이면 `future`는 통째로 버려진다.** 되돌아간 시점에서 새로운 이야기가 시작되는 것이니, "다시하기"로 돌아갈 미래가 더 이상 없다는 뜻이다. (그림판에서도 마찬가지로 동작한다.)

## 쉽게 말하면

포토샵이나 그림판의 "작업 내역" 패널을 떠올리면 된다. 매 작업마다 카메라로 사진을 찍어 앨범에 순서대로 끼워 넣는다고 상상하면 쉽다. Ctrl+Z는 "한 장 뒤로 넘겨서 그 사진 속 모습으로 되돌리기"고, Ctrl+Y는 "다시 한 장 앞으로 넘기기"다. 앨범은 30장이 꽉 차면 제일 오래된 사진부터 버리고 새 사진을 끼운다 — 서랍이 무한정 크지 않기 때문이다.

## 제대로 됐는지 어떻게 확인했나?

이번 단계는 "TDD"(테스트를 먼저 실패하게 작성하고, 그걸 통과시키도록 구현하는 방식)로 진행했다. 아래 7개 상황을 자동 테스트로 만들고 전부 통과시켰다.

1. 오브젝트를 추가한 뒤 Ctrl+Z에 해당하는 동작을 하면 사라지고, 다시하기를 하면 재등장한다.
2. 변경을 2번 한 뒤 되돌리기를 2번 하면 맨 처음(빈 캔버스) 상태로 돌아간다.
3. 되돌린 뒤 새로 변경하면 "다시하기"가 불가능해진다.
4. 변경을 31번 하면 기록이 30개로 제한되어, 제일 오래된 상태로는 더 이상 못 돌아간다.
5. 되돌리기 자체가 캔버스를 복원하는 과정에서 발생시키는 부가 이벤트가 새로운 기록으로 잘못 쌓이지 않는다.
6. 실제 키보드 Ctrl+Z 입력으로 되돌리기가 동작한다.
7. 텍스트를 편집하는 중에는 Ctrl+Z를 가로채지 않는다(브라우저 기본 텍스트 되돌리기가 그대로 작동하도록).

이후 정해진 3가지 검증을 모두 통과시켰다:
- `npm run lint` — 코드 스타일 검사 통과
- `npm run test` — 전체 자동 테스트 87개 전부 통과(기존 80개 + 이번에 추가한 7개)
- `npm run build` — 실제 배포용으로 빌드까지 문제없이 완료

## 전체 그림에서 지금 어디?

```
step 0: 그리기 도구(연필/브러시/에어브러시)  ✅ 완료
step 1: 지우개                              ✅ 완료
step 2: 도구 팔레트 UI                       ✅ 완료
step 3: 되돌리기/다시하기(이번 단계)          ✅ 완료  ← 지금 여기
step 4: XP 그림판 UI 정밀 매칭                남음(사람 개입 필요 — 화면 캡처 비교)
```

기능적으로는 이번 단계로 PRD에 적힌 "모든 캔버스 편집이 Ctrl+Z/Ctrl+Y로 되돌려진다"는 목표가 충족됐다. 남은 step 4는 기능이 아니라 "생김새"를 다듬는 작업이라, 실제 화면을 눈으로 보면서 레퍼런스 이미지와 맞춰야 해서 자동으로 진행할 수 없다.

## 아직 안 된 것 (솔직하게)

- **되돌리기/다시하기 버튼이 화면에 없다.** 이번 단계는 PRD에 명시된 대로 키보드 단축키(Ctrl+Z/Ctrl+Y)만 만들었다. 마우스로 누를 수 있는 버튼은 의도적으로 만들지 않았다.
- **아주 드문 경우 하나가 남아있다.** 이미 저장된 다이어리 페이지를 열자마자(그림·사진이 화면에 다 그려지기도 전에) 바로 Ctrl+Z를 누르면, 페이지를 불러오는 과정 자체가 "여러 단계의 변경"으로 잘못 기록됐을 가능성이 있다 — 이 경우 되돌리기가 예상보다 여러 번 필요하거나 화면이 살짝 어색하게 보일 수 있다. 실제로 자주 부딪힐 상황은 아니다(페이지를 열자마자 0.1초 안에 Ctrl+Z를 누르는 경우라서). 이 부분은 기존 저장 기능 코드를 건드리지 않는다는 이번 단계의 제약 때문에 완전히 막지는 못했고, 다음에 필요하면 별도로 다뤄야 한다.
- 브라우저에서 마우스로 실제로 그림을 그리고 Ctrl+Z를 눌러보는 수동 확인은 아직 안 했다 — 자동 테스트로는 캔버스 상태 변화까지는 검증했지만, 실제 화면에서 손맛이 어떤지는 사람이 봐야 한다.
