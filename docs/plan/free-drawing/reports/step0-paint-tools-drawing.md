# Step 0: 그리기 도구 로직 (연필·브러시·에어브러시 + 획 박제)

## 이 단계를 왜 했나?

다이어리 캔버스에 손으로 그림을 그리는 기능을 넣는 첫 단계다. XP 그림판처럼
연필/브러시/에어브러시로 선을 긋고, 그은 선은 종이에 "박제"되어(클릭해도
잡히지 않음) 스티커처럼 움직이지 않게 만드는 것이 목표다.

이번 단계는 일부러 화면(버튼, 팔레트)은 만들지 않고 **동작 규칙만** 만들었다.
"어떤 도구를 고르면 캔버스가 어떻게 반응해야 하는가"라는 규칙을 먼저 완성하고
검증해 두면, 다음 단계에서 버튼 UI는 이 규칙에 끼우기만 하면 되기 때문이다.

## 무엇을 만들었나?

**usePaintTools** 라는 훅(React에서 상태와 동작을 묶어 재사용하는 부품)을 새로 만들었다.
하는 일은 세 가지다.

1. **도구 전환**: 도구는 4가지 — 선택(기본)/연필/브러시/에어브러시.
   - "선택"이면 그리기 모드를 끈다 → 기존처럼 스티커·사진·텍스트를 잡고 옮길 수 있다.
   - 그리기 도구를 고르면 그리기 모드를 켜고, 도구에 맞는 붓을 캔버스에 꽂는다.
     - 연필: 굵기 1 고정의 가는 선 (색만 바뀜)
     - 브러시: 사용자가 정한 굵기의 선 (기본 굵기 4)
     - 에어브러시: 스프레이 분사. 굵기를 키우면 분사 범위와 점 개수(밀도)가
       같이 커진다. 예: 굵기 4 → 점 20개씩, 굵기 8 → 점 40개씩 분사.
2. **색·굵기 즉시 반영**: 그리는 도중 색이나 굵기를 바꾸면 지금 꽂혀 있는 붓에
   바로 적용된다. 단, 연필은 굵기를 바꿔도 항상 1을 유지한다(그림판의 연필과 동일).
3. **획 박제**: 선 하나를 다 긋는 순간, 그 선에 "선택 불가, 클릭 무반응" 딱지와
   "이건 자유 그리기 획이다(isFreeDrawing), 지우개로 지울 수 있다(erasable)" 딱지를
   붙인다. 이 딱지 덕분에 그린 선은 스티커처럼 잡히지 않고, 다음 단계의 지우개가
   "지워도 되는 것"만 골라낼 수 있다.

또 하나, 저장 규칙 파일(useFabricCanvas)에 한 줄을 보탰다: 캔버스를 저장할 때
위의 두 딱지(isFreeDrawing, erasable)도 함께 저장하도록 했다. 이게 없으면
새로고침 후 그린 선이 다시 "잡히는" 상태로 되살아난다.

## 데이터가 어떻게 흐르나?

```
사용자가 도구/색/굵기를 고름 (다음 단계에서 만들 버튼)
        |
        v
  usePaintTools 상태 변경         예: tool='brush', color='#ff0000', width=12
        |
        v
  캔버스에 붓 설정                 그리기 모드 ON + 빨간색 굵기12 붓 장착
        |
        v
  사용자가 드래그해서 선을 그음
        |
        v
  선 완성 이벤트(path:created)     생긴 선 오브젝트에 딱지 4개 부착:
        |                          선택불가 / 클릭무반응 / 자유그리기 / 지우개가능
        v
  기존 자동 저장 (500ms 뒤)        딱지까지 포함해 localStorage에 저장
```

풀이: 선을 하나 그으면 캔버스가 "선이 생겼다"는 신호(path:created)를 보내고,
훅이 그 순간 딱지를 붙인다. 그다음은 기존 자동 저장 장치가 알아서 처리한다 —
캔버스에 뭔가 추가되면 0.5초 뒤 저장하는 구조가 이미 있어서, 그리기용 저장
코드를 따로 만들 필요가 없었다.

숫자 예시: 굵기 8로 에어브러시를 고르면, 마우스를 움직이는 동안 지름 8 범위에
점 40개(굵기 x 5)씩 계속 뿌려지고, 손을 떼는 순간 그 점 무더기 전체가 하나의
박제된 획으로 저장된다.

## 쉽게 말하면

다이어리를 코팅된 보드판이라고 하면, 스티커·사진은 보드에 자석으로 붙인
물건이라 언제든 떼어 옮길 수 있다. 이번에 만든 그리기는 보드에 유성 매직으로
직접 쓰는 것이다 — 한번 그으면 손으로 집어 옮길 수 없고, 나중에 전용 지우개
(다음 단계)로 지우는 수밖에 없다. 이번 단계는 그 매직펜 3자루(가는 펜, 굵은 붓,
스프레이)와 "그은 것은 옮길 수 없다"는 규칙을 만든 것이다.

## 만든 파일

- `src/hooks/usePaintTools.js` — 신규. 도구 전환·붓 설정·획 박제 로직.
- `src/hooks/usePaintTools.test.js` — 신규. 위 동작 11개를 자동 검증하는 테스트.
- `src/hooks/useFabricCanvas.js` — 수정. 저장 시 포함할 속성 목록에
  isFreeDrawing, erasable 두 개 추가 (그 외 무수정).
- `docs/plan/free-drawing/steps/index.json` — step 0 완료 표시.

## 제대로 됐는지 어떻게 확인했나?

세 가지 자동 검사를 모두 통과했다 (모두 종료 코드 0):

- `npm run lint` — 코드 규칙 검사(맞춤법 검사기 같은 것): 통과
- `npm run test` — 자동 테스트 58개 전부 통과. 기존 테스트 47개가 그대로
  통과해 기존 기능이 깨지지 않았음을 확인했고, 이번에 새로 쓴 11개가
  다음을 검증한다:
  - 처음에는 "선택" 모드이고 그리기 모드가 꺼져 있다
  - 연필/브러시/에어브러시를 고르면 그리기 모드가 켜지고 붓 종류·굵기가 맞다
  - "선택"으로 돌아오면 그리기 모드가 꺼진다
  - 그은 선(에어브러시 산출물 포함)에 박제 딱지 4개가 붙는다
  - 색·굵기 변경이 현재 붓에 즉시 반영되고, 연필은 굵기 1을 유지한다
  - 캔버스가 아직 준비되지 않았어도 오류가 나지 않는다
- `npm run build` — 실제 배포용으로 묶는 작업: 통과

참고: 계획서에는 "기존 테스트 40개"라고 적혀 있었지만 실제로는 47개였다
(계획서 작성 후 다른 작업에서 늘어난 것으로 보임). 47개 전부 유지됐으므로
목적("기존 테스트를 깨뜨리지 않는다")은 충족이다.

## 전체 그림에서 지금 어디?

```
[Step 0] 그리기 도구 로직        <-- 완료 (지금 여기)
[Step 1] 지우개                  <-- 다음
[Step 2] XP 그림판식 도구 팔레트 UI
[Step 3] 전체 되돌리기 (Ctrl+Z / Ctrl+Y)
[Step 4] 그림판 룩 실측 매칭 (수동)
```

다음이 지우개인 이유: 이번 단계에서 그린 선에 "지우개로 지울 수 있음(erasable)"
딱지를 미리 붙여 놓았다. 지우개는 이 딱지가 붙은 것만 지우고 스티커·사진·텍스트는
건드리지 않는 방식이라, 딱지 규칙이 완성된 직후가 지우개를 만들 자연스러운
타이밍이다.

## 아직 안 된 것 (솔직하게)

- **화면에서 아직 아무것도 못 한다.** 이번 단계는 로직만 만들었고 버튼·팔레트
  UI가 없어서, 앱을 켜도 그리기 기능이 보이지 않는다. 훅을 화면에 연결하는
  것은 Step 2에서 UI와 함께 한 번에 한다 (지금 연결하면 조작 수단 없는 죽은
  코드가 되기 때문).
- **지우개 없음** — Step 1.
- **되돌리기(Ctrl+Z) 없음** — Step 3.
- **실제 브라우저에서 마우스로 그려보는 검증은 아직 안 했다.** 자동 테스트는
  "선이 완성됐다"는 신호를 인위적으로 발생시켜 규칙을 검증한 것이고, 실제
  드래그로 선이 그려지는 경험은 UI가 붙는 Step 2에서 눈으로 확인해야 한다.
- 에어브러시의 "굵기 x 5 = 점 개수" 배율은 fabric 기본값과 맞아떨어지게 정한
  값일 뿐, 실제 손맛은 Step 2~4에서 그려 보고 조정될 수 있다.
